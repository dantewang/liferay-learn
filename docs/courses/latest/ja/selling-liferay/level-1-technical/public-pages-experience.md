# ログイン前の体験

**概要**

* デモでは第一印象が重要
* 見込み客がLiferayを見たことがない場合、公開ページから始める
* 利用可能な時間を明確に理解する
* ユースケースがわかっている場合は、そのユースケースに沿ったデモを使用する。
* ユースケースが分からなくても、常に魅力的なルック＆フィールのデモを使用する。

```{note}
* 本モジュールのライブワークショップの [録画](https://learn.liferay.com/web/guest/d/se1-2-public-pages-experience) をご覧ください。
* 上記ワークショップで使用されたプレゼンテーション資料はこちら [PDF](https://learn.liferay.com/documents/d/guest/se1-2-public-pages-experience-pdf) からダウンロードしてください。
```

## 背景

CEO、CTO、CMO、エンタープライズ・アーキテクトなど、意思決定者でいっぱいの会場でプレゼンをする場合、目標は持続的なインパクトを与えることだ。

このモジュールの目的は、見込み客が最初に目にするページが、永続的なインパクトを与えるようにするために必要なガイダンスを提供することです。

デモは、見込み客が初めてLiferayを目にする場である可能性が高い。 すぐにログインしたくなるかもしれませんが、Liferayのデモの出発点は、ログインする前に、魅力的でコンテンツが豊富なホームページやランディングページであるべきです。

忍耐強く、特定のユースケースを見せるようになるには、通常ログイン後になりますが、ログインの段階を素通りするのは間違いです。なぜなら、トップページはLiferayの多くのアウトオブボックスの機能を紹介する絶好の機会を提供し、見込み客がLiferayが提供する可能性と見たものに興奮するのを助けるからです。

この第一印象は常に記憶されるものであり、チャンスを生かすか殺すかのステップではないかもしれないが、見込み客はこの最初のページを記憶しているのだから、そのチャンスを逃してはならない。

これらの公開ページのさまざまな側面を簡単にフォローできるように、あるいはログイン前の体験を容易にするために、デモ中にカバーするトピックは以下のセクションに分類されています：

* ホームページまたはランディングページ
* その他の公開ページ
* ナビゲーションと検索
* テーマとスタイル
* スケーラビリティ

## デモの紹介

おそらく見込み客が実際のLiferay製品を目にするのはこれが初めてでしょうから、最初に見せたいくつかの画面と最初に述べたいくつかの事柄が永続的なインパクトを持つことを忘れないでください。

重要なのは、アジェンダやデモで取り上げられる内容を通じて、期待値を設定することである。 これがないと、見込み客が後で取り上げるトピックについて質問してくることがよくある。

デモを始める前に知っておくべき本当に重要なことは、どのくらいの時間が使えるかということだ。 理想的なのは、ほとんどの場合、セッションの前にそれがわかっていることですが、明確でない場合は、尋ねることは決して悪いことではありません。 最悪のシナリオは、デモの最も価値のある部分に到達するのが、聴衆のほとんどが退席した後になってしまうことだ！

最初の画面を見せる前に、最初の1、2分を使って、これから見せるデモについて話す。 見込み客が会いたいと言ったものが具体的なものであれば、そのことを再確認し、時間が有効に使われていることを確認する。

あるいは、これはLiferayのアウトオブボックス・テンプレートの一つを使った入門デモかもしれません。 このモジュールではマスタークラス・テンプレートが使われているので、次のように導入することができる：

"今日はMasterclassというLiferayのテンプレートの1つを使います。 Masterclassは、Liferayのすぐに使える機能を使って構築された学習管理ソリューションです」。

顧客が解決しようとしている問題を発見する機会がない場合、これは予備的な位置づけである。 もし整列されていないデモを使用する場合は、必ずその理由を説明すること。ユースケースが共有されていないためであれば、そう言うこと。 アウトバウンドで発生した機会で、見込み客が興味を持っているが、解決すべき問題があるかどうかまだわからない場合は、その旨を伝える。
