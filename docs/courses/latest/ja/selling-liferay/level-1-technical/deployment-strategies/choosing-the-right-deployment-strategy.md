# 正しい配備戦略の選択

**概要**

* Liferayの好ましいアプローチは、まずSaaSである。
* それぞれの見込み客固有の要件や状況によっては、Liferay Experience Cloud Self-Managed、またはDXP Self-Hostedの方がより適切に実装できる場合があります。
* Liferayは、SaaSにつながる主な検討事項と、SaaSにならない場合を特定するその他の事項を特定した。
* 展開戦略は、一般的にデモや初期段階の販売会議では議論されないが、これらのセッションで学んだ情報は、機会を1つの展開アプローチまたは別のアプローチに導くのに役立つ。

```{note}
* 本モジュールのライブワークショップの [録画](https://learn.liferay.com/web/guest/d/se1-8-deployment-strategies) をご覧ください。
* 上記ワークショップで使用されたプレゼンテーション資料はこちら [PDF](https://learn.liferay.com/documents/d/guest/se1-8-deployment-strategies-pdf) からダウンロードしてください。
```

## SaaS、PaaS、それともセルフホスト？

上記で取り上げた導入オプションを検討した結果、ライフバンクにとって適切なオプションはSaaSアプローチであることが明らかになった。なぜなら、ビジネス上の問題に対処するために必要なアプリケーションの構築に労力を集中できるからである。

この場合、Liferay Experience Cloud SaaSは論理的で明白な選択でした。 現実のシナリオでは、物事はそれほど明白ではないかもしれない。

デモやディスカッションでは、見込み客とすべての展開方法の詳細を説明しないことが重要である。

その代わり、デモのような機会は、見込み客が何を言っているのかを聞き、彼らにとって重要なことを知るために使うべきである。

それが、デモの文脈におけるこのセッションの本当のポイントだ。 もちろん、これは一般的なセールスの重要な部分のひとつである。話をするときやデモを運転するときでさえ、耳を傾け、学び、より多くの発見をすることが重要なのだ。

見込み客との仕事が増えるにつれて、最も適切なアプローチが明らかになるはずだが、機会を正しい方向に導くための情報も増える。

## 正しい配備戦略の選択

> すべての展開オプションが可能ですが、LiferayのアプローチはSaaS Firstです。

このアプローチの背後にある理由は単純です。顧客はビジネス上の問題を解決することに集中する必要があり、それはインフラやプラットフォームソフトウェアではなく、Liferay DXPアプリケーションに依存しているからです。

すぐに使える機能と拡張の可能性を探ることは、一般的にLiferayが見込み客に適しているかどうかを判断するのに役立ちます。

### カスタマイズされたソリューションの構築

最も最近の変更の一つは、Liferay DXPを拡張する好ましい方法です - [Client Extensions](https://learn.liferay.com/w/dxp/building-applications/client-extensions) として知られているものですが。

ほとんどすべての顧客が、その顧客独自の機能を必要とします。新しいアプローチは、すべてのカスタムコードがLiferayのコアの外側で実行される疎結合モデルを提供します。

これは、Liferayの伝統的な強みの1つである、特定のニーズに合わせて拡張したりカスタマイズしたりする能力を変えるものではありません。

ほとんどの場合、新しいアプローチは適合する。これは、見込み客をLiferay Experience Cloud SaaSに導く意思決定プロセスの一部となるだろう。

しかし、別のアプローチでしか実現できない特定のユースケースもあるでしょう。カスタムのJava OSGiモジュールを構築する従来のアプローチを使用してカスタマイズすることは可能です。このアプローチが必要な場合は、Liferay Experience Cloud SaaS以外のデプロイアプローチを検討することになります。

## Liferay ExperienceクラウドSaaSが正しいアプローチである場合

すでに述べたように、Liferay DXPの **好ましい** デプロイアプローチはLiferay Experience Cloud SaaSであり、これをサポートするために3つの主な質問があります：

1. この問題はOOTB Liferayの機能で解決できますか？
2. カスタマイズの要件は、エクステンションの望ましいアプローチで解決できるか？
3. 今後の機能（ロードマップまたはリクエスト）で解決できますか？

上記の通りであり、そもそもSaaSアプローチに抵抗がないのであれば、推奨される導入アプローチはLiferay Experience Cloud SaaSである。

## Liferay ExperienceクラウドSaaSが適切でない場合

また、SaaS以外の導入アプローチを検討する際の参考となる点もいくつかある。

高いレベルで言えば、これらの考慮事項は以下の通りである：

* SaaSは単純に選択肢に入らないというわけではなく、法的な理由や見込み客の嗜好に関係している場合もある。 単に、SaaS全般に対して強い反対があるだけかもしれない。
* カスタマイズには、従来のJava（OSGi）開発が必要になる場合がある。
* データ主権に関する懸念があるかもしれない-例えば、現在ローカルデータセンターがない場合など。
* データ・セキュリティやデータ管理の問題がある可能性がある。

これらの考慮事項のいずれかが存在する場合、SaaSはアプローチではないかもしれません。このような場合、Liferay Experience Cloud - Self-Managed（PaaSが受け入れられる場合）またはDXP Self-Hostedを検討することができます。

しかし、これらの検討事項や反対意見を単に額面通りに受け取るのではなく、しっかりと確認し、必要であればLiferayのソリューション・コンシェルジュ・デスクで検証することが重要です。

さらに、製品や市場が進化し、SaaSがより受け入れられるようになれば、こうしたケースは徐々に減っていくはずだ。

## 新規顧客および新規プロジェクトの販売決定フロー

Liferayは、新規顧客や既存顧客との新規プロジェクトで使用する販売意思決定フローを開発しました。 パートナーはこのフローを利用して、それぞれの顧客やプロジェクトに最適なアプローチを決定できます。 これにより、SaaS、PaaS、マネージドサービスを備えたPaaS、またはセルフホスト型のいずれかを選択する主要な決定ステップに対応し、どのデプロイ方法が機会に適しているかを示すガイダンスを提供します。また異常値についてはソリューションコンシェルジュデスクに誘導します。

## 概要

デモ、ディスカバリー・ミーティング、フォローアップ・ミーティングなどを実施する際には、可能性のあるすべての展開オプションを念頭に置くことが重要であり、過去に実施されたことに基づいて見込み客を方向性に誘導してはならない。

カスタマイズに特定のテクノロジーを使ったり、UI開発に特定のアプローチをとったりと、既存の知識を利用するのは簡単だが、古いアプローチは見込み客にとって最良の結果をもたらさない可能性があることを常に念頭に置いておく必要がある。

どんなデモでも、フィードバックや質問から学ぶこと。 これは、見込み客に最適な配備方法を提案する際の意思決定プロセスに役立つ。

おめでとう、これでこのモジュールは終わりだ。 次に、Liferay [Success Stories](../success-stories-analysis.md) を分析し、焦点を絞ったデモを提供する方法について学びます。
