# ディスカバリー・コール

**概要**

* ディスカバリー・コールは以下の目的で使用される。
  * クオリファイと基本的なディスカバリーの実行
  * 次のデモで何を見せる必要があるのかを発見する。
* セールス・エンジニアは両方のフェーズに関わるべきである。
* ディスカバリーコールにおける技術的な資格は、全体的なアーキテクチャにおけるLiferayの位置づけ、技術スタックの好み、全体的なソリューションアーキテクチャにおける適合に関連する。
* デモ・ディスカバリーは、オーディエンス、ペルソナ、要件、UX & ユーザー・ジャーニーのマッピング、テクノロジー・スタック、現行システム、サイジングの理解に重点を置く。

```{note}
* 本モジュールのライブワークショップの [録画](https://learn.liferay.com/web/guest/d/se1-1-discovery-call) をご覧ください。
* 上記ワークショップで使用されたプレゼンテーション資料はこちら [PDF](https://learn.liferay.com/documents/d/guest/se1-1-discovery-call-pdf) からダウンロードしてください。
```

## 背景

Liferayのデモを発表するために必要な作業は、すべてディスカバリーコールから始まります。

ディスカバリー・コールについて話すとき、最も一般的な目的は、機会全般の発見と適格性確認である。 この文脈では、 **必ずしも** 新しい機会を発見することや、機会を適格に見極めることを意味するわけではないが、もちろん、それは多くの場合可能であり、そうなるであろう。  ディスカバリー・コールのもう一つの重要なアウトプットは、見込み客にデモを行う準備として必要な要件を収集することである。

このモジュールは、有意義なデモを提供できるように、ディスカバリー・コールから何を得るべきかを理解できるように設計されています。 同時に、収集した情報は営業チームの他のメンバーとも共有し、機会を適格化するのに役立てるべきである。

セールス・エンジニアであれ、プリセールス・コンサルタントであれ、あるいはパートナー企業の場合はインプリメンテーション・コンサルタントであれ、技術的なセールスの役割のほとんどは、セールス・エンジニアがセールス・プロセスに関与する前に、最初の資格認定が行われています。

最初の資格認定は、BDR、SDR、アカウント・エグゼクティブなどが行ったかもしれない。 しかし、セールス・エンジニアをプロセスの比較的早い段階から関与させることが重要であり、特にセールス・エンジニアを最初の **技術的な** 資格認定プロセスの主要な部分として参加させるべきである。

機会を発見することは、セールス・エンジニアリング・プロセスの始まりに過ぎない。

このモジュールは、発見のさまざまな段階を反映し、大きく2つのパートに分かれている：

1. 機会の発見
2. デモの要件を発見する
