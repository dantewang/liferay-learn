# Liferayの価格

**概要**

* Liferayは、Liferay DXPを含む3つのデプロイアプローチを提供します：
  * Liferayエクスペリエンスクラウド（SaaS）
  * Liferayエクスペリエンスクラウドセルフマネージド（PaaS）
  * DXP Self-Hosted
* それぞれの展開方法には、独自の価格モデルがあります。
* パートナーとの共同販売は、Liferayの販売戦略の重要な要素です。
* 価格設定モデルを理解することは、機会に対するボールパーク・プライシングを作成するのに役立ちます。

## 背景

このモジュールでは、Liferayの3つの主要製品と製品の価格について説明します。 修了後は、大まかな価格を提示し、最も一般的な価格に関する質問に答え、見積書を作成することができるようになります。

さらに、このモジュールでは、価値を正当化し、容量やユーザーなどに関する顧客のニーズを理解するために、どのようなタイプの質問をし、どのような会話をすべきかを取り上げ、適切なオファリングとサイジングの選択に役立てる。

詳細ではあるが、これはまだ **価格の概要** 。 パートナーはLiferayチームの適切なメンバーと協力し、価格設定の詳細を確認してください。 Liferayの社員は、関連するセールスマネージャーと協力してください。

```{note}
* 本モジュールのライブワークショップの [録画](https://learn.liferay.com/web/guest/d/sa1-2a-liferay-pricing-saas) をご覧ください。
* 上記ワークショップで使用されたプレゼンテーション資料はこちら [PDF](https://learn.liferay.com/documents/d/guest/sa1-2a-liferay-pricing-saas-pdf) からダウンロードしてください。
```

## パートナーとの共同販売

Liferayは、共同マーケティング、共同デリバリー、共同販売を意味するPartner Together戦略に取り組んでいます。 Co-sellingとは、Liferayがパートナーと共にパートナーの機会を提供することです。 新規顧客か既存顧客かによって、チームの構成は異なる：

**新しいロゴの機会：**

* チャネルアカウントマネージャー(CAM)および/またはLiferayアカウントエグゼクティブ(AE)が販売をサポートします。
* イネーブルメント・チームのセールス・エンジニアやソリューション・アーキテクトがサポートすることもあります。

**既存顧客の機会：**

* Liferayカスタマーサクセスマネージャー（CSM）はお客様とパートナーをサポートします。

このチーム体制により、パートナーが単独で価格設定を決定するような事態は起こらないはずだ。

### 価値を売る

ほとんどの買い手は、購入プロセスの最初に価格設定を知りたがるものだ。

なぜなら、この段階では顧客のニーズが十分に理解されておらず、この理解なくして正しい価格をつけることは不可能だからである。

もし買い手が製品の **価値** を理解せず、価格を **コスト** として解釈し、 **投資** として解釈しないのであれば、買い手はそれを行う必要がある。

すべての顧客の質問に即座に答えられることは期待されていませんし、価格を即座に提示する必要もありません。 より重要なのは、顧客のニーズを理解することであり、顧客が特定の質問をする理由を理解することである。

特に案件の初期段階では、買い手と詳細を共有する前に、社内（パートナーはリフェレイチームと）、そして様々な買い手と、ニーズとソリューション、オファーの内容について何度も戦略を練ることをお勧めします。

しかし、真剣な顧客が見積もり価格を要求した場合、それを提示することは可能でなければならない。

投資能力の低い見込み客に見積もり価格を提示することも、早期の失格戦術としては有効である。

### ディスカバリーに関する基本的な質問

ドラフト価格に関する基本的な質問には、以下のようなものがある：

* ソリューション ユースケース
* 成功の指標
* テーラリング／カスタマイズの要件
* エンドユーザー・タイプ & 数量：登録/匿名
* 保管量、トランザクション数、アクティビティの種類
* サポート要件
* ミッションの重要性
* など。

**ソリューション ユースケース**

見込み客のニーズを理解することは本当に重要で、彼らが探しているソリューションのタイプは、主要な価格指標のいくつかを知るのに役立ちます。 認証されたアクセスが必要かどうか、コマース（取引）機能が必要かどうか、トラフィック（匿名のページビュー）が多いかどうかなどは、すべて価格に影響する。

例えば、見込み客は公開ウェブサイトを構築しようとしているかもしれない。 そのサイトは、認証されたアクセスはほとんどないが（例えば、コンテンツのニュースサイト）、ページビューは多い可能性がある。

あるいは、コマース・ポータルの場合、認証されたアクセスが多く、購買が必要なソリューションが考えられるが、その場合はトランザクションも必要となる。

**エンドユーザーの数量とタイプ**

毎月ログイン可能な登録ユーザー数を把握することは、料金設定を計算する上で重要な出発点となる。 登録ユーザーに対する価格設定は、MALU（Monthly Active Logged-in Users）という概念を使用します。 すべての登録ユーザーが毎月ログインするわけではないので、月間登録ユーザー数が直接的に関連するわけではないが、月間ベースでどれだけのアクティブユーザーがいるのかを示す指標となり、それが年間バンドに累積される。

このユーザーベースのうち、どの月にどの程度の割合がアクティブ（＝ログイン）なのかを推定できれば、さらに精度を高めることができる。

匿名トラフィックの場合、ユーザー数はページビュー数よりも重要度が低いので、この情報があれば価格設定にも役立ちます。

**サポートとミッションクリティカル**

アプリケーションのミッションクリティカル度やビジネスクリティカル度は、通常、要求されるサポートレベルに直結する。 ミッション・クリティカルなアプリケーションやビジネス・クリティカルなアプリケーションの場合、顧客は通常、問題に対して24時間365日のサポートを必要としている。

**その他の考慮事項**

その他に考慮すべき点としては、ソリューションに必要なストレージの容量、どのようなカスタマイズが必要か、カスタマイズの回数などがある。

## Liferayの展開モデル

Liferayは3つのデプロイメントモデルをサポートしています：

* Liferayエクスペリエンスクラウド（SaaS / DXP-as-a-Service）
* Liferayエクスペリエンスクラウドセルフマネージド（PaaS）
* Liferay DXPセルフホスティング（DXP SH）

![Liferayの3つの導入モデルは、Liferay Experience Cloud SaaS、Liferay Experience Cloud Self-Managed（PaaS）、DXP Self-Hostedである。](./liferay-pricing/images/01.png)

### SaaSファースト

LiferayのアプローチはSaaS第一主義だ。 これが可能かどうかを簡単に見極めるために、営業チーム内で話し合うことができるいくつかの簡単な質問があります：

* 顧客のビジネス上の問題は、Liferayのすぐに使える機能で解決できるか？
* それとも、カスタマイズに望ましいアプローチであるクライアント・エクステンションを使用して、これらの要件を満たすことができるのでしょうか？
* あるいは、すでにロードマップに載っている機能、あるいは顧客が要望し、販売サイクルの間に構築できる機能など、今後の機能で問題を解決できるだろうか？

これらのいずれかが当てはまり、顧客がそもそもSaaSに慣れているのであれば、Liferay Experience Cloud SaaSを提案すべきである。

SaaSであることが不可能なシナリオも考えられる：

* 顧客はSaaSは選択肢にないと言うかもしれない。
* データをめぐる懸念があるかもしれない：
  * 保管場所
  * どのように保護されているか
  * 誰が管理しているのか

このような場合、PaaS - Liferay Experience Cloud Self-Managedが選択肢になるかもしれませんし、Liferayのクラウド全般が選択肢にない場合は、DXP Self-Hostedが利用できます。

また、 [営業決定フロー](https://learn.liferay.com/documents/d/guest/new-customer-new-project-sales-decision-flow) も、このステップを支援するために使用できることを忘れないでください。

![Liferay の新規顧客と新規プロジェクトの販売決定フローは、新規のオポチュニティにどの展開モデルが最も適しているかのガイダンスを提供します。](./liferay-pricing/images/02.png)
