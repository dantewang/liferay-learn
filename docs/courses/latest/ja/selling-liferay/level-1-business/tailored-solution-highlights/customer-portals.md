# カスタマーポータル

カスタマー・ポータルは、顧客がベンダーから購入する製品やサービスに関するすべての関連情報へのシングル・アクセス・ポイントを提供します。

ここで重要なのは、組織が提供する製品／サービスの顧客またはエンドユーザーが、ソリューションの主要ユーザーであるということである。

## セグメンテーションとパーソナライゼーション

カスタマーポータルの重要な目標のひとつは、顧客に合わせた体験を提供することである。 LiferayはOOTBセグメンテーションとパーソナライゼーション機能を提供し、適切なコンテンツが適切なオーディエンスに配信されるようにします。

さまざまなタイプのバイヤーがサイトにログインすると、それぞれの要件に合わせた情報が表示される。

**テクニカル・バイヤー・ダッシュボード**

![LiferayBoticsのデモのダッシュボードには、IoT統合によって委託されたロボット、最近の注文、出荷、タスクと通知が表示される。](./customer-portals/images/01.png)

テクニカル・バイヤーはダッシュボードを見ることで、機器のステータスを素早くチェックし、消耗品を再注文し、新製品やサービスについて知ることができる。

**ビジネス・バイヤー・ダッシュボード**

![LiferayBoticsデモのダッシュボードには、最近の請求書と製品情報が表示されている。](./customer-portals/images/02.png)

ビジネスバイヤーが同じページにアクセスすると、パーソナライズされたダッシュボードが表示され、注文状況や出荷状況、請求書や支払いにすぐにアクセスできる。

Liferayは、人口統計データ（例：国、性別、言語）や行動パターン（例：訪問ページ、デバイス、最終ログイン日）に基づいてセグメントを作成するためのセグメントビルダーなど、この望ましいビジネス成果を達成するのに役立つ主要な機能を提供します。 ユーザーは、任意のページで必要な数だけ異なるエクスペリエンスを作成することができ、それぞれが異なるユーザーセグメントにリンクされている。 これらの異なるエクスペリエンスはそれぞれ、ユーザーのニーズに応じて異なるウィジェット、フラグメント、データを表示することができる。

**統合と相互運用性**

顧客ポータルの重要な価値の1つは、企業全体で使用されている多くの異種システムをまとめ、統一することです。したがって、顧客ポータルが統合に優れたテクノロジーで構築されていることが重要です。

**IoT機器との統合**

![委託されたデバイスの詳細には、IoT統合によって収集されたパフォーマンス、デバイスの健全性、進行中のジョブが表示されます。](./customer-portals/images/03.png)

統合にはさまざまな形がある。 例えば、顧客は現場や製造工場のIoT機器との統合を望んでいるかもしれない。 この統合は、重要な情報を素早く表面化し、消耗品やスペアパーツの再注文の促進や、機器の寿命を延ばす（そうすることで、その機器への投資価値を最大化する）ための予防的サービスのスケジュールを立てるのに役立つ。

**記録システムとの統合**

![発注済み注文リストには、記録システムとの統合による注文が表示されます。](./customer-portals/images/04.png)

顧客ポータルの統合には、バックオフィスの記録システムから主要な注文や請求データを表示することも含まれる。 多くの場合、これらのバックオフィスシステムはセルフサービスを直接提供しないか、セルフサービス体験に多くの不満が残る。 これらのバックオフィスシステムのフロントエンドとしてLiferayを使用すると、顧客は一貫したユーザーエクスペリエンスとブランディングで適切なレベルのアクセスを得ることができます。

これにより、顧客ポータルは、ビジネス関係に対する比類のないビューをユーザーに提供し、将来の購入を妨げる可能性のある摩擦を軽減する、ガラスの一枚板となる。

## ケーススタディ - エアバス

![エアバス・ヘレコプターズのケーススタディでは、課題と主な目的を明らかにし、ソリューションについて説明し、主な成功指標を提示しています。](./customer-portals/images/05.png)

エアバス・ヘリコプターズは、セグメンテーションとパーソナライゼーション、そして統合と相互運用性の両方を効果的に活用した顧客ポータルの好例である。 エアバス・ヘリコプターズは、民間および軍用ヘリコプターのソリューションを顧客に提供している。 2016年、エアバス・グループのこの部門は、顧客ポータルを再設計し、24,000人のユーザーを抱えるヘリコプター運航会社のカスタマージャーニーに適応した、より近代的なプラットフォームを提供するためにLiferayを採用した。

エアバスが解決すべき重要な課題は、顧客との関係を改善し、業務を簡素化することだった。 ユーザーにはさまざまなニーズがあり、エアバスは、ユーザーによりパーソナライズされた体験を提供できる最新のカスタマーポータル・ソリューションを求めていた。 エクスペリエンスをパーソナライズすることで、ユーザーへのオンボーディングと教育のプロセスは非常に簡単になり、顧客は真の生産性向上を実現した。

もう1つの課題は、ヘリコプターのオペレーターが多くの異なるサードパーティのビジネス・アプリケーションにアクセスしなければならないことだった。 ユーザーは、複数のログインと異なるユーザー・エクスペリエンスで、異なるアプリケーション間を行き来していた。 エアバスは、これらのアプリケーションを統合し、単一のカスタマージャーニーの一部として提示できるプラットフォームを必要としていた。

Liferayの役割や人口統計、行動属性によってユーザーをセグメント化する機能を活用することで、エアバスは適切なタイミングで適切な情報を表示するカスタマージャーニーを提供しました。 また、Liferayの優れた統合機能を利用して、サードパーティのアプリケーションが顧客ポータルのコンテキストに表示されるようにし、エンドユーザーによりまとまりのある体験を提供しました。

より詳細な情報については、liferay.comの [Airbus Helicopters](https://www.liferay.com/resources/case-studies/m-1) 。
