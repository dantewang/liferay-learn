# 求める顧客像

本モジュールでは、Liferayが求める顧客像を紹介し、Liferayが提供するサービスで求める顧客像を顧客の以下の情報からまとめます。

1. 企業・団体タイプ
1. 業界
1. ソリューションのニーズ
1. 複雑さ
1. その他のニーズ

上記のような情報により、売り手は案件やRFI／RFQを見込み、事前に適切に認定できるようになります。

```{note}
* 本モジュールのライブワークショップの[録画](https://learn.liferay.com/web/guest/d/l0-3-ideal-customer-profile) をご覧ください。
* 上記ワークショップで使用されたプレゼンテーション資料はこちら [PDF](https://learn.liferay.com/documents/d/guest/l0-3-ideal-customer-profile-pdf)からダウンロードしてください。
```

## 販売と引き際

自社の強みと弱み、そして販売製品の強みと弱みを把握することが重要です。

無限のリソースがない限り、すべての案件を勝ち取ることは不可能であるため、案件や見込み顧客に優先順位をつける必要があります。

* 顧客要件を解決するには、Liferayを他のテクノロジーと併用したり、他のパートナーと協力したりする必要があります。
* 案件を得る立場にない場合、その案件から手を引かなければならないこともあります。
* 現実に基づいていない販売戦略は、成功する確率が低くなります。 現実主義と楽観主義は対立するものではありません。

　 販売におけるいろいろな段階で現実を直視することは、勝利を確実にするために戦略を調整したり、あるいは勝ち取れる案件に時間と資金を再配分したりするのに役立ちます。

クライアントの購買基準を満たしたり、効果的に変更したりするには、クライアントのニーズをよく理解することが必要です。 また、弊社のサービスがそのニーズにどれだけ応えられるかを把握しなければなりません。

* ひとつのベンダーがすべての買い手の要求をすべて満たすことは考えられません。
* 重要なのは、ニーズを十分に満たし、競合他社よりも優れていることです。
* 大型契約の獲得には多額の費用がかかります。
* コンペで次点となった場合、契約を勝ち取るのと同じくらいのコストがかかりますが、最後には何の報酬もありません。

まずは、Liferayがどこで能力を発揮しているか把握しておきましょう。
