# ポータルの遺産とLiferayの調整

**概要**

* Liferayはカスタマイズが容易です。
* Liferayは、多くのオーディエンスとユースケースにエクスペリエンスを提供することに優れています。

DXPを取り巻く環境は、コンテンツ管理システムのプロバイダーとしてスタートしたベンダーや、CRMやERPのバックグラウンドを持つベンダーが大半を占めている。 Liferayは、強力な水平ポータルの伝統を持っている点で異なる。

Liferayのポータルの伝統は、私たちにユニークな視点を与えてくれます：

* 強力な統合
* 顧客関係重視
* 安全なデータの取り扱い
* デジタルの敏捷性

## 統合機能

デジタル・エクスペリエンス・プラットフォームは、企業がデジタルトランスフォーメーションを受け入れるための力を与えてくれる。 効果的なデジタルトランスフォーメーションの鍵は、異種システム間の障壁を取り払い、シームレスな体験を提供することである。

Liferayのようなポータル・ヘリテージ・プラットフォームは、バックエンドやレガシーシステムの統合に優れています。 これにより、顧客、従業員、その他のエンドユーザーは、企業内の複数のアプリケーションにわたって必要なデータにアクセスできるようになる。

例えば、CRMや企業のマーケティングオートメーションツールにロックされた顧客データは、ポータル・ヘリテージDXPを通じて簡単に統合・共有することができる。 これは、企業や顧客の時間と労力を節約し、企業がターゲットとする視聴者をよりよく理解し、サポートするのに役立つ。

## 長期的な顧客関係重視

ポータルは、セキュアなインターフェイスを通じて、企業とターゲット・オーディエンスを結びつける。 パーソナライゼーションは、ユーザーが必要とするコンテンツや機能を提供する。 CMSテクノロジーは従来、匿名の見込み客にコンテンツを提供することに重点を置いてきたため、こうした認証されたエクスペリエンスに苦労することがあった。

## セキュアなデータ処理能力

CMSとは異なり、ポータルは多数のユーザーと情報の流れをサポートし、しばしば顧客や企業の機密情報を扱う。 ポータル・ヘリテージDXPは暗号化されたシステムで構築されており、安全で洞察に満ちた情報管理のために、データの保存と分析の両方に対応している。

## デジタル・アジリティ

ガートナーが強調しているように、ポータルは共通のアーキテクチャ・フレームワークとして使用できる。 企業はウェブサイトを構築し、作成したフレームワークをすぐに再利用して、より多くのタッチポイントを構築することができる。 これにより、すべてのデジタル・プレゼンスの機能と外観が統一され、ゼロからの再構築を回避することで時間とコストを節約することができる。

ポータル・ヘリテージ・デジタル・エクスペリエンス・プラットフォームをソリューション・ファクトリーにすることで、顧客は個々のエクスペリエンスの質をコントロールしながら、デジタル戦略を迅速に進めることができる。

## 仕立てが簡単

Liferayは、最も柔軟で拡張性の高いDXPとして認められています。

Liferayをカスタマイズするアプローチは時代とともに変化し、柔軟性を犠牲にすることなく使いやすさを向上させてきました。 歴史的に、Liferayを拡張するプロセスはこのように発展してきました：

* ソースの変更
* EXT環境
* プラグインSDK
* 延長ポイント
* クライアント拡張

### エクステンションの進化

当初、Liferayをカスタマイズするユーザーはソースコードから製品を構築し、何でも変更することができました。 この究極の柔軟性は、独自のバグを導入する可能性によってユーザーに犠牲を強いることになり、新しいバージョンに簡単にアップグレードすることも難しかった。

一歩前進するごとに、プラットフォームを拡張するために必要な知識を減らし、同時に問題を引き起こす可能性を減らしていった。 EXT環境は、Liferayのコードの上にユーザーコードを重ねることで、Liferayをカスタマイズした。 これによってユーザーコードはLiferayのコードから分離されましたが、ソースを直接修正するのと同じ欠点が多くありました。 次にPlugins SDKが登場し、実行時にLiferayの動作を変更するインストール可能なファイルへのカスタマイズを保持するようになりました。

Liferay DXP 7.0はOSGiを導入した。 これにより、カスタマイズはモジュラーモデルに移行し、Liferayのコアコードから隔離された "コンテナ "に保管され、Extension Pointを介して通信できるようになりました。 例えば、開発者はコマースコンテキストに新しい支払い方法や注文ルールを実装したり、ドキュメントやメディアを保存するための新しいストレージメカニズムを実装したりすることができる。

しかし、こうした改良のすべてが、アップグレードを困難なものにしていた。 このプロセスにおける最新のステップは、最も強力で保守性の高い拡張メカニズムとして、クライアント・エクステンションを導入することです。 Client ExtensionsはLiferayのコアコードとは別に実行されるため、アップグレードが容易で、Liferay DXPをSaaSプラットフォームとして提供することができます。

## 多くのオーディエンスとユースケースにエクスペリエンスを提供する

多くのDXPベンダーは、主に一般向けウェブサイトなどの消費者ベースのソリューションに注力している。 Liferayのポータルの伝統は、複数のシステムからのデータと機能を組み合わせた認証されたエクスペリエンスを必要とするビジネスベースのソリューションに集中することを可能にします。

ビジネスに特化した認証ソリューションの例

* カスタマーポータル
* サプライヤー・ポータル
* パートナー・ポータル
* イントラネット・ソリューション

もちろん、これは完全なリストにはほど遠い。

これらのB2Bソリューションはすべて、要件や統合の必要性に固有の複雑さを備えている。 この複雑さに対応できるソリューションに注力することで、Liferayは最も一般的なB2Cのユースケースにも対応できる体制を整えている。
