# 強みと注意点

本モジュールでは、Liferayのサービスに特に適したニーズ、購入者、状況のほか、注意すべき点についてまとめます。

* ユースケース：単純か、複雑か。
* ベンダーの組み合わせ：1社のみか、フレキシブルに対応か。
* 購買プロセスを推進するのはITかマーケティングか。
* 主にB2Cか、B2B、B2E、B2B2Cなど混在しているか。
* ターンキーアプリケーションかカスタマイズソリューションか。
* 統合の必要性：単純か、複雑か。
* オープンソースのサポートがあるか。
* 見込み顧客への直接的なアクセスか、間接的なアクセスか。
* 既存のサポートされていないポータルか。
* 中小企業か組織か。
* 複雑なガバナンスやセキュリティのニーズがあるか。
* 製品の適合性はどうか。
* ポータルとコマースの連携はどうか。

## ユースケース

Liferayが得意とするのは、複数のユースケースのニーズ、セルフサービスのニーズ、あらゆるタイプの複雑性です。

逆に、シンプルなWebサイトを作るだけなら、もっと安いオプションもあります。 その場合でも、顧客が将来的にカスタマーポータルや社内ポータルのニーズを持っているのであれば、複数のソリューションに単一のプラットフォームを使用することの長期的なメリットを検討するよう、提案する価値があります。

## ベンダーの組み合わせ

もし見込み顧客がすでに複数のベンダーの技術を導入しているのであれば、このことは彼らが1つのベンダーに完全にコミットしているわけではないことを示しています。 しかし、単一のテクノロジーベンダーと強く結びついている場合、そのベンダーに不満がない限り、他社を選んでもらうのは難しいでしょう。

## 購買プロセスを推進しているのは誰か。

やや単純化されていますが、IT担当が購買プロセスを主導している場合、通常Liferayにとってプラスになります。

しかし、マーケティング担当が購買プロセスを完全に推進している場合は、さらに困難になる可能性があります。というのも、Liferayがネイティブに備えていないマーケティング機能を期待されるかもしれないからです。 このような場合、Liferayの強力な統合実績により、これらの機能を全体的なソリューションに組み込むことができ、状況を有利に変えることができます。 マーケティング主導の購買プロセスでは、さらに深く掘り下げて、技術的な購買への影響を見つけることが重要です。 これらは実装時に必要となり、役立つ可能性があります。

これは、よりビジネスに重点を置くことが多いSaaSのバイヤーにも当てはまります。カスタマイズされたソリューションには、やはりある程度の技術的な専門知識が必要となります。

## 消費者向けまたは広範囲か

B2B、B2B2C、B2E、G2C、G2B（またはこれら複数）のユースケースは、多くの場合、Liferayに適していますが、 **純粋な** B2Cの場合、最適ではありません。

## ターンキーアプリケーションまたはカスタマイズソリューション

一般的に、エンタープライズのニーズとスケーラビリティの要件に加え、柔軟性とカスタマイズのニーズがあれば、Liferayの立場は強化されます。 しかし、そのニーズが主にターンキーアプリケーションやポイントソリューションのようであれば、Liferayにとって適切ではない可能性があります。 このような例と同様に、カスタマイズなしで本当にニーズを満たせるのか、買い手に問いかける価値があります。

## 統合

Liferayの統合と相互運用性の機能は強力なため、複雑で複数の統合ニーズがあれば、Liferayが最適です。

しかし、ネイティブ統合の必要性がある場合は、Liferayの最善のシナリオではないかもしれません。ただし、十分なレベルで顧客のニーズをカバーしているネイティブ統合が存在することはまれであることに注意してください。

## オープンソース

明らかに、買い手がオープンソースを希望しつつ、エンタープライズレベルのサポートとサービスを必要としている場合、Liferayにとって非常に大きなチャンスです。

## 見込み顧客へのアクセス

一般的に、見込み顧客と弊社の間に買い手である企業やコンサルタントがいる場合よりも、見込み顧客に直接アクセスできる場合は良い兆候ではありますが、それが課題となる可能性があります。

つまり、見込み客に直接アクセスできさえすれば、いつでも容易に進みます。

## 既存のポータル

Liferayにとって、もうひとつの大きなチャンスは、サポートされていないレガシーポータルがある場合です。特にそれがエンタープライズである場合です。 エンタープライズポータルソリューションには優れたオプションがそれほど多くありません。

## 企業の規模

買い手が中小企業や小規模組織である場合、Liferayのようなプラットフォームにかける十分な予算と十分なリソースを持っていることは稀です。

投資の期待に関する議論は、案件のかなり早い段階で行う必要があります。

## ガバナンス & セキュリティ

複雑なガバナンスが必要な場合や、多くの異なるユーザーグループが存在し、きめ細かなアクセス権や管理、セキュリティが必要な場合、これらはLiferayに適している指標となります。

## 製品の適合性

一般的に、顧客はデジタルエクスペリエンスプラットフォームを購入するのに市場に出向くことはありませんが、マーケティングオートメーションプラットフォーム、顧客データプラットフォーム、マスターデータ管理、CRMなどのようなものを探していることは明らかです。 Liferayはこれらのニーズの一部をカバーすることはできますが、優れたCRMやCDP、MDMではないことを覚えておいてください。

販売コストや獲得が難しい案件への投資は、獲得できる案件と同額であることを念頭に置いてください。

## コマースとポータル

消費者向けベンダーにとってB2Bコマースの案件を獲得しにくい理由のひとつは、通常、ポータル機能も必要とされることです。 この両方の要件が揃えば、Liferayが有利となる可能性が高いです。

## Liferay Experience Cloud SaaSの理想的な機会を知る

Liferayは営業チーム及び、時には見込み客とも案件の獲得と失注に関するインタビューを行い、Liferayがどこで成功できるか（できないか）の理解を深められるようにしています。

Liferay Experience Cloud SaaSは、比較的新しい製品であり、他の製品よりも成功する機会の種類について定期的に多くの要素がわかってきています。 これまでに、以下のような共通点が確認されています。

* 公開サイト
* コマース機能を搭載
* 統合とローコード機能
* 費用5～10万ドルから開始

### 公開サイト

まず、重要な公開Webサイトを更新することを目指しているが、将来のフェーズで追加のニーズがある見込み顧客は、SaaSサービスに適しています。

### コマース機能

案件獲得に関するインタビューのほぼすべてで、商品カタログなどのコマースとコマース機能の要件は、顧客にとって将来的に、エンドカスタマーのデジタルセルフサービスのニーズを改善・解消するために非常に重要であることが明らかになりました。

顧客は公開Webサイトのプロジェクトから始めたにもかかわらず、ネイティブコマース機能を利用でき、サブスクリプション価格に含まれているというLiferayが提供する価値を高く評価していました。

### 統合とローコード機能

統合をサポートする機能、ローコード機能の搭載、組み込みワークフローエンジン、強力なセキュリティ機能などが、いくつかの案件獲得において言及されました。

### 費用5～10万ドルから開始

Liferay Experience Cloud SaaSの案件における一般的な開始点は、5～10万ドル（ユーロ）から **開始される** 年間サブスクリプション費用にあるようです。

このリストは、網羅的なものでも決定的なものでも **ありません** が、Liferay Experience Cloud SaaS案件を特定する際のヒントになるでしょう。
