## ビジネスの問題を解決する

* お客様のビジネス課題をLiferay DXPで解決
* すべてのチャンスはここから始まる
* ビジネス上の問題を先に考え、展開戦略を後に考える

Liferayソリューションの最終的なポイントは、顧客のビジネス上の問題を解決することです。 機会が始まったとき、その製品が最終的にどのように展開されるかは重要ではありません。重要なのは、その問題がLiferay DXPで解決できるかということです。

その答えが "イエス "なら、それは素晴らしいスタート地点だ。 この後のモジュールでは、理想的な顧客像、当社独自の価値提案、当社が取り組む課題について取り上げます。

しかし今は、見込み客についてもっと知ることが、最適な展開方法を選択するのに役立つだろう。

## オーダーメイド・ソリューションの構築

Liferay Experience Cloud SaaSを導入アプローチとして選択した場合でも、Liferayで実装されたソリューションはターンキーアプリケーションではありません。 Liferay DXPは、箱から出してすぐに広範な機能を提供します。 どのような場合でも、ソリューションの構築を開始するには、これを設定する必要がある：

* ページを追加する必要がある
* コンテンツは作られなければならない
* ユーザーとロールの定義
* など。

しかし、ほとんどの場合、ビジネスのすべての問題がこれらの機能だけで解決するわけではない。 Liferay DXPの主な強みは、常に特定のニーズに合わせてプラットフォームを拡張・カスタマイズできることです。 プラットフォームの拡張には、 [Client Extensions](https://learn.liferay.com/web/guest/w/dxp/building-applications/client-extensions)を作成するのが望ましい。 Client ExtensionsはLiferayのコードに縛られず、どのデプロイアプローチでも使用できます。 クライアント拡張機能により、開発者は慣れ親しんだテクノロジーを使用することができ、コアプラットフォームへのアップグレードを容易に行うことができます。

![クライアント・エクステンションは疎結合で、最新のテクノロジーを使用し、容易なアップグレードをサポートする。](../liferay-product-offering/images/05.png)

非常に複雑なカスタマイズが必要な場合には、OSGiモジュールをコア・プラットフォームにデプロイするJavaベースの開発アプローチを使用することもできますが、この場合、SaaS以外のデプロイメント・アプローチが必要になります。

## SaaSが可能かどうかの判断

Liferayは、 __ のすべての顧客が、SaaS展開で独自のビジネス問題を解決できるわけではないことを認識しています。

顧客が _、_ 、SaaSを快適に利用していると仮定した場合、以下のことが当てはまるのであれば、SaaSファーストのアプローチをとるべきである：

* ビジネス上の問題は、すぐに使えるLiferayの機能で解決できますか？
* カスタマイズの要件は、エクステンションの望ましいアプローチで解決できますか？
* カスタマイズの要件は、すでにロードマップにある、あるいは要望のある次期機能で解決できますか？

```{note}
この最後の点については、ソリューション・コンシェルジュ・デスクで確認してください。Liferayは、お客様の要件をサポートするために新しい機能を追加することに積極的に取り組んでおり、多くの場合、販売サイクル中に見込み顧客のために新しい機能を開発することができます。
```

## SaaSが不可能な場合の判断

Liferay Experience Cloud SaaSはデフォルトのオプションであるべきですが、お客様にとって最適なアプローチではない状況があることも認識しています：

* 顧客はSaaSに前向きではない
* データ主権について懸念がある - 例えば、Liferayがまだ顧客のローカルにデータセンターを持っていない場合など。
* データ・セキュリティやデータ管理に関する懸念は他にもある。

このような場合は、他の配備オプションを検討する必要があります：

* Liferayエクスペリエンスクラウドセルフマネージド（PaaS可の場合）
* Liferay DXPセルフホスト

これらのアプローチはどちらもマネージドサービスオプション（Liferayまたは弊社のパートナーによって提供され、顧客の懸念に対処しながらSaaSのようなエクスペリエンスを提供することができます。

これらの懸念はすべて、ソリューション・コンシェルジュ・デスクで厳しく検証する必要がある。

```{note}
これらのケースはすべて、製品が進化し、SaaSに対する抵抗が時間とともに減っていくにつれて、徐々に減っていくはずだ。
```

## 新規顧客および新規プロジェクトの営業決定フロー

Liferayは、新規顧客や既存顧客との新規プロジェクトで使用する営業意思決定フローを開発しました。 パートナーはこのフローを利用して、それぞれの顧客やプロジェクトに最適なアプローチを決定することができる。 これは、SaaS、PaaS、マネージドサービス付きPaaS、またはセルフホスト型に導く主要な決定ステップをカバーし、どの展開アプローチがオポチュニティに適しているかを示すガイダンスを提供します。
