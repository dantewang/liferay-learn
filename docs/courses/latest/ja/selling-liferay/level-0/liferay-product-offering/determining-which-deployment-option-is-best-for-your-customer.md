前述のように、すべてのデプロイオプションが可能ですが、弊社の販売アプローチはSaaS第一です。 とはいえ、実際に何を意味するのか、これは常に顧客にとって正しい選択肢となるのかという疑問もあるかと思うので、ここではその疑問についてご説明していきます。

## ビジネス上の問題を解決する

* お客様のビジネス課題をLiferay DXPで解決
* すべての機会はここから始まる
* ビジネス上の問題を先に考え、その後にデプロイ戦略を考える

Liferayソリューションの最終目的は、顧客のビジネス上の問題を解決することです。 機会が開始されたとき、その製品が最終的にどのようにデプロイされるかは重要ではありません。重要なのは、その問題がLiferay DXPで解決できるかということです。

その答えが 「イエス」であれば、そこから開始するのが最適です。 この後のモジュールでは、求める顧客像、弊社独自の価値提案、弊社が取り組む課題について取り上げます。

しかし現時点では、見込み顧客について知識を深めることで、最適なデプロイ方法を選択することができます。

## カスタマイズされたソリューションの構築

Liferay Experience Cloud SaaSをデプロイ方法として選択した場合でも、Liferayで実装されたソリューションはターンキーアプリケーションではありません。 Liferay DXPは、設定なしで使える広範な機能を提供します。 どのような場合でも、ソリューションの構築を開始するには、以下を設定する必要があります。

* ページの追加
* コンテンツの作成
* ユーザーとロールの定義
* など。

しかし、ほとんどの場合、ビジネス上の問題すべてがこうした機能だけで解決するわけではありません。 Liferay DXPの主な強みは、特定のニーズに合わせてプラットフォームを拡張・カスタマイズできることです。 プラットフォームを拡張するための推奨アプローチは、[クライアント拡張](https://learn.liferay.com/web/guest/w/dxp/building-applications/client-extensions)を作成することです。 クライアント拡張はLiferayのコードに縛られず、どのデプロイ方法でも使用できます。 クライアント拡張により、開発者は使い慣れた技術を用いて、コアプラットフォームへのアップグレードを容易に行うことができます。

![クライアント拡張は疎結合であり、最新のテクノロジーを使用して、容易なアップグレードをサポートします。](../liferay-product-offering/images/05.png)

非常に複雑なカスタマイズが必要な場合には、OSGiモジュールをコアプラットフォームにデプロイするJavaベースの開発アプローチを使用することもできますが、この場合、SaaS以外のデプロイ方法が必要になります。

## SaaSが可能かどうかを判断する

Liferayは、 **すべての** 顧客が、SaaSデプロイで独自のビジネス問題を解決できるわけではないことを認識しています。

顧客がSaaSに **慣れている** と仮定し、以下のことが当てはまるのであれば、SaaSファーストのアプローチをとるべきです。

* ビジネス上の問題は、Liferayの標準機能で解決できるか。
* カスタマイズの要件は、拡張機能への推奨アプローチで解決できるか。
* カスタマイズの要件は、すでにロードマップにある機能、または要求された今後の機能で解決できるか。

```{note}
この最後の点については、ソリューションコンシェルジュデスクで確認してください。Liferayは、お客様の要件をサポートするために新機能を追加することに積極的に取り組んでおり、多くの場合、販売サイクル中に見込み顧客向けに新機能を開発することができます。
```

## SaaSが不可能な場合の判断

Liferay Experience Cloud SaaSがデフォルトのオプションではありますが、顧客にとって最適なアプローチではない以下のような状況があることもLiferayは認識しています。

* 顧客はSaaSに前向きではない
* データ主権について懸念がある。例えば、Liferayがまだ顧客の地域にデータセンターを持っていない場合など。
* データセキュリティやデータ管理に関する懸念が他にもある。

このような場合は、他のデプロイオプションを検討する必要があります。

* Liferay Experience Cloud Self-Managed（PaaS可の場合）
* Liferay DXP Self-Hosted

これらのアプローチはどちらも、LiferayまたはLiferayパートナーが提供するマネージドサービスオプションを使用して、顧客の懸念に対応しつつSaaSのようなエクスペリエンスを提供できます。

これらの懸念はすべて、ソリューションコンシェルジュデスクで厳しく認定・検証する必要があります。

```{note}
これらのケースはすべて、製品が進化し、SaaSに対する抵抗が時間とともに減少するにつれて、徐々に減少するはずです。
```

## 新規顧客および新規プロジェクトの販売決定フロー

Liferayは、新規顧客や既存顧客との新規プロジェクトで使用する販売意思決定フローを開発しました。 パートナーはこのフローを利用して、それぞれの顧客やプロジェクトに最適なアプローチを決定できます。 これにより、SaaS、PaaS、マネージドサービスを備えたPaaS、またはセルフホスト型のいずれかを選択する主要な決定ステップに対応し、どのデプロイ方法が機会に適しているかを示すガイダンスを提供します。また異常値についてはソリューションコンシェルジュデスクに誘導します。
