# Liferay 객체 생성

**한눈에**

* 높은 수준의 Liferay 데모에는 기본 기능을 보여주는 사전 구축된 개체의 데모가 포함되어야 합니다.
* 이 섹션에는 두 가지 목적을 위해 처음부터 객체를 생성하는 방법에 대한 세부 정보가 포함되어 있습니다.
  * 객체에 대한 보다 자세한 학습 기회를 제공합니다.
  * Prospect와 함께 보다 자세한 기술 데모를 위한 프레임워크를 제공합니다.

## Liferay 개체에 대해 알아보기

Liferay 객체는 데이터의 컨테이너로 생각할 수 있습니다. 다양한 소스에서 데이터를 저장하고 검색하는 데 사용할 수 있습니다. 개체는 고도로 구성 가능하며 사용자는 애플리케이션의 특정 요구 사항을 충족하기 위해 사용자 정의 필드, 관계 및 메서드를 정의할 수 있습니다.

## 객체 생성

![새 객체 생성을 시작하려면 레이블, 복수형 레이블 및 객체 이름을 정의하십시오.](./creating-objects/images/01.png)

객체를 생성하는 첫 번째 단계는 도면 객체입니다. 개체 초안은 개체 응용 프로그램을 만드는 데 필요한 데이터를 저장하는 응용 프로그램 템플릿입니다. 개체 초안을 작성하려면 다음 단계를 따르십시오.

1. 전역 메뉴를 열고 **Control Panel** 탭으로 이동한 다음 **Objects** 을 클릭합니다.
2. **추가** 버튼을 클릭하세요.
3. 개체 초안에 대한 세부 정보를 제공합니다.
    1. **라벨** : 개체 관리 및 지원되는 응용 프로그램 컨텍스트(예: 작업 흐름, 표시 페이지 템플릿, 양식)에서 개체를 식별합니다.
    2. **Plural Label** : UI에서 개체 응용 프로그램의 표시 이름을 결정합니다.
    3. **개체 이름** : 개체의 정의 이름을 결정하며 개체가 게시된 후에는 변경할 수 없습니다.
4. **저장** 클릭하세요.

저장하고 나면 구성이나 데이터 요소 없이 빈 개체 초안이 생성됩니다. 개체 초안은 시스템 개체와 함께 개체 관리 UI에서 볼 수 있습니다. 생성 프로세스를 완료하려면 개체 초안을 [편집](https://learn.liferay.com/w/dxp/building-applications/objects/creating-and-managing-objects/creating-objects?p_l_back_url=%2Fsearch%3Fq%3DObjects&highlight=Objects#editing-object-drafts) 한 다음 Liferay 인스턴스에 [게시](https://learn.liferay.com/w/dxp/building-applications/objects/creating-and-managing-objects/creating-objects?p_l_back_url=%2Fsearch%3Fq%3DObjects&highlight=Objects#publishing-object-drafts) 해야 합니다.

## 객체 편집 및 게시

![개체 편집에는 세부 정보, 필드, 관계, 레이아웃, 작업, 보기, 유효성 검사, 상태 관리자에 대한 탭이 제공됩니다.](./creating-objects/images/02.png)

개체를 게시하려면 먼저 요소를 추가하고 해당 설정을 구성해야 합니다. 여기에는 필드 추가, 유효성 검사 정의, 데이터 범위 설정 등이 포함됩니다.

개체 초안을 편집하려면 다음 단계를 따르십시오.

1. 개체 관리 페이지에서 새로 생성된 개체 초안을 클릭하여 편집을 시작합니다. 개체 기능 및 구성 옵션은 세부 정보, 필드, 관계, 레이아웃, 작업, 보기, 유효성 검사 및 상태 관리자 탭으로 구성됩니다.
1. 사용자 정의 필드를 추가하려면 **필드** 탭을 클릭하세요. 각 필드는 특정 유형의 값을 저장하기 위한 데이터베이스 열을 나타냅니다. [객체에 필드 추가](https://learn.liferay.com/w/dxp/building-applications/objects/creating-and-managing-objects/fields/adding-fields-to-objects) 참조하십시오. 기본적으로 모든 개체에는 작성자, 생성 날짜, 외부 참조 코드, ID, 수정 날짜 및 상태와 같은 시스템 필드가 포함됩니다.
1. '회사'의 개체 범위를 선택합니다. 객체는 객체의 데이터가 저장되는 방식을 결정합니다. 
    * **회사**(기본값): 회사로 범위가 지정되면 개체의 데이터가 Liferay 인스턴스별로 저장됩니다.
    * **사이트** : 사이트별로 범위가 지정되면 개체의 데이터가 사이트별로 저장됩니다.
1. 개체의 페이지 위젯을 표시합니다. 활성화된 동안 사이트 페이지에 개체의 위젯을 추가할 수 있습니다. [사이트에 개체 위젯 배포](https://learn.liferay.com/w/dxp/building-applications/objects/deploying-object-widgets-to-sites) 참조하십시오.
1. 저장을 클릭합니다.

구성 및 편집이 완료되면 객체를 [할 수 있습니다](https://learn.liferay.com/w/dxp/building-applications/objects/creating-and-managing-objects/creating-objects) .

Liferay는 Liferay UI를 통해 개발자가 즉시 사용할 수 있는 객체에 대한 상당한 양의 요소를 생성하기 때문에 이 단계는 실제로 많은 마법이 일어나는 곳입니다. 게시 프로세스에는 다음 작업이 포함됩니다.

* 초안의 데이터 정의가 포함된 객체에 대한 데이터베이스 테이블이 생성됩니다.
* CRUD 작업을 위해 새로운 Headless API가 자동으로 생성됩니다.
* 개체 항목을 표시하기 위해 컬렉션 공급자가 생성됩니다.
* 개체에 대한 페이지 위젯이 생성됩니다.
* 개체는 Info 프레임워크와 통합되므로 개체를 디스플레이 페이지 템플릿의 콘텐츠 유형으로 선택할 수 있습니다.
* 객체는 권한 프레임워크와 통합되므로 새 애플리케이션과 해당 리소스에 대한 권한을 관리할 수 있습니다.
* 개체는 워크플로 프레임워크와 통합되므로 개체에 대한 사용자 정의 워크플로를 구성할 수 있습니다.
* 개체는 Forms와 통합되므로 개체를 데이터 저장 옵션으로 선택할 수 있습니다.

개체 기능이 없으면 이러한 모든 단계를 코드 작성으로 수행해야 하기 때문에 이는 매우 중요합니다. 생성된 요소를 통해 사용자는 객체를 표시하고, 객체에 항목을 추가하고, 컬렉션 표시에 해당 항목을 나열하고, 디스플레이 페이지에 개별 객체 항목을 표시하는 등의 작업을 수행할 수 있습니다.

## 객체용 UI 만들기

![개체가 게시되면 위젯과 동일한 방식으로 모든 페이지에 배포될 수 있습니다.](./creating-objects/images/03.png)

비즈니스 애플리케이션에 대한 데이터 모델이 정의되면 다음 단계는 Liferay의 위젯 프레임워크를 사용하여 사용자 인터페이스를 만드는 것입니다. 여기에는 애플리케이션의 데이터를 표시하고 관리하는 데 사용되는 위젯과 구현에 필요한 사용자 정의 사용자 인터페이스 구성 요소를 정의하는 작업이 포함됩니다.

**페이지에 위젯 추가**

* 게시된 개체에 대해 위젯 표시가 활성화되면 해당 위젯이 위젯 메뉴의 개체 아래에 나타납니다.
* 위젯은 다른 페이지 위젯과 마찬가지로 사이트 페이지에 추가할 수 있습니다.
* 추가 기능 및 구성 옵션에 액세스하려면 위젯의 옵션 버튼을 클릭하세요.

Liferay 객체는 Forms 애플리케이션과 통합되므로 사용자 입력을 수신하고 객체 항목을 생성하도록 양식을 설계할 수 있습니다. 이렇게 하려면 먼저 새 양식을 추가하고 개체의 저장소 유형을 사용하도록 구성합니다. 그런 다음 원하는 개체를 선택하고 양식을 디자인한 다음 해당 필드를 선택한 개체의 필드에 매핑합니다. \

자세한 내용은 [객체와 함께 양식 사용](https://learn.liferay.com/w/dxp/building-applications/objects/using-forms-with-objects#using-forms-with-objects) 참조하십시오.

## 헤드리스 객체

![객체용 헤드리스 API는 자동으로 생성되며 API 탐색기를 사용하여 찾아볼 수 있습니다.](./creating-objects/images/04.png)

```{note}
Headless API에 대한 토론은 청중이 적절하고 주로 기술적 성격을 지닌 경우에만 데모에 포함되는 것이 가장 좋습니다. 객체의 주요 이점은 개발할 필요 없이 비즈니스 애플리케이션을 구축할 수 있다는 것입니다. 주로 비즈니스 청중과 헤드리스 API에 대해 논의하면 혼란을 야기하고 이 메시지를 희석시킬 수 있습니다.
```

기술적인 청중을 위해 논의할 추가 주제는 다음과 같습니다.

* 헤드리스 API는 사용자 정의 개체에 대해 자동으로 생성됩니다.
* Liferay 인스턴스의 /o/api URL을 사용하여 새 API를 탐색할 수 있습니다.
* Headless API의 일반적인 사용 사례는 객체에 대한 사용자 정의 JavaScript 클라이언트 확장 UI를 개발하는 것입니다.
* 헤드리스 API를 사용하여 객체와 외부 시스템 통합을 활성화할 수도 있습니다.

일반적으로 Liferay의 헤드리스 API에 대한 자세한 내용은 [REST 서비스 사용](https://learn.liferay.com/w/dxp/headless-delivery/consuming-apis/consuming-rest-services#consuming-rest-services) 참조하세요.

## 고급 기능

![State Manager는 Liferay 객체에 대한 워크플로를 정의하는 방법을 제공합니다.](./creating-objects/images/05.png)

위에 설명된 추가 학습 및 잠재적인 심층 데모 단계에도 불구하고 객체로 달성할 수 있는 것의 표면적인 부분만 긁힌 것입니다. 데모용 개체를 생성할 수 있는 기회를 제공하기에 충분한 정보가 포함되었습니다. 

사전 구축된 데모에 잠재 고객이 관련될 수 있는 내용이 추가될 때마다 데모는 더욱 매력적이며 더 나은 성공 가능성을 만들어냅니다!

그러나 Objects에는 정교한 비즈니스 애플리케이션에 사용할 수 있는 더 많은 기능이 포함되어 있습니다.

* 다른 사용자 정의 개체, 시스템 개체(예: 사용자, 계정 등) 및 개체 자체와의 일대다 및 다대다 관계를 추가할 수 있습니다. 예를 들어, 동일한 테이블에 "관련" 개체 항목을 표시하기 위해 관계를 생성할 수 있습니다.
* 레이아웃은 개체 항목이 표시되는 방식을 제어하기 위해 레이아웃을 쉽게 정의할 수 있습니다.
* 작업을 통해 개체를 사용자에게 할당하고 개체에 대한 알림을 보낼 수 있습니다.
* 올바른 정보가 캡처되었는지 확인하기 위해 유효성 검사를 정의할 수 있습니다.
* State Manager를 사용하면 전체 비즈니스 프로세스에서 개체가 진행되는 방식을 제어하는 워크플로를 구축할 수 있습니다.

그리고 그것이 전부는 아닙니다. 사용자가 개체 항목을 생성할 수 있는 훌륭한 사용자 인터페이스는 양식 통합, 양식 컨테이너 및 양식 조각과 같은 것을 사용하여 구축할 수 있습니다.

그리고 앞서 언급했듯이 자동 생성된 헤드리스 API를 사용하여 완전히 사용자 정의 UI를 클라이언트 확장으로 구축할 수 있습니다.

## 개요

Liferay 데모의 효과를 극대화하고 플랫폼에서 제공되는 로우 코드/노코드 기능을 선보일 기회를 제공하려면 항상 Liferay 개체 데모를 포함하십시오.

Liferay 개체는 다음을 제공합니다.

* 사용자 지정 애플리케이션을 만들기 위한 빠르고 쉬운 노코드 접근 방식입니다.
* 다음을 포함한 포괄적인 기능: 
    * 데이터 모델 정의
  * 사용자 인터페이스 생성
  * 자동 생성된 API
  * 클라이언트 확장을 통해 사용
  * 외부 시스템과 통합
* 데이터 관리에 대한 표준, 모듈식 접근 방식 - 더 나은 애플리케이션을 더 빠르게 구축합니다.

이 학습 경로는 사전 구축된 데모를 사용하는 데 적합하지만 개체는 해당 데모에 추가 기능을 추가하는 빠르고 쉬운(코드 없음!) 방법을 제공합니다. 완전한 애플리케이션이 아니라 잠재 고객의 특정 요구 사항에 맞게 플랫폼을 얼마나 빨리 확장할 수 있는지 보여주기 위해 잠재 고객의 요구 사항에 맞는 것을 만드는 것을 목표로 하십시오. 

Object로 비즈니스 애플리케이션 구축을 시작하는 방법을 알아보려면 Objects 문서 참조와 함께 여기에 있는 콘텐츠를 사용하세요.
