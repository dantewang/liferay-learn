# 커머스 역할 참조

커머스에는 계정 관리자, 계정 회원, 구매자, 주문 관리자의 네 가지 역할 [](#out-of-the-box-roles) 이 포함됩니다. 이러한 역할을 수정하거나 사용 사례에 더 적합하도록 추가 계정 역할을 만들 수 있습니다. 일부 [일반적인 사용자 지정 역할(](#custom-roles) )에는 카탈로그 관리자, 재고 관리자, 배송 관리자, 인스턴스 주문 관리자, 할인 관리자 등이 있습니다.

```{note}
영업 에이전트 역할은 Liferay 7.3.x 및 이전 커머스 버전에서 바로 사용할 수 있습니다. Liferay DXP 7.4 이상에서는 Minium 및 Speedwell 가속기를 통해서만 사용할 수 있습니다. 
```

## 보기 권한

커머스 역할과 관련된 권한을 보고 관리하려면 다음 단계를 따르세요:

1. *글로벌 메뉴* (![Global Menu](../../images/icon-applications-menu.png))를 열고 *제어판* &rarr; *사용자* &rarr; *역할*으로 이동합니다.

   ![제어판에서 역할을 클릭합니다.](./commerce-roles-reference/images/01.png)

1. 계정 역할 탭을 클릭하고 역할을 선택합니다:

   * 계정 관리자
   * 계정 회원
   * 구매자
   * 주문 관리자

   ```{note}
   Liferay 7.3 및 이전 버전에서 계정 관리자는 사이트 역할이며 계정 구성원 역할은 기본 역할이 아닙니다. 
   ```

1. *권한 정의* 탭을 클릭합니다.

   여기에서 역할에 할당된 모든 애플리케이션 및 리소스 권한의 요약을 볼 수 있습니다.

   권한을 제거하거나 추가할 수도 있습니다.

   ![권한 정의 탭을 클릭하여 역할 권한을 보고 관리합니다.](./commerce-roles-reference/images/02.png)

## 즉시 사용 가능한 역할

다음 역할은 DXP 인스턴스에서 바로 사용할 수 있습니다. 이러한 역할은 기본 DXP 역할에 포함되어 있으며 필요한 경우 수정할 수 있습니다. 자세한 내용은 [역할 권한 정의하기](https://learn.liferay.com/dxp/latest/en/users-and-permissions/roles-and-permissions/defining-role-permissions.html) 를 참조하세요.

### 계정 관리자

> 계정 역할

계정 관리자는 계정 슈퍼사용자입니다. [구매자 주문 승인](../../order-management/order-workflows/enabling-or-disabling-order-workflows.md#enabling-workflow) 워크플로우가 활성화된 경우 주문을 승인하거나 거부할 수 있으며 계정의 주문 관리자 및 구매자를 감독할 수 있습니다. 계정을 수정하고, 사용자를 초대하거나 추가하고, 다른 계정 구성원에게 역할을 할당할 수 있습니다. 계정 관리자는 계정을 만들거나 삭제할 수 없습니다. 이 권한은 인스턴스 관리자에게만 제공됩니다.

계정 권한은 계정 관리자가 B2B 설정에서 계정을 수정할 수 있는 방법을 정의하며, 오픈 카트 권한은 스토어 유형에 관계없이 관리자가 주문을 수정할 수 있는 방법을 정의합니다.

### 계정 공급업체

{bdg-secondary}`사용 가능한 Liferay DXP 7.4 U84+/GA84+`

> 계정 역할

이 역할은 공급업체 계정과 연결하기 위해 Liferay에 도입된 새로운 계정 역할입니다. 역할에 연결된 권한은 없지만 할당 시 사용자에게 공급업체라는 일반 역할이 할당됩니다. 이 역할은 공급업체 계정과 연결된 올바른 채널 및 카탈로그에 대한 일반 역할의 액세스 범위를 지정합니다. 일반 역할을 직접 사용하면 공급업체에 모든 카탈로그 및 채널에 대한 액세스 권한이 부여됩니다. 아래 [공급업체](#supplier) 역할을 참조하세요.

### 구매자

> 계정 역할

구매자는 자신이 처리하는 계정을 대신하여 구매할 수 있습니다. 구매자 주문 승인 워크플로우가 있는 경우, 구매자는 주문 승인 여부를 결정하는 주문 관리자에게 주문을 제출해야 합니다. 구매자는 같은 계정의 다른 구매자가 보낸 주문을 보고 결제할 수도 있습니다.

계정 권한은 구매자가 주소를 보고 관리하는 기능을 정의하고, 카트 열기 권한은 구매자가 주문을 보고, 생성하고, 결제하고, 기타 주문 속성을 관리하는 기능을 정의합니다.

| 계정 > 계정 입력: 주소 관리               | 계정 주소 관리 기능             |

### 카탈로그 관리자

> 일반 역할

카탈로그 관리자 역할은 특정 카탈로그 내에서 제품에 액세스, 생성, 편집 및 관리할 수 있는 권한을 부여합니다. 이 역할은 여러 공급업체 또는 회사 팀에서 서로 다른 제품을 별도의 카탈로그에서 관리하는 다중 카탈로그 시나리오에서 유용합니다. 카탈로그 관리자는 재고를 관리할 수 없습니다. 이 책임에 대해서는 [인벤토리 관리자](#inventory-manager)을 참조하세요.

| 권한                 | 묘사                             |
|:------------------ |:------------------------------ |
| 카탈로그: 제어판에서 액세스    | 글로벌 메뉴에서 카탈로그 애플리케이션에 액세스하는 기능 |
| 카탈로그: 보기           | 카탈로그 애플리케이션을 볼 수 있는 기능         |
| 통화 > 커머스 통화: 통화 관리 | 통화 관리 기능                       |
| 포털: 제어판 메뉴 보기      | 글로벌 메뉴에 액세스하는 기능               |
| 제품: 제어판에서 액세스      | 글로벌 메뉴에서 제품 애플리케이션에 액세스하는 기능   |
| 제품 제품: 보기          | 제품 애플리케이션을 볼 수 있는 기능           |

위의 권한 외에도 역할이 관리할 개별 카탈로그에 대한 보기 및 업데이트 권한도 할당해야 합니다.

이렇게 하려면 *글로벌 메뉴* (![Global Menu](../../images/icon-applications-menu.png))를 열고 *커머스* 탭을 클릭한 다음 *제품 관리* &rarr; *카탈로그*로 이동합니다. 그런 다음 원하는 카탈로그에 대한 *작업* 버튼(![Actions Button](../../images/icon-actions.png))을 클릭하고 *권한*을 선택합니다. 확인란을 사용하여 *보기* 및 *업데이트* 권한 역할을 할당합니다.

```{note}
카탈로그 관리자가 카탈로그의 제품만 관리하고 카탈로그 자체는 수정하지 않게 하려면 `카탈로그`를 제거하세요: 제어판에서 액세스` 및 `카탈로그: 보기` 권한을 제거합니다.
```

### 배송 관리자

> 일반 역할

배송 관리자 역할을 통해 사용자는 배송을 처리할 수 있습니다. 여기에는 배송 세부 정보, 배송 날짜 추가, 배송에 품목 추가 및 배송 상태 업데이트 기능이 포함됩니다. 커머스 3.0+ 및 라이프레이 7.3+에서는 사용자가 주문 애플리케이션을 통해 배송을 생성합니다. 배송 관리자가 배송을 작성하고 처리하기만 하도록 하려면 다음 권한으로 충분합니다. 그렇지 않으면 [주문 관리자](#order-manager-regular-role) 권한을 추가로 추가할 수 있습니다.

| 권한                       | 묘사                              |
|:------------------------ |:------------------------------- |
| 장바구니 열기 > 상거래 주문: 주문 보기  | 상태와 관계없이 주문을 볼 수 있는 기능          |
| 계정 그룹 > 커머스 계정: 모든 계정 관리 | 계정 그룹 애플리케이션에 액세스하는 데 필요합니다.    |
| 창고 > 커머스 인벤토리: 재고 관리     | 창고에 연결된 인벤토리 엔티티에 액세스하고 수정하는 기능 |
| 배송 관리                    | 배송 개체에 액세스하고 수정하는 기능            |
| 포털: 제어판 메뉴 보기            | 글로벌 메뉴에 액세스하는 기능                |
| 배송: 제어판에서 액세스            | 글로벌 메뉴에서 배송 애플리케이션에 액세스하는 기능    |
| 배송: 보기                   | 배송 애플리케이션을 볼 수 있는 기능            |
| 배송 > 커머스 창고: View        | 창고 엔티티 보기 기능                    |

## Liferay 7.3 및 이전 버전

### 영업 에이전트

> 일반 역할

영업 에이전트는 Liferay의 일반 역할이며, 양수인은 관리 권한을 부여하지 않고도 역할에 할당된 모든 계정을 관리할 수 있습니다. *정규 역할* 탭에서 찾을 수 있습니다.

![권한 정의 탭에서 기본 영업 상담원 권한을 확인합니다.](./commerce-roles-reference/images/06.png)

| 권한       | 묘사                               |
|:-------- |:-------------------------------- |
| 조직 관리    | 조직 추가 또는 제거 기능                   |
| 가용 계정 관리 | 영업 담당자가 회원으로 가입한 계정을 관리할 수 있는 기능 |
