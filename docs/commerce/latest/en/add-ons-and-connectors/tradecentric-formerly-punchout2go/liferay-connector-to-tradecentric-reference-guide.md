# Liferay Commerce Connector to TradeCentric Reference Guide

{bdg-primary}`Subscription`

There are several configuration terms in TradeCentric (formerly PunchOut2Go).

| Term                   | Description                                                                                                                |
| :--------------------- | :------------------------------------------------------------------------------------------------------------------------- |
| Punch out session      | The set of actions between when a user successfully logs into Liferay and transfers the cart back to the punch out system. |
| HttpSession            | The existing session storage built into Liferay that stores data related to the punch out session.                         |
| Punch out start URL    | The start URL for the punch out session that goes to the catalog page for the specified Liferay Commerce channel.          |
| Punch out return URL   | The return URL that takes the user back to their procurement system.                                                       |
| Punch out cart         | The cart containing the buyer's items selected for purchase.                                                               |
| Purchase order         | The final approved draft of the order submitted by the buyer.                                                              |
| Punch out access token | A single-use token used to auto-login the punch out user.                                                                  |
| Punch out vendor       | The third-party vendor that connects punch out catalogs with the buyer's procurement systems.                              |
