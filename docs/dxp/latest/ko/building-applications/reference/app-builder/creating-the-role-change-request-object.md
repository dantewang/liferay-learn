# 역할 변경 요청 객체 생성

> **구독자**

소프트웨어 엔지니어링 부서 내에서 직원의 역할 변경 요청을 수집하는 인사 애플리케이션인 역할 변경 요청 앱을 생각해 보세요. 이 앱은 직원이 요청 양식을 제출하는 것으로 시작됩니다. 그런 다음 처리는 현재 관리자, 수신 관리자, 마지막으로 인사 부서로 진행됩니다. 각 단계마다 다른 형태가 제시되고, 객체는 데이터를 축적한다. 프로세스가 끝나면 전체 데이터 개체가 완성됩니다.

![The Role Change Request object flows through several steps in this Workflow Powered App.](./creating-the-role-change-request-object/images/01.png)

여기서는 지원 개체, 양식 보기 및 테이블 보기를 만듭니다. Workflow Powered App 자체를 생성하려면 [역할 변경 요청 앱 생성](./creating-the-role-change-request-app.md)을 참조하세요.

App Builder 개체, 양식 보기 및 테이블 보기 생성에 대한 자세한 지침은 [표준 응용 프로그램 만들기](./creating-a-standard-application.md)를 참조하세요. 동일한 개체가 표준 앱 또는 워크플로 기반 앱을 지원할 수 있습니다. 차이점은 App Builder 프레임워크의 요소에서 앱을 생성하는 방법에 있습니다. 응용 프로그램 메뉴(![응용 프로그램 메뉴](../../../images/icon-applications-menu.png))의 개체 항목에서 개체, 양식 보기 및 테이블 보기를 생성할 수 있습니다. &rarr; 응용 프로그램 &rarr; 앱 빌더.

## **객체:** **Role Change Request** 라는 객체를 생성합니다.

## **양식:** **Employee Request** 양식을 만듭니다.

| 급식 유형   | 필드 라벨    | 옵션                                                                     | 요구하는 | 기타 속성 |
| :------ | :------- | :--------------------------------------------------------------------- | :--- | :---- |
| 원본      | 이름       | 해당 없음(해당 사항 없음)                                                        | ✔    | 해당 없음 |
| 원본      | 현 관리자 이름 | 해당 없음                                                                  | ✔    | 해당 없음 |
| 원본      | 현재 역할    | 해당 없음                                                                  | ✔    | 해당 없음 |
| 목록에서 선택 | 요청된 역할   | 지원 엔지니어 <br /> 백엔드 개발자 <br /> 프론트엔드 개발자 <br /> 테크니컬 라이터 <br /> IT 엔지니어 | ✔    | 해당 없음 |

| 급식 유형 | 필드 라벨                                    | 옵션           | 요구하는 | 기타 속성 |
| :---- | :--------------------------------------- | :----------- | :--- | :---- |
| 단일 선택 | 현 관리자: 이전이 승인되었나요?                       | 예 <br /> 아니요 | ✔    | 해당 없음 |
| 단일 선택 | 직원이 요청한 역할에 필요한 기술과 경험을 보여주고 있습니까?       | 예 <br /> 아니요 | ✔    | 해당 없음 |
| 원본    | 성공적인 이전을 가능하게 하는 직원의 기술과 특성을 자세히 설명하세요.  | 해당 없음        | ✘    | 여러 줄  |
| 원본    | 성공적인 이동을 방해할 수 있는 직원의 기술과 특성을 자세히 설명하세요. | 해당 없음        | ✘    | 여러 줄  |
| 날짜    | 언제 이전이 가능합니까?                            | 해당 없음        | ✘    | 해당 없음 |
