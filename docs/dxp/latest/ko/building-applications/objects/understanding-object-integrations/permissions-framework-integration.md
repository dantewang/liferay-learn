# 권한 프레임워크 통합

> 사용 가능: Liferay DXP/Portal 7.4+

개체가 생성되면 Liferay의 권한 프레임워크와 자동으로 통합됩니다. 즉, [애플리케이션](#application-permissions) 및 [리소스](#resource-permissions) 권한을 할당하여 개별 개체 및 해당 항목에 대한 사용자 액세스를 관리할 수 있습니다. 역할 기반 액세스 제어를 사용하면 적절한 사용자만 애플리케이션의 데이터를 보고 사용할 수 있도록 할 수 있습니다.

역할 권한을 정의할 때 활성 개체는 해당 범위 및 패널 범주 키에 따라 나타납니다.

## 애플리케이션 권한

애플리케이션 권한은 일반적인 애플리케이션 관련 작업을 수행할 수 있는 권한을 부여하며 [리소스 관련 권한](#resource-permissions)을 포함하지 않습니다.

사용자 정의 개체에는 정의된 범위에 따라 다음과 같은 표준 애플리케이션 권한이 포함됩니다.

| 허가                                 | 묘사                           |
|:---------------------------------- |:---------------------------- |
| 제어판에서 액세스 (*회사 범위만*)               | 전역 메뉴에서 개체에 액세스              |
| 사이트 및 자산 라이브러리 관리에서 액세스(*사이트 범위만*) | 사이트 메뉴 또는 자산 라이브러리에서 개체에 액세스 |
| 설정                                 | 개체의 구성 옵션 보기 및 설정            |
| 권한                                 | 개체의 권한 보기 및 수정               |
| 환경 설정                              | 개체의 기본 설정 보기 및 설정            |
| 보기                                 | 개체의 응용 프로그램 페이지 보기           |

```{note}
*사이트*로 범위가 지정된 개체의 경우 응용 프로그램 또는 리소스 권한이 모든 사이트에 부여되는지 아니면 특정 사이트에만 부여되는지 결정할 수도 있습니다.
```

## 리소스 권한

리소스 권한은 개체 애플리케이션의 리소스를 보고 작업할 수 있는 액세스 권한을 부여합니다. 이러한 권한 중 일부는 [개체 항목 생성과 관련](#creating-entries)하고 다른 권한은 [기존 항목에 대한 작업 수행](#acting-on-existing-entries)(예: 편집, 삭제)을 위한 것입니다.
각 개체에는 다음과 같은 리소스 권한도 있습니다.

### 리소스 관련 작업

| 허가       | 묘사                    |
|:-------- |:--------------------- |
| 객체 항목 추가 | 개체 항목 만들기             |
| 권한       | 개체 항목과 관련된 권한 보기 및 관리 |

### 데이터베이스 엔티티에 대한 작업

| 허가 | 묘사                      |
|:-- |:----------------------- |
| 삭제 | 개체 항목 삭제                |
| 권한 | 개별 개체 항목에 대한 권한 보기 및 수정 |
| 수정 | 개체 항목 업데이트              |
| 보기 | 개체 항목 보기                |

## 개별 객체 엔티티에 대한 권한 관리

> Liferay DXP 7.4 U10+ 및 Liferay Portal 7.4 GA14+ 사용 가능

사용자 지정 개체를 사용하면 개별 데이터베이스 엔터티에 대한 권한을 관리하여 개체 데이터에 대한 액세스를 제어할 수 있습니다.

다음과 같이하세요:

1. 원하는 사용자 지정 개체로 이동합니다.

1. 원하는 엔터티에 대해 *작업* 버튼(![Actions Button](../../../images/icon-actions.png))을 클릭하고 *권한*를 선택합니다.
   
   ![원하는 엔터티에 대한 작업 버튼을 클릭하고 권한을 선택합니다.](./permissions-framework-integration/images/01.png)

1. 확인란을 사용하여 원하는 역할에 [데이터베이스 엔터티](#actions-on-database-entities) 권한을 부여합니다. 

   ```{note}
   역할 관리 수준에서 정의된 권한은 엔터티 수준에서 정의된 권한을 재정의합니다.
   ```

![확인란을 사용하여 원하는 역할에 권한을 할당합니다.](./permissions-framework-integration/images/02.png)

1. *저장*을 클릭합니다.
