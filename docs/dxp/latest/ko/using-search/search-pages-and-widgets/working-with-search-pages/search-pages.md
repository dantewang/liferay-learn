# 수색 페이지

[`localhost:8080/web/guest/search`](http://localhost:8080/web/guest/search) 에서 기본 검색 페이지에 액세스합니다. 일부 콘텐츠가 포함된 Liferay DXP를 실행 중인 경우 해당 사이트로 이동하여 검색을 실행할 수 있습니다. 기본적으로 이 페이지는 기본 사이트의 숨겨진 페이지입니다. 사용자가 *Search Bar* 위젯에 검색어를 입력하면 이 숨겨진 페이지로 라우팅됩니다. 검색 페이지는 사용자가 검색 결과를 찾아보는 곳이기도 합니다.

![기본 검색 페이지는 숨겨져 있으며 직접 URL을 통해 또는 검색 창 위젯을 사용하여 액세스할 수 있습니다.](./search-pages/images/05.png)

사용자는 사이트 사용자에게 정확하고 맞춤화된 검색 경험을 제공하기 위해 새 검색 페이지를 만들고, 다른 검색 위젯을 추가하고, 사용 가능한 다양한 위젯 구성을 조정하여 검색 경험을 사용자 정의하도록 선택할 수 있습니다. [기본](#default-search-pages) 검색 페이지를 사용하면 자신만의 사용자 정의 검색 페이지를 만드는 좋은 출발점이 될 수 있습니다.

```{note}
새로 생성된 사이트는 기본적으로 생성된 검색 페이지를 가져오지 않습니다. 계정을 만들어야 합니다. 기존 검색 페이지 템플릿을 사용하거나 새 검색 페이지를 직접 만들 수 있습니다.
```

![비어 있는 새 사이트에는 검색 페이지를 만들고 구성해야 합니다.](./search-pages/images/01.png)

## 기본 검색 페이지

즉시 사용 가능한(기본 사이트, 테마 및 검색 구성) 검색 환경에는 최종 사용자를 위한 두 가지 구성 요소가 있습니다.

1. 각 페이지의 헤더에 포함된 검색 표시줄.
1. 검색 요청이 라우팅되고 결과가 표시되는 기본 검색 페이지입니다.

뒤에서 검색 막대 위젯은 친근한 URL `/search`이 있는 숨겨진 검색 페이지를 가리킵니다.

![검색 표시줄에 대해 대상 페이지를 구성해야 합니다.](./search-pages/images/02.png)

검색 쿼리를 입력하면 검색 결과 위젯에 결과가 표시되는 기본 검색 페이지로 리디렉션됩니다.

![기본 검색 페이지는 전체 검색 환경을 제공합니다.](./search-pages/images/03.png)

기본 검색 페이지에는 검색 표시줄, 검색 결과, 검색 제안, 검색 옵션 및 여러 검색 패싯 위젯이 포함되어 있습니다. 위의 예에서 Type 패싯, User 팩트 및 Last Modified 패싯은 왼쪽 사이드바에서 볼 수 있습니다. 사용 가능한 다양한 [검색 패싯](../search-facets.md) 및 사용자 정의 방법에 대해 자세히 읽어보십시오.

기본 검색 페이지는 검색 페이지 템플릿을 기반으로 하지만 기본적으로 템플릿에서 변경 사항을 상속하지 않습니다. 즉, 템플릿의 상속 구성을 변경하지 않고 검색 페이지를 직접 사용자 지정할 수 있습니다.

![기본적으로 검색 페이지는 페이지 템플릿에서 변경 사항을 상속하지 않습니다.](./search-pages/images/04.png)

기본 페이지에 몇 가지만 변경해야 하는 경우 페이지를 버리지 말고 수동으로 만드십시오. 페이지에서 위젯 추가, 구성 및 제거를 포함하여 필요한 구성을 변경하기만 하면 됩니다. 반면에 기본 검색 페이지에서 완전히 중단하려면 처음부터 시작하는 것이 좋습니다.

## 검색 페이지 만들기

페이지 템플릿으로 검색 페이지를 만들거나 단일 사이트에 대해 수동으로 구성된 검색 페이지를 사용할 수 있습니다. 검색 페이지 템플릿에 대해 알아보려면 [검색 페이지 템플릿 사용](./using-a-search-page-template.md) 을 참조하십시오. 사이트별로 개별적으로 검색 페이지를 구성하는 방법에 대해 알아보려면 [검색 페이지 만들기](./creating-a-search-page.md) 을 참조하십시오.

접근 방식을 선택하고 검색 페이지를 시작하고 실행하기 위해 여기를 읽은 후 다양한 검색 위젯에 대한 자세한 기사와 구성 문서를 읽어 구성 옵션의 전체 제품군을 이해하십시오.
