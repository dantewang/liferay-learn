# 별칭 리디렉션 사용

별칭 리디렉션은 절대 소스 및 대상 URL을 명시적으로 정의합니다. 예측 가능성 때문에 별칭 리디렉션은 대부분의 사용 사례에 권장됩니다. 별칭 리디렉션을 영구적(301) 또는 임시(302)로 만들 수 있습니다. 두 유형 모두 자동으로 사용자를 지정된 목적지로 안내하지만 각각 사이트에 대해 고유한 영향을 미칩니다. 상황에 가장 적합한 유형을 고려하십시오.

**영구** : 301 코드를 사용하여 요청한 리소스가 영구적으로 재배치되었음을 사이트에 액세스하는 검색 엔진 및 기타 도구에 알립니다. 검색 엔진은 검색 결과에서 이전 URL을 새 목적지로 교체하고 브라우저는 불확실한 시간 동안 새 목적지를 캐시합니다. 301 코드는 도착 URL을 무한정 사용하려는 경우에만 사용해야 합니다. 이는 SEO를 개선하고 의도하지 않은 캐싱을 방지하는 데 도움이 됩니다.

**임시** : 302 코드를 사용하여 검색 엔진 및 사이트에 액세스하는 기타 도구에 요청된 리소스가 일시적으로 이동되었음을 알립니다. 302 코드는 사이트 순위에 영향을 주지 않고 문제를 해결하면서 A/B 테스트를 수행하거나 트래픽을 리디렉션하려는 경우에 유용합니다. Liferay는 기본적으로 302 코드를 사용합니다.

SEO에 대한 각 리디렉션 유형의 의미에 대해 자세히 알아보려면 [사이트 최적화](../../../optimizing-sites.md) 을 참조하세요. 또한 테스트할 때 302 코드를 사용하는 방법에 대해 알아보려면 [A/B 테스트](../../../optimizing-sites/ab-testing/ab-testing.md) 참조하세요.

```{note}
대량 리디렉션을 수행해야 하는 경우 [패턴 리디렉션 사용](./using-pattern-redirects.md)을 참조하세요.
```

## 별칭 리디렉션 만들기

1. **사이트 메뉴**(![Site Menu](../../../../images/icon-product-menu.png))을 열고 **구성** &rarr; **리디렉션** 로 이동합니다.

   ![리디렉션 애플리케이션의 별칭 탭으로 이동합니다.](./using-alias-redirects/images/01.png)

1. 별칭 탭에서 **추가**(![Add button](../../../../images/icon-add.png))을 클릭합니다.

1. **소스 URL** 필드에 고유한 상대 URL을 입력하여 리디렉션을 트리거하는 링크를 결정합니다. 입력한 값은 위에 표시되는 기본 URL에 추가됩니다.

   ![원본 URL 필드에 고유한 상대 URL을 입력합니다.](./using-alias-redirects/images/02.png)

1. **목적지 URL** 필드에 절대 URL을 입력하여 원하는 목적지에 대한 링크를 지정하십시오.

   이 대상은 내부 또는 외부일 수 있지만 항상 절대 URL로 입력해야 합니다. **URL 확인** 버튼(![Check URL Button](../../../../images/icon-shortcut.png))을 클릭하여 URL을 확인할 수도 있습니다.

   ![도착 URL 필드에 절대 URL을 입력합니다.](./using-alias-redirects/images/03.png)

1. **유형** 드롭다운 메뉴를 사용하여 리디렉션을 영구(301) 또는 임시(302)로 정의합니다.

   ![리디렉션이 영구적인지 일시적인지 확인합니다.](./using-alias-redirects/images/04.png)

1. 리디렉션에 대해 자동 **만료 날짜** 설정할 수 있습니다. 만료를 방지하려면 이 필드를 비워 두십시오.

   ![리디렉션이 만료되는지 확인합니다.](./using-alias-redirects/images/05.png)

1. 완료되면 **만들기** 클릭하여 리디렉션을 저장합니다.

저장되면 Liferay는 구성을 검증합니다. 다른 리디렉션이 지정된 소스 URL을 가리키는 경우 Liferay는 사용자에게 알리고 리디렉션 체인을 피하기 위해 기존 리디렉션을 업데이트할지 묻습니다. 충돌을 줄이기 위해 Liferay는 사용자가 동일한 소스 URL에 대해 여러 개의 별칭 리디렉션을 정의하고 URL을 자신에게 직접 리디렉션하는 것을 방지합니다.

![Liferay는 리디렉션 체인 생성을 방지하도록 도와줍니다.](./using-alias-redirects/images/06.png)

```{warning}
순환 리디렉션을 생성하지 마십시오. 이는 콘텐츠 친화적 URL에 대한 리디렉션을 정의할 때 쉽게 생성할 수 있습니다(예: 동일한 콘텐츠에 대한 서로 다른 친숙한 URL 간의 리디렉션). Liferay는 이러한 루프를 안정적으로 감지할 수 없습니다.
```

## 별칭 리디렉션 편집 및 제거

URL 리디렉션을 추가한 후에는 작업 버튼(![Actions button](../../../../images/icon-actions.png))을 클릭하고 편집 또는 삭제를 선택하여 언제든지 편집하거나 제거할 수 있습니다.

![언제든지 리디렉션을 편집하거나 제거할 수 있습니다.](./using-alias-redirects/images/07.png)

```{note}
리디렉션에 대한 변경 사항은 이전 리디렉션 구성을 캐시한 브라우저에 즉시 표시되지 않을 수 있습니다.
```
