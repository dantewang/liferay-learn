# 반응형 사이트 구축 개요
<!--TASK: Restructure article to include Content Page and Template information. Structure article with two main categories: Widget pages vs Content Pages/Templates. Use viewports language. -->
전 세계 모든 페이지 뷰의 절반 이상이 전화 및 태블릿과 같은 모바일 장치에서 발생합니다. 모든 사이트는 데스크톱에서와 마찬가지로 모바일 장치에서도 보기 좋게 표시되어야 합니다. Liferay DXP는 귀하의 사이트를 방문하는 장치에 관계없이 가능한 최고의 경험을 제공하도록 도울 수 있습니다.

즉시 사용 가능한 Liferay는 반응형 레이아웃을 사용합니다.

* Liferay의 UI 프레임워크를 사용하는 DXP 위젯 및 사용자 정의 위젯은 화면 크기에 맞게 자동으로 확장됩니다.

    ![위젯의 크기를 조정합니다.](./building-a-responsive-site/images/01.png)

* 탐색 및 제품 메뉴와 같은 UI 요소는 작은 화면에서 계속 사용할 수 있도록 자동으로 조정됩니다.

    ![기본 탐색은 크기를 조정합니다.](./building-a-responsive-site/images/02.png)

* 화면 너비가 작을 때 Liferay는 모든 내용을 읽을 수 있도록 열을 결합합니다.

   ![너비가 작을 때 열이 결합됩니다.](./building-a-responsive-site/images/03.png)

* 웹 개발자의 경우 테마 도구는 최적의 모바일 성능을 보장하는 데 도움이 됩니다.

기본 테마를 사용하는 경우 모바일 장치에 페이지를 표시하기 위해 해야 할 일은 페이지를 만드는 것뿐입니다. 모든 것이 의도한 대로 표시되는지 확인하는 도구도 있습니다. 장치 시뮬레이터(![Simulation](../../../images/icon-simulation.png))는 다른 장치에서 페이지가 어떻게 보이는지 보여줍니다.

## 장치 시뮬레이터 사용

페이지를 생성하거나 게시하기 전에 페이지를 검토할 때 모든 페이지의 오른쪽 상단에 있는 장치 시뮬레이터를 사용하여 테스트해야 합니다. 시뮬레이터는 다양한 디스플레이 유형에 따라 다양한 해상도로 현재 페이지를 보여줍니다. 미리 정의된 세 가지 옵션이 있습니다.

**데스크탑:** 페이지를 전체 크기로 표시하도록 너비를 고정합니다.

**태블릿:** 태블릿 크기의 상자에 페이지를 넣습니다.

**모바일:** 스마트폰 크기의 상자에 페이지를 넣습니다.

![시뮬레이션 패널은 여러 화면 크기를 정의합니다.](./building-a-responsive-site/images/04.png)

표시할 수 있는 두 가지 옵션도 있습니다.

**AutoSize:** 브라우저 창의 너비에 따라 페이지를 축소 및 확장합니다.

**사용자 지정:** 페이지 테스트를 위한 특정 크기를 입력합니다.

최신 모바일 브라우저는 데스크톱 브라우저와 동일한 기술을 기반으로 구축되기 때문에 시뮬레이터에 표시되는 동작은 모바일 장치의 동작과 일치해야 합니다. 레이아웃이 잘 보이고 모든 기능이 유지되는지 확인하는 것 외에도 자동 기능(예: 열이 낮은 해상도에서 결합되는 방식)이 의도하지 않은 영향을 미치지 않도록 하는 것도 중요합니다.

## 모바일 친화적인 페이지 디자인

모든 종류의 장치에서 좋은 경험을 제공하는 페이지를 구축한다는 것은 여전히 모든 수준의 웹 개발 및 게시에서 작업하는 것을 의미합니다. 테마 개발자는 Liferay의 프레임워크를 사용하는 테마를 만들어 모든 종류의 디스플레이에서 콘텐츠를 잘 확장해야 합니다. 디자이너는 페이지를 디자인할 때 여러 화면 크기를 염두에 두어야 합니다. 그리고 무엇이든 게시하기 전에 철저한 검토를 거쳐 최상의 경험을 제공하는지 확인해야 합니다.

## 모바일 장치 규칙 만들기(7.2만 해당)

> 모바일 장치 규칙은 Liferay 7.2 이전 버전에서만 사용할 수 있습니다. Liferay Mobile Device Detection Lite는 더 이상 사용되지 않으며 7.3에서 제거되었습니다.

모바일 장치 감지 시 사이트 동작을 제어하는 모바일 장치 규칙을 생성할 수 있습니다. 각각 고유한 특성을 가진 다양한 장치가 있습니다. 이들의 차이점을 구분하기 위해서는 장치의 특성을 알아내는 방법이 필요합니다. 일부 특성은 장치에서 제공하지만 대부분은 그렇지 않습니다. 이러한 이유로 수천 개의 장치에 대한 정보가 포함된 데이터베이스가 있습니다. 이러한 데이터베이스를 사용하면 장치 유형에서 장치에 대한 모든 세부 정보를 알 수 있습니다. 모바일 장치 규칙은 규칙에서 장치 특성을 사용할 수 있도록 장치 데이터베이스에 연결할 수 있습니다.

모바일 장치 규칙 생성에 대한 자세한 내용(Liferay 7.2만 해당)은 [모바일 장치 규칙 생성](./creating-mobile-device-rules.md)을 참조하십시오.
