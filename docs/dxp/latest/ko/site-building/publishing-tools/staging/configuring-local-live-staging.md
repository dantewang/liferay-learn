# 로컬 라이브 스테이징 구성

로컬 라이브 스테이징을 사용하면 스테이징 및 라이브 환경이 모두 동일한 Liferay 서버에서 호스팅됩니다. 활성화되면 Liferay DXP는 스테이징 환경 역할을 하는 사이트 또는 자산 라이브러리의 로컬 복제본을 생성하고 원본은 라이브 환경이 됩니다. 두 환경 모두 동일한 JVM, 데이터베이스, 선택한 애플리케이션 데이터 및 구성(예: 포털 속성)을 공유합니다.

## 사이트에 대한 로컬 라이브 스테이징 설정

1. *사이트 메뉴 바로가기* &rarr; *출판* &rarr; *준비*.

   ![사이트 메뉴에서 스테이징으로 이동합니다.](./configuring-local-live-staging/images/01.png)

1. *Local Live*을 선택하면 *Page Versioning* 및 *Staged Content*에 대한 추가 필드가 표시됩니다.

   ![로컬 라이브를 선택합니다.](./configuring-local-live-staging/images/02.png)

1. (사이트 전용) *페이지 버전 관리* 활성화 여부를 선택합니다.

   ![페이지 버전 관리를 활성화합니다.](./configuring-local-live-staging/images/03.png)

1. 준비하려는 데이터 및 콘텐츠 유형을 선택합니다.

   ![준비하려는 데이터 및 콘텐츠 유형을 선택합니다.](./configuring-local-live-staging/images/04.png)

   ```{warning}
   응용 프로그램이 확인되면 해당 데이터가 Staging에 복사되며 Live에서 직접 편집하지 못할 수 있습니다. 응용 프로그램을 선택 해제할 때 스테이징의 변경 사항이 손실될 수 있으므로 먼저 게시되었는지 확인하십시오. 자세한 내용은 [스테이징에서 데이터 및 콘텐츠 유형 관리](./managing-data-and-content-types-in-staging.md)를 참조하세요.
   ```

1. 복제 프로세스를 시작하려면 *저장* 을 클릭하십시오. 이 프로세스의 기간은 사이트 크기에 따라 다릅니다.

   ```{tip}
   페이지 버전 관리를 활성화한 후에만 업데이트가 기록되므로 복제 시간을 줄이고 사이트 업데이트 기록의 보다 완전한 기록을 기록하려면 사이트를 조기에 준비하십시오.
   ```

프로세스가 완료되면 로컬 라이브 스테이징을 사용할 준비가 된 것입니다. 스테이징 환경의 게시 기능 탐색에 대한 자세한 내용은 [사이트 스테이징 UI 참조](./site-staging-ui-reference.md) 을 참조하십시오.

## 자산 라이브러리에 대한 스테이징 설정

1. *전역 메뉴* (![Global Menu](../../../images/icon-applications-menu.png))을 열고 응용 프로그램 탭에서 *자산 라이브러리* 를 클릭합니다.

1. 원하는 자산 라이브러리를 열고 게시에서 *스테이징* 을 클릭합니다.

1. *로컬 라이브*을 선택합니다.

   ![로컬 라이브를 선택합니다.](./configuring-local-live-staging/images/05.png)

1. 준비할 애플리케이션 데이터를 선택합니다.

   자산 라이브러리의 경우 *문서 및 미디어* 및 *웹 콘텐츠*에 대한 데이터를 스테이징할 수 있습니다.

   ![준비할 애플리케이션 데이터를 선택합니다.](./configuring-local-live-staging/images/06.png)

1. *저장* 을 클릭하여 복제 프로세스를 시작합니다. 이 프로세스의 기간은 자산 라이브러리의 크기에 따라 다릅니다.

   프로세스가 완료되면 자산 라이브러리에서 로컬 라이브 스테이징을 사용할 준비가 된 것입니다.

## 로컬 라이브 스테이징 비활성화

어떤 이유로든 사이트 또는 자산 라이브러리에 대해 스테이징을 비활성화해야 하는 경우 스테이징 환경에서 비활성화할 수 있습니다. 그러나 로컬 라이브 스테이징을 비활성화하면 게시되지 않은 모든 콘텐츠와 함께 스테이징 환경이 삭제된다는 점에 유의하십시오. 이러한 이유로 스테이징을 비활성화하기 전에 필요한 모든 정보가 다른 곳에 게시되거나 보존되었는지 확인하십시오.

```{tip}
이 프로세스는 스테이징 환경의 크기에 따라 다소 시간이 걸릴 수 있습니다. DXP 인스턴스가 사용 중일 때는 스테이징을 비활성화하지 않는 것이 가장 좋습니다.
```

로컬 라이브 스테이징을 비활성화하려면 다음 단계를 따르십시오.

1. 사이트 또는 자산 라이브러리에서 스테이징 애플리케이션을 엽니다.

1. Application 표시줄에서 *Actions* ( ![Actions button](../../../images/icon-actions.png) )을 클릭하고 *Staging Configuration을 선택합니다.*.

   ![애플리케이션 표시줄에서 작업 버튼을 클릭하고 스테이징 구성을 선택합니다.](./configuring-local-live-staging/images/07.png)

1. 스테이징 구성에 대해 *없음* 을 선택하고 *저장*을 클릭합니다.

```{important}
대부분의 콘텐츠 유형의 경우 스테이징은 각 콘텐츠 항목의 최신 승인 버전만 게시합니다. 이러한 이유로 로컬 라이브 스테이징을 비활성화하면 이전 콘텐츠 버전(예: 문서)이 손실됩니다. 스테이징이 비활성화되면 Live에서 가장 최근에 게시된 버전만 보존됩니다. 웹 콘텐츠는 유일한 예외입니다.
```
