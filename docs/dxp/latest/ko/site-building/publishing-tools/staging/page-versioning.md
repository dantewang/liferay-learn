# 페이지 버전 관리

페이지 버전 관리를 활성화하면 귀하와 귀하의 팀이 사이트 위젯 페이지의 여러 버전에서 동시에 작업할 수 있습니다. 이러한 버전을 **변형** 이라고 하며 스테이징 바를 통해 만들고 관리합니다. 페이지 버전 관리는 기록을 유지하며 필요한 경우 준비된 페이지에 대한 변경 사항을 이전 버전으로 되돌릴 수 있습니다.

```{note}
스테이징은 위젯 페이지에 대한 변형 및 페이지 기록만 지원합니다. 콘텐츠 페이지 버전 관리의 경우 발행물을 사용하십시오. 자세한 내용은 [출판 도구 비교](../comparing-publishing-tools.md)를 참조하세요.
```

스테이징 구성 페이지를 통해 초기 페이지 설정 중 또는 나중에 언제든지 페이지 버전 관리를 활성화합니다. 활성화되면 개별 페이지 또는 페이지 그룹에 대한 변형을 생성, 병합 및 게시할 수 있습니다. 다른 사용자 역할에 대한 변형 권한을 관리할 수도 있습니다.

```{important}
페이지 버전 관리를 사용하고 나중에 비활성화하면 페이지 버전 기록이 제거되고 가장 최근에 게시된 페이지 버전만 보존됩니다. 이는 로컬 라이브 및 원격 라이브 스테이징 모두에 해당됩니다.
```

## 변형 유형

스테이징은 두 가지 변형 유형을 제공합니다.

* **사이트 페이지 변형** : 여러 사이트 페이지에 대한 변경 사항을 단일 변형으로 그룹화합니다.

* **페이지 변형** : 개별 페이지의 변형을 만듭니다. 사이트 페이지 변형 내에서 생성됩니다.

   ```{note}
   페이지 변형이 활성화된 사이트로 위젯 페이지를 가져오면 해당 페이지를 사이트 페이지의 기본 변형의 새 변형으로 가져옵니다.

   콘텐츠 페이지의 경우 [Experiences](../../../site-building/personalizing-site-experience/experience-personalization/content-page-personalization.md)을 사용하여 대체 페이지 버전을 만듭니다.
   ```

스테이징 사이트의 모든 변형은 동일한 콘텐츠를 공유하므로 두 변형 유형 모두 페이지에만 영향을 미치고 사이트 콘텐츠에는 영향을 미치지 않습니다. 즉, 각 변형은 동일한 콘텐츠 또는 응용 프로그램을 다른 방식으로 사용하고 구성할 수 있으며 각 사이트 페이지 변형은 다른 페이지를 포함할 수 있습니다.

또한 페이지 메타데이터는 버전이 지정되지 않습니다(예: 레이아웃 유형, 친숙한 URL). 이는 메타데이터에 대한 변경 사항이 모든 변형에 적용됨을 의미합니다. 예를 들어 페이지 템플릿이 수정된 경우 이러한 수정 사항은 템플릿에서 변경 사항을 상속하도록 구성된 모든 페이지에 전파됩니다.

```{important}
페이지 템플릿은 스테이징 프레임워크에서 인식되지 않습니다. 이는 준비된 사이트에서 기존 페이지 템플릿을 보거나 편집할 수 없음을 의미합니다. 준비된 사이트에서 만든 경우 준비가 비활성화되면 손실됩니다.
```

## 변형 만들기

1. 준비 표시줄에서 **작업** 버튼( ![Actions button](../../../images/icon-actions.png) )을 클릭하고 **사이트 페이지 변형** 또는 **페이지 변형** 을 선택합니다.

   ![Staging bar에서 Actions 버튼을 클릭하고 만들려는 변형 유형을 선택합니다.](./page-versioning/images/03.png)

1. 모달 창 우측 하단의 **Add** 버튼을 클릭합니다.

   ![모달 창 오른쪽 하단의 추가 버튼을 클릭합니다.](./page-versioning/images/04.png)

1. 새 변형에 대해 **이름** 및 **설명** 입력합니다. 언제든지 변형에 대해 이러한 필드를 수정할 수 있습니다.

   ![새 유사 콘텐츠의 이름과 설명을 입력합니다.](./page-versioning/images/05.png)

   사이트 페이지 변형의 경우 다음 옵션 중에서 선택하여 새 변형이 기존 변형에서 페이지를 복사하는지 결정할 수 있습니다.

   * **모든 사이트 페이지 변형** : 기존 사이트 페이지 변형에서 모든 페이지를 복사합니다.

   * **없음(빈 사이트 페이지 변형)** : 비어 있는 새 사이트 페이지 변형을 만듭니다.

   * **[기존 변형]** : 단일 사이트 페이지 변형에서 모든 페이지를 복사합니다.

   ![새 사이트 페이지 변형이 기존 변형에서 페이지를 복사하는지 확인](./page-versioning/images/06.png)

   ```{note}
   기존 변형에서 페이지를 복사하기로 결정하면 게시 준비가 된 것으로 표시된 최신 버전만 복사됩니다.
   ```

1. ** 추가를 클릭하여 새 변형을 만듭니다.

일단 생성되면 스테이징 바의 드롭다운 메뉴를 통해 페이지 또는 페이지 그룹의 각 버전 간에 전환할 수 있습니다. 자세한 내용은 [사이트 준비 UI 참조](./site-staging-ui-reference.md)을 참조하십시오.

![준비 막대의 드롭다운 메뉴를 통해 페이지 또는 페이지 그룹의 각 버전 간에 전환할 수 있습니다.](./page-versioning/images/02.png)

## 변형 병합

두 개의 사이트 페이지 변형을 병합할 수 있습니다. 이렇게 하면 콘텐츠에 영향을 주거나 덮어쓰지 않고 모든 페이지와 페이지 변형이 사이트 페이지 변형에 추가됩니다.

1. Staging bar에서 **Actions** 버튼( ![Actions button](../../../images/icon-actions.png) )을 클릭하고 **Site Pages Variation** 를 선택합니다.

   ![사이트 페이지 변형을 선택합니다.](./page-versioning/images/07.png)

1. 병합의 기반으로 사용할 변형에 대해 **작업** 버튼( ![Actions button](../../../images/icon-actions.png) )을 클릭하고 **병합** 선택합니다.

   ![병합의 기반으로 사용할 변형에 대한 작업 버튼을 클릭하고 병합을 선택합니다.](./page-versioning/images/08.png)

1. 기본 변형 위에 병합하려는 변형을 선택합니다.

   ![기본 변형 위에 병합하려는 변형을 선택합니다.](./page-versioning/images/09.png)

사이트 페이지의 두 변형이 병합되면 기본 변형에 없는 새 페이지가 여기에 추가됩니다. 페이지가 두 변형 모두에 존재하고 페이지의 적어도 하나의 버전이 게시 준비가 된 것으로 표시된 경우 준비됨으로 표시된 최신 버전이 기본 변형의 해당 페이지에 대한 새 변형으로 추가됩니다. 게시할 준비가 된 것으로 표시되지 않은 이전 변형은 복사되지 않습니다.
