
# 사이트 준비 UI 참조

사이트에 대해 로컬 또는 원격 라이브 스테이징을 활성화한 후 대부분의 페이지 구성 옵션은 스테이징 환경에서만 액세스할 수 있습니다. 이렇게 하면 사용자가 라이브 페이지를 직접 편집할 수 없습니다.

스테이징이 활성화되면 스테이징 환경에 추가 옵션이 나타납니다. 이러한 옵션은 스테이징 바, 라이브에 게시 메뉴 및 스테이징 페이지에 있습니다.

## 스테이징 바 및 편집 도구

사이트에 대해 준비가 활성화되면 각 사이트 페이지의 준비 표시줄을 통해 추가 옵션에 액세스할 수 있습니다. 이 표시줄에서 스테이징 및 라이브 환경 사이를 전환하고 스테이징된 페이지에 대해 **라이브에 게시** 클릭할 수 있습니다.

![사이트에 대해 준비가 활성화되면 각 사이트 페이지에서 준비 표시줄에 액세스할 수 있습니다.](./site-staging-ui-reference/images/01.png)

다음 표와 그림은 위젯 페이지의 스테이징 바 옵션과 편집 도구를 설명합니다.

## 살 위하여 간행하십시요

**Publish to Live** 클릭하면 게시 프로세스 구성을 위한 추가 옵션이 있는 모달 창이 나타납니다. **단순** 또는 **고급** 설정을 사용하여 게시할 수 있습니다.

![Publish to Live를 클릭하여 게시 모달 창에 액세스합니다.](./site-staging-ui-reference/images/03.png)

## 간단한 게시

**이름** : 게시 프로세스를 설명하는 이름을 **이름** 필드에 입력합니다.

**마지막 게시 프로세스 이후 변경 사항** : 마지막 게시 프로세스 이후에 적용된 단계적 변경 사항을 봅니다.

**게시할 페이지** : 현재 게시 프로세스에 포함된 페이지 수를 봅니다.

```{note}
스테이징 프레임워크에서 추적한 페이지 삭제 수를 표시합니다. 이 숫자는 라이브 사이트에서 삭제될 페이지 수가 아니라 스테이징 사이트에서 페이지 삭제 수를 계산합니다. 게시할 페이지 삭제 수와 스테이징 및 라이브 사이트 중 하나에 있는 실제 페이지 수 사이에 불일치가 있을 수 있습니다.
```

준비가 되면 **라이브에 게시** 클릭하여 게시를 시작합니다.

![Publish to Live를 클릭하여 게시를 시작합니다.](./site-staging-ui-reference/images/04.png)

## 고급 게시

고급 게시를 사용하면 게시 프로세스를 사용자 지정하는 데 사용할 수 있는 정보 및 설정에 액세스할 수 있습니다. **Custom** 탭에서 콘텐츠, 날짜, 페이지 등을 지정할 수 있습니다. 또는 **게시 템플릿** 탭을 클릭하여 게시 프로세스를 위해 미리 구성된 템플릿을 선택합니다. 고급 설정에 액세스하려면 모달 창에서 **고급 게시 프로세스** 로 전환을 클릭합니다.

![고급 설정을 사용하여 게시 프로세스를 사용자 정의하십시오.](./site-staging-ui-reference/images/05.png)

사용자 지정 고급 설정은 다음 섹션으로 나뉩니다.

**이름** : 게시 프로세스를 설명하는 이름을 **이름** 필드에 입력합니다.

**날짜** : 변경 사항을 즉시 게시할지 또는 나중에 게시하도록 **예약** 할지 여부를 선택합니다. 게시 프로세스의 반복 여부와 빈도를 결정할 수도 있습니다.

![변경 사항을 게시할 시기를 선택합니다.](./site-staging-ui-reference/images/06.png)

**삭제** : 가져오기 전에 응용 프로그램 데이터를 **할지 여부 선택** **개별 삭제 복제** .

![가져오기 전에 애플리케이션 데이터를 삭제할지 여부와 개별 삭제를 복제할지 여부를 선택합니다.](./site-staging-ui-reference/images/07.png)

**페이지** : 게시할 페이지 세트 변형 및 개별 페이지를 선택하고 선택한 페이지의 모양과 느낌을 선택합니다. 스테이징 사이트에 없는 라이브 사이트의 모든 페이지를 삭제하는 **누락된 페이지 삭제** 선택할 수도 있습니다. 페이지 메뉴에서 게시할 페이지를 선택하면 위젯과 해당 참조가 항상 게시됩니다.

![게시할 페이지 집합 변형 및 개별 페이지를 선택하고 선택한 페이지의 모양과 느낌을 선택합니다.](./site-staging-ui-reference/images/08.png)

사용자 지정 테마로 페이지를 게시하려면 스테이징 구성의 **모양 및 느낌** 제목 아래에서 테마 설정 옵션을 선택해야 합니다. 그렇지 않으면 기본 테마가 적용됩니다.

**콘텐츠** : 게시 프로세스에 포함할 콘텐츠를 선택합니다. 라디오 버튼을 사용하여 선택한 유형의 콘텐츠를 **모두** 게시하거나 날짜 및 시간 매개변수를 지정하여 게시된 콘텐츠를 제한할 수 있습니다. 이 옵션을 사용하여 스테이징에서 변경되어 게시 대기 중인 콘텐츠를 확인할 수도 있습니다.

![게시 프로세스에 포함할 콘텐츠를 선택합니다.](./site-staging-ui-reference/images/09.png)

| 옵션         | 묘사                                 |
|:---------- |:---------------------------------- |
| **모두** | 게시할 수 있는 모든 콘텐츠를 표시합니다.            |
| **최근 게시일에서** | 마지막 게시 이후 업데이트된 콘텐츠만 표시합니다.        |
| **날짜 범위** | 특정 기간 동안 업데이트된 콘텐츠를 보여줍니다.         |
| **마지막** | 지난 몇 시간 또는 며칠 동안 업데이트된 콘텐츠를 표시합니다. |

```{note}
콘텐츠 옵션을 선택한 후 *개수 새로 고침*을 클릭하여 결과 수를 업데이트합니다.
```

또한 각 콘텐츠 유형에 대해 **변경** 클릭하여 **버전 기록** , **어휘** , **미리보기 및 썸네일** 과 같이 각 유형에 포함된 항목 및 기타 설정을 관리할 수 있습니다. 자세한 내용은 [준비](./managing-data-and-content-types-in-staging.md) 의 데이터 및 콘텐츠 유형 관리를 참조하십시오.

```{important}
기본적으로 다음 위젯을 사용하여 표시되는 콘텐츠는 게시 프로세스에 항상 포함됩니다. 동적 데이터 목록 표시, 양식, 웹 콘텐츠 표시 및 Wiki 표시. Liferay DXP/Portal 7.4+의 경우 표시된 콘텐츠가 게시 프로세스의 구성에 명시적으로 포함된 경우에만 게시되도록 이 동작을 비활성화할 수 있습니다. 자세한 내용은 [표시된 콘텐츠의 자동 게시 구성](./configuring-automatic-publishing-of-displayed-content.md)을 참조하십시오.
```

**권한** : 변경 사항이 게시될 때 페이지 및 위젯에 대한 권한을 포함할지 여부를 선택합니다.

![변경 사항이 게시될 때 페이지 및 위젯에 대한 권한을 포함할지 여부를 선택합니다.](./site-staging-ui-reference/images/10.png)

페이지나 콘텐츠를 게시할 때마다 고급 옵션을 수동으로 구성하는 대신 나중에 사용할 수 있도록 스테이징 구성을 [**게시 템플릿**](#publish-templates) 으로 저장할 수 있습니다. 게시 프로세스를 위해 저장된 템플릿을 보고 선택하려면 **게시 템플릿** 탭을 클릭합니다.

![게시 프로세스를 위해 저장된 템플릿을 보고 선택하려면 게시 템플릿 탭을 클릭합니다.](./site-staging-ui-reference/images/11.png)

준비가 되면 **라이브에 게시** 클릭하여 게시를 시작합니다.

## 스테이징 페이지

스테이징을 활성화하면 **게시** 탭에서 사용할 수 있는 옵션이 수정됩니다. 라이브 환경에서는 **내보내기** 기능에만 액세스할 수 있습니다. 스테이징 환경에서는 **가져오기** 및 **스테이징** 기능에만 액세스할 수 있습니다.

스테이징 페이지에서 게시 프로세스 보기, 게시 템플릿 생성 및 관리, 사이트 스테이징 수정 또는 비활성화와 같은 새로운 옵션에 액세스할 수도 있습니다.

![스테이징 페이지에서 새 옵션에 액세스할 수도 있습니다.](./site-staging-ui-reference/images/12.png)

### 이전 및 향후 게시 프로세스

스테이징을 사용하여 게시하면 **프로세스** 로 캡처되고 나중에 참조할 수 있도록 저장됩니다. 이러한 프로세스를 보고 관리하려면 **사이트 메뉴** &rarr; **게시** &rarr; **스테이징** 로 이동하십시오.

**프로세스** 탭에는 완료된 스테이징 프로세스 목록이 있습니다. 여기에서 프로세스의 **작업** 버튼( ![Actions button](../../../images/icon-actions.png) )을 클릭하여 **다시 실행** , **지우** 이전 간행물의 **요약** 볼 수 있습니다.

![스테이징 페이지에서 이전 게시 프로세스를 보고 관리합니다.](./site-staging-ui-reference/images/13.png)

**추가**(![Add button](../../../images/icon-add.png)) 버튼을 클릭하여 사용자 정의 구성 또는 미리 정의된 게시 템플릿을 사용하여 새 게시 프로세스를 생성할 수도 있습니다.

**예약** 탭에서 예약된 모든 게시 프로세스를 볼 수 있습니다.

### 템플릿 퍼블리싱

유사한 스테이징 구성을 정기적으로 만드는 경우 고유한 **게시 템플릿** 을 만들어 시간을 절약할 수 있습니다. 템플릿을 생성하려면 **스테이징** 페이지의 **애플리케이션** 표시줄에서 **작업**( ![Actions button](../../../images/icon-actions.png) )을 클릭합니다. 그런 다음 **Publish Templates** 선택합니다.

![스테이징 페이지의 애플리케이션 표시줄에서 작업 버튼을 클릭합니다.](./site-staging-ui-reference/images/14.png)

여기에서 빈번한 게시 구성을 위한 템플릿을 만들고 관리할 수 있습니다. 그런 다음 **작업** 버튼( ![Actions button](../../../images/icon-actions.png) )을 클릭하고 **게시** 선택하여 템플릿을 사용할 수 있습니다.

![빈번한 게시 구성을 위한 템플릿을 만들고 관리합니다.](./site-staging-ui-reference/images/15.png)

Staging Bar의 **Publish to Live** 버튼을 사용하여 페이지나 콘텐츠를 게시할 때 템플릿을 사용할 수 있습니다. 자세한 내용은 [Advanced Publishing](#advanced-publishing) 참조하십시오.

### 스테이징 구성

Staging을 설정한 후 **Staging** 페이지의 **Application** 막대에서 **Actions**( ![Actions button](../../../images/icon-actions.png) )을 클릭하여 Staging Configuration 페이지에 액세스할 수 있습니다. 그런 다음 **Staging Configuration** 선택합니다.

![Staging 구성에 액세스하려면 Staging 페이지의 Application 표시줄에서 Actions 버튼을 클릭합니다.](./site-staging-ui-reference/images/16.png)

여기에서 사이트의 스테이징을 비활성화하거나 [페이지 버전 관리](page-versioning.md) 설정을 수정할 수 있습니다. 단, **Staged Content** 옵션은 초기 설정 후 수정할 수 없습니다.
