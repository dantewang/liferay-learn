# 단일 자산 및 위젯 게시

전체 게시 프로세스를 거치지 않고 필요한 경우 단일 자산 및 위젯을 게시할 수 있습니다. 모든 콘텐츠를 한 번에 게시하는 것보다 훨씬 빠릅니다.

## 단일 자산 게시

이러한 방식으로 자산을 게시하려면 사용자에게 *애플리케이션 정보 내보내기/가져오기* 및 *게시 준비*권한이 있어야 합니다. 자세한 내용은 [스테이징 권한 관리](./managing-staging-permissions.md) 을 참조하십시오.

1. *상품* 메뉴에서 *콘텐츠 & 데이터* 로 이동하여 게시할 콘텐츠 또는 데이터 유형을 선택합니다.

1. 게시하려는 자산에 대해 *작업* 버튼( ![Actions button](../../../images/icon-actions.png) )을 클릭하고 *게시를 선택합니다. Live*.

   ![게시하려는 자산에 대한 작업 버튼을 클릭하고 라이브에 게시를 선택합니다.](./publishing-single-assets-and-widgets/images/01.png)

확인하면 게시 상태를 볼 수 있는 프로세스 페이지로 이동합니다.

표준 프로세스와 유사하게 단일 자산 게시에는 이미지, 사용자 지정 템플릿 또는 폴더와 같은 연결된 종속성이 포함됩니다.

```{note}
단일 자산 게시는 페이지 범위 콘텐츠에 대해 지원되지 않습니다.
```

## 단일 위젯 게시

> 위젯 페이지에서만 사용 가능

스테이징은 일반적으로 콘텐츠를 게시하는 데 사용되지만 위젯을 게시할 수도 있습니다. 예를 들어 위젯의 제목을 수정하고 변경 사항을 Live에 게시할 수 있습니다. 이는 위젯 구성이 항상 준비되기 때문에 가능합니다. 페이지에 있는 위젯을 게시하려면 먼저 페이지를 게시해야 합니다.

위젯을 변경한 후 위젯에 대한 *작업* 버튼( ![Actions button](../../../images/icon-actions.png) )을 클릭하고 *스테이징*을 선택하여 위젯을 게시할 수 있습니다.

![위젯의 작업 버튼을 클릭하고 스테이징을 선택합니다.](./publishing-single-assets-and-widgets/images/04.png)
