# 페이지 게시 구성

기본적으로 하위 페이지를 Live에 게시하면 상위 페이지도 게시됩니다. 이것은 분리된 페이지를 방지합니다. 그러나 항상 상위 페이지를 Live에 게시하면 성능에 영향을 미치고 변경 사항을 조기에 상위 페이지에 게시할 수 있습니다. 원하는 경우 항상 상위 페이지를 포함하는 대신 선택한 페이지만 게시하도록 이 동작을 구성할 수 있습니다.

```{note}
이 구성은 스테이징에만 영향을 미칩니다. 사이트를 LAR 파일로 수동으로 내보내면 항상 모든 상위 페이지가 포함됩니다.
```

시스템에 대한 하위/상위 게시를 구성하려면 다음 단계를 따르십시오.

1. *글로벌 메뉴* (![Global Menu](../../../images/icon-applications-menu.png))을 열고 *제어판*을 클릭한 다음 *시스템 설정* &rarr; *인프라*로 이동합니다.

1. 왼쪽 메뉴의 Virtual Instance Scope에서 *Staging* 을 클릭합니다.

1. *기본적으로 상위 페이지 게시*를 선택하거나 선택 취소합니다.

   선택하면 상위 페이지가 항상 Live에 게시됩니다.

   선택하지 않으면 상위 페이지는 Live에 아직 존재하지 않는 경우에만 게시됩니다. 상위 페이지가 이미 게시된 경우 상위 페이지가 이미 라이브 사이트에 있는지 확인하는 데 사용되는 참조 요소만 추가됩니다.

1. *저장*을 클릭합니다.
