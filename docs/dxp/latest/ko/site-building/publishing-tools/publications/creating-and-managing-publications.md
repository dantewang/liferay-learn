# 출판물 생성 및 관리

> 사용 가능: Liferay DXP/Portal 7.3+

Publications를 통해 사용자는 인스턴스 범위의 변경 블록을 만들 수 있습니다. 각 간행물은 인스턴스 범위이므로 Liferay UI의 어디에서나 액세스할 수 있고 여러 사이트의 변경 사항을 포함할 수 있습니다. 기본적으로 사용자는 자신이 만든 발행물만 보고 관리할 수 있습니다. 그러나 사용자는 서로를 초대하여 출판물에서 공동 작업을 할 수 있습니다. 자세한 내용은 [출판물 공동 작업](./collaborating-on-publications.md) 참조하십시오.

[$LIFERAY_LEARN_YOUTUBE_URL$]=https://www.youtube.com/embed/QifaRHHuoX8

출판물이 [활성화](./enabling-publications.md)되면 출판물 응용 프로그램 페이지 또는 출판물 드롭다운 메뉴를 통해 출판물을 생성, 편집, 게시 또는 제거할 수 있습니다.

```{important}
Liferay 버전을 업그레이드하기 전에 모든 활성 간행물을 게시했는지 확인하십시오. 업그레이드 후 활성 게시는 오래된 것으로 표시되며 게시하거나 변경할 수 없습니다. 또한 업그레이드 전에 게시된 발행물은 되돌릴 수 없습니다. 이러한 게시에 대해 *되돌리기* 옵션이 제거되었습니다.
```

## 새 발행물 만들기

1. **출판물** 드롭다운 메뉴를 클릭하고 **새 출판물 만들기** 선택합니다.

   ![출판물 드롭다운 메뉴에서 새 출판물 만들기를 클릭합니다.](./creating-and-managing-publications/images/01.png)

   또는 출판물 애플리케이션을 열고 **추가**(![Add button](../../../images/icon-add.png))**진행** 탭을 클릭합니다.

1. 게시에 대해 **이름** 및 **설명** 입력합니다.

   또는 [게시 템플릿](./using-publication-templates.md) 선택하여 이러한 필드를 자동으로 채웁니다.

   ```{note}
   간행물 이름은 고유하지 않아도 됩니다. 그러나 조직에는 고유한 이름을 사용하는 것이 좋습니다.
   ```

1. **만들기** 클릭합니다.

![발행물의 이름과 설명을 입력하고 만들기를 클릭합니다.](./creating-and-managing-publications/images/02.png)

발행물이 만들어지면 변경 내용을 추가할 수 있습니다. 편집 및 게시 프로세스에 대한 자세한 내용은 [변경 및 게시](./making-and-publishing-changes.md) 참조하십시오.

```{note}
생성 후 각 발행물은 발행물에서 변경된 내용과 충돌하지 않는 한 프로덕션에서 변경된 내용으로 자동 업데이트됩니다.
```

## 진행 중인 간행물 관리

**진행 중** 탭에서 보기 권한이 있는 모든 활성 게시되지 않은 간행물의 표를 볼 수 있습니다. 테이블은 **수정 날짜** 로 정렬되며 각 게시의 이름, 설명, 마지막 수정 시간, 생성 시간 및 소유자를 포함합니다.

![진행 중 탭에는 게시되지 않은 모든 활성 발행물이 나열됩니다.](./creating-and-managing-publications/images/03.png)

여기에서 게시의 작업 버튼( ![Actions button](../../../images/icon-actions.png) )을 클릭하여 다음 작업을 수행할 수 있습니다.

**출판물 작업** : 출판물을 선택하여 변경 사항을 추가합니다. 선택한 후에는 간행물 드롭다운 메뉴를 통해 간행물과 제작 간을 쉽게 전환할 수 있습니다. 자세한 내용은 [출판물 변경하기](./making-and-publishing-changes.md#making-changes-in-a-publication) 을 참조하십시오.

**편집** : 발행물의 이름이나 설명을 수정합니다.

**변경 사항 검토** : 발행물에 포함된 모든 변경 사항을 봅니다. 자세한 내용은 [출판물 변경 검토](./making-and-publishing-changes.md#reviewing-publication-changes) 참조하십시오.

**사용자 초대** : 사용자를 초대하여 발행물에서 공동 작업을 수행합니다. 자세한 내용은 [출판물 공동 작업](./collaborating-on-publications.md) 참조하십시오.

**게시** : 게시에 대한 게시 프로세스를 시작합니다. 이렇게 하면 충돌을 해결할 수 있는 게시 페이지로 사용자가 리디렉션됩니다. 충돌이 없으면 **게시** 클릭하여 변경 사항을 적용할 수 있습니다. 자세한 내용은 [변경 사항 게시](./making-and-publishing-changes.md#publishing-changes) 및 [충돌 해결](./resolving-conflicts.md) 참조하십시오.

**일정** : 게시의 변경 사항을 프로덕션에 적용하도록 예약합니다. 자세한 내용은 [게시 변경 사항](./making-and-publishing-changes.md#publishing-changes) 참조하십시오.

**권한** : 사용자 역할에 게시 관련 권한을 할당합니다. 이러한 방식으로 구성된 사용 권한은 개별 게시로 범위가 지정됩니다. <!--TASK: Link to the permissions article once finished.-->

**삭제** : 인스턴스에서 발행물을 제거합니다. 선택하면 선택을 확인하라는 메시지가 표시됩니다.

```{warning}
발행물을 삭제하면 데이터베이스에서 모든 변경 사항이 영구적으로 제거되며 취소할 수 없습니다. 간행물을 삭제하기 전에 보존하려는 데이터를 모두 저장했는지 확인하십시오.
```

## 예약 게시 관리

**일정** 탭에서 보기 권한이 있는 모든 예약 게시의 테이블을 볼 수 있습니다. 기본적으로 예약된 게시는 **이름** 로 나열되지만 수정된 날짜 또는 각 게시가 게시 예정일로 주문할 수 있습니다.

![일정 탭에는 인스턴스에 대해 예약된 모든 게시가 나열됩니다.](./creating-and-managing-publications/images/04.png)

일단 예약되면 게시에 변경 사항을 추가할 수 없습니다. 그러나 작업 버튼(![Actions button](../../../images/icon-actions.png))을 클릭하여 다음 작업을 수행할 수 있습니다.

**예약 취소** : 예약된 게시를 취소하고 게시를 **진행 중** 상태로 되돌립니다.

**일정 변경** : 프로덕션에 변경 사항을 게시할 대체 날짜 및 시간을 설정합니다.

**변경 사항 검토** : 발행물에 포함된 모든 변경 사항을 봅니다.

## 출판 이력 관리

**기록** 탭에서 다른 사용자가 게시한 게시를 포함하여 DXP 인스턴스에 대해 게시된 모든 게시의 테이블을 볼 수 있습니다. 기본적으로 발행물은 **Published Date** 로 정렬되지만 이름으로 정렬할 수도 있습니다.

![기록 탭에는 인스턴스에 대해 이전에 게시된 모든 게시가 나열됩니다.](./creating-and-managing-publications/images/05.png)

여기에서 게시에 대해 **되돌리기** 클릭하여 변경 사항을 되돌리는 새 게시를 자동으로 만들 수 있습니다. 게시 되돌리기의 일부로 DXP 인스턴스를 추가로 변경할 수도 있습니다. 자세한 내용은 [변경 사항 되돌리기](./reverting-changes.md) 참조하십시오.

```{note}
인스턴스에 게시된 모든 게시는 기록 탭에 나열되지만 게시 소유자와 공동 작업자만 게시의 변경 사항 목록을 보거나 되돌릴 수 있습니다.
```
