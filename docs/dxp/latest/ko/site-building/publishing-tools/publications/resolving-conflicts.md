
# 충돌 해결

게시 프로세스를 시작하면 Liferay는 호환성 검사를 수행하여 게시와 프로덕션 간에 충돌이 있는지 확인합니다. 일부 충돌은 자동으로 해결될 수 있지만 다른 충돌은 수동 해결이 필요합니다. 변경 사항을 게시하려면 모든 충돌을 해결해야 합니다.

[$LIFERAY_LEARN_YOUTUBE_URL$]=https://www.youtube.com/embed/4w16zEFoP8o

```{note}
게시 충돌은 게시에서 이미 변경이 이루어진 *후에* 프로덕션이 변경된 경우에만 발생합니다. 이는 해당 변경 사항이 발행물의 변경 사항과 충돌하지 않는 한 프로덕션 변경 사항으로 각 발행물이 자동으로 업데이트되기 때문입니다.
```

## 충돌 자동 해결

일부 충돌의 경우 Liferay는 자동 해결을 제공할 수 있습니다. 기본적으로 자동 해결은 특정 종류의 충돌에 대해 미리 구성된 솔루션이므로 추가 작업 없이 변경 사항을 게시할 수 있습니다. 이러한 솔루션에는 현재 발행물에서 콘텐츠를 추가, 수정 또는 제거하거나 제작 중인 콘텐츠 버전을 발행물의 버전으로 교체하는 것이 포함될 수 있습니다.

자동 솔루션을 사용할 수 있으면 게시하는 동안 알림을 받게 됩니다. 그런 다음 **보기** 클릭하여 솔루션의 세부 정보를 검토할 수 있습니다. 자동 솔루션을 구현하려면 **Publish** 클릭하면 됩니다.

![자동 솔루션을 구현하려면 게시를 클릭하기만 하면 됩니다.](./resolving-conflicts/images/01.png)

## 충돌 수동 해결

일부 충돌의 경우 Liferay는 자동 해결을 제공할 수 없습니다. 즉, 게시를 마치기 전에 추가 작업을 수행하여 충돌을 해결해야 합니다. 수동 해결이 필요한 경우 Publications는 충돌 유형을 식별하고 이를 해결하라는 메시지를 표시합니다.

![간행물의 충돌하는 변경 사항을 편집하거나 삭제합니다.](./resolving-conflicts/images/02.png)

[충돌 유형](#understanding-conflict-types) 에 따라 간행물에서 다음 작업 중 하나를 수행하라는 메시지를 표시할 수 있습니다.

* **편집** : 이것은 현재 발행물의 충돌 원인으로 리디렉션합니다. 여기에서 충돌을 해결하는 데 필요한 사항을 변경할 수 있습니다.

* **폐기** : 폐기할 모든 변경 사항 목록을 표시합니다. 그런 다음 **폐기** 클릭하여 간행물에서 영구적으로 제거할 수 있습니다.

```{warning}
삭제는 영구적이며 되돌릴 수 없습니다. 간행물에서 삭제하기 전에 유지하려는 모든 변경 사항을 저장했는지 확인하십시오.
```

## 충돌 유형 이해

사용자는 **중복** , **수정** , **삭제 수정** , **수정 삭제** , **누락 요구 사항** , **추가** 등 6가지 일반적인 유형의 충돌이 발생할 수 있습니다.

### 중복 충돌

![발행물은 복제 충돌을 해결하도록 알려줍니다.](./resolving-conflicts/images/03.png)

이러한 충돌은 프로덕션의 어떤 항목이 게시에서 변경된 것과 동일한 고유 값을 사용할 때 발생합니다. 이는 고유한 값(예: 폴더, URL, 역할 이름)이 필요한 모든 유형의 구성 요소로 인해 발생할 수 있습니다.

다음 예를 고려하십시오. 게시에 'Marketing'이라는 새 폴더를 추가했고 생산 중인 사용자가 동일한 이름을 사용하여 폴더를 만든다고 가정합니다. 동일한 리포지토리의 폴더는 고유한 이름을 가져야 하므로 변경 사항을 게시하려고 하면 중복 충돌이 발생합니다.

중복 충돌을 해결하기 위해 사용자는 발행물의 값을 편집하거나 변경 사항을 취소할 수 있습니다.

### 수정 충돌

![간행물은 수정 충돌을 해결하도록 알려줍니다.](./resolving-conflicts/images/04.png)

이러한 충돌은 게시에서 수정한 엔터티가 프로덕션에서 수정될 때(예: 편집된 웹 콘텐츠 기사가 이동될 때) 발생합니다. 일부 수정 충돌은 자동으로 해결될 수 있지만 다른 충돌은 수동 해결이 필요합니다.

수정 충돌을 해결하기 위해 충돌하는 자료를 편집하거나 간행물에서 충돌하는 변경 내용을 삭제하거나 프로덕션 버전을 교체할 수 있습니다.

### 삭제 수정 및 수정 삭제 충돌

![간행물은 삭제 수정 및 수정 삭제 충돌을 해결하도록 알려줍니다.](./resolving-conflicts/images/05.png)

이러한 충돌은 엔터티가 게시에서 수정된 다음 프로덕션에서 삭제(삭제 수정)되거나 그 반대(수정 삭제)에 발생합니다. 이 충돌에 대해 제공되는 유일한 해결 방법은 게시의 충돌하는 변경 내용을 삭제하는 것입니다.

### 누락된 요구 사항 충돌

![간행물은 누락된 요구 사항 충돌을 해결하도록 알려줍니다.](./resolving-conflicts/images/06.png)

이러한 충돌은 상위 항목이 프로덕션에서 제거된 상태에서 하위 엔터티가 발행물에서 생성되거나 편집될 때 발생합니다. 예를 들어 발행물에서 하위 폴더를 추가하거나 수정할 때 프로덕션에서 폴더를 삭제하는 것이 포함됩니다.

누락된 요구 사항 충돌을 해결하려면 간행물에서 충돌하는 변경 사항을 삭제하거나 프로덕션에서 삭제를 되돌릴 수 있습니다. 프로덕션에 게시된 변경 사항을 되돌리는 방법은 [변경 사항 되돌리기](./reverting-changes.md) 참조하십시오.

### 더하기 충돌

![간행물은 추가 충돌을 해결하도록 알려줍니다.](./resolving-conflicts/images/07.png)

이러한 충돌은 사용자가 이미 되돌린 삭제를 포함하여 게시를 되돌리려고 할 때만 발생합니다. 시스템에서 이미 다시 추가된 항목을 다시 추가할 수 없기 때문에 사용자는 삭제가 포함된 동일한 게시를 두 번 되돌릴 수 없습니다.
