# 컨텐츠 페이지 개인화

콘텐츠 페이지를 다양한 사용자 그룹에 맞게 조정하여 각 그룹에 일반적인 것이 아닌 개인화된 경험을 제공할 수 있습니다. 콘텐츠 페이지 개인화를 활용하여 A/B 테스트를 수행할 수도 있습니다(자세한 내용은 [콘텐츠 페이지 경험 및 A/B 테스트](#content-page-experiences-and-ab-testing)참조).

다양한 사용자를 위해 콘텐츠 페이지를 개인화하는 것은 2단계 프로세스입니다.

1. [개인화된 경험을 제공하려는 사용자에 대한 세그먼트](../segmentation/creating-and-managing-user-segments.md) 을 정의합니다.
1. [세그먼트에 대한 맞춤형 콘텐츠로 경험](./creating-and-managing-experiences.md) 을 생성합니다.

## 경험 작동 방식 이해

> 일부 경험 기능은 Liferay 7.4+, 7.3 수정팩 1+ 및 7.2 수정팩 11+에서만 사용할 수 있습니다.

동일한 콘텐츠 페이지에 대해 여러 경험을 생성할 수 있으며 각 경험은 특정 요구 사항을 충족합니다. 예를 들어 로그인하는 사용자와 익명으로 검색하는 사용자를 위해 서로 다른 경험을 만들 수 있습니다. 각 경험에는 각 세그먼트에 가장 적합한 콘텐츠와 레이아웃이 포함되어 있습니다.

동일한 콘텐츠 페이지에 대해 여러 경험을 만들 때 다음 정보를 고려하십시오.

- 모든 콘텐츠 페이지에는 *기본* 경험이 포함됩니다. 새 경험을 만들 때 *기본값* 이 템플릿으로 사용됩니다.
- 기본 환경을 수정할 수 있지만 기본을 기반으로 하는 기존 환경은 변경되지 않습니다.
- 기본을 제외한 모든 경험을 복제할 수 있습니다.
- 위로(![Up](../../../images/icon-angle-up.png)) 및 아래로(![Down](../../../images/icon-angle-down.png)) 컨트롤을 사용하여 선택 대화 상자에서 경험을 재정렬할 수 있습니다.
- 동일한 콘텐츠 페이지에 여러 경험이 있는 경우 순서에 따라 선호도가 결정됩니다.

  - 목록 맨 위에 있는 경험이 아래 경험보다 우선합니다.
  - 사용자가 둘 이상의 경험에 대한 기준을 충족하면 먼저 나열된 경험이 활성 경험입니다.
  - *누구나* 세그먼트를 대상으로 하는 경험 아래에 배치되면 경험이 비활성화됩니다.
  - 새 경험은 *기본* 경험 아래에 나열되며 기본적으로 비활성화됩니다.

  ```{tip}
  여러 경험이 동일한 세그먼트를 대상으로 하는 경우 Liferay DXP는 경험 이름 옆에 활성 항목을 표시합니다.
  ```

경험의 기본 설정이 작동하는 방식을 더 잘 이해하려면 4개의 사용자 정의 경험이 있는 콘텐츠 페이지의 이 예를 고려하십시오.

- *익명 사용자* 및 *익명 사용자 프로모션*, 인증되지 않은 사용자 세그먼트에 적용됩니다.
- *인증된 사용자*: *인증된 사용자* 세그먼트에 적용됩니다.
- *No Customization*: *Any* Segment에 적용됩니다.

  ![사용자 인터페이스의 경험 순서에 따라 기본 설정이 결정됩니다.](./content-page-personalization/images/06.png)

이 예에서:

- 인증되지 않은 사용자에게는 경험 *익명 사용자* ( _활성_ 레이블이 있는 사용자)이 표시됩니다. 이 사용자는 *Anonymous User*아래에 나타나기 때문에 *Anonymous User Promotions* Experience를 볼 수 없습니다.
- 인증된 사용자에게는 *인증된 사용자* 경험이 표시되지 않습니다. *사용자 정의 없음* 경험이 *모든 사람* 세그먼트에 적용되고 첫 번째로 나열되기 때문입니다.
- *익명 사용자* 또는 *인증된 사용자* 세그먼트에 속하지 않는 모든 사람은 *모든 사용자*을 대상으로 하기 때문에 *사용자 지정 없음* 경험을 봅니다.

## 콘텐츠 페이지 경험 및 A/B 테스트

[DXP에서](../../optimizing-sites/ab-testing/creating-ab-tests.md) /B 테스트를 생성할 때 테스트에 대한 경험을 선택합니다. 이 경험은 기본 경험 또는 귀하가 만든 다른 경험일 수 있습니다.

A/B 테스트 결과를 방해할 수 있는 경험의 변경을 방지하기 위해 실행 중인 A/B 테스트의 일부인 경험을 편집할 수 없습니다. 이 상황에서 경험은 이름 옆에 사물함 아이콘(![locker](../../../images/icon-lock.png))을 표시합니다.

Liferay DXP 및 Liferay Analytics의 A/B 테스트에 대한 자세한 내용은 [A/B 테스트](../../optimizing-sites/ab-testing/ab-testing.md)을 참조하십시오.
