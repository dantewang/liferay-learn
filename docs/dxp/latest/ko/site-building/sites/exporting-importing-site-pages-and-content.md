# 사이트 페이지 및 콘텐츠 내보내기/가져오기

Liferay 사이트에는 LAR(Liferay Archive) 파일로 사이트 데이터를 추출하고 전송하기 위한 내보내기/가져오기 애플리케이션이 포함되어 있습니다. 사이트 수준에서 이러한 도구에 액세스할 수 있습니다.

```{note}
또한 개별 사이트 애플리케이션 및 [페이지 위젯](../creating-pages/page-fragments-and-widgets/using-widgets/configuring-widgets/exporting-importing-widget-data.md에 대한 내보내기 및 가져오기 기능에 액세스할 수 있습니다. ) .
```

## 사이트 LAR 속성 및 요구 사항

LAR에는 다음과 같은 속성과 요구 사항이 있습니다.

* LAR은 버전에 따라 다릅니다. 각 서버의 Liferay 버전이 동일한 경우 다른 Liferay 서버의 사이트로 가져올 수 있습니다.

* 일부 이름 충돌은 자동으로 처리됩니다. 예를 들어, 가져오는 LAR과 사이트 모두에 동일한 친숙한 URL이 있는 페이지가 있는 경우 친숙한 URL에 숫자가 추가되고 충돌이 더 이상 발생하지 않을 때까지 증가합니다. 카테고리 이름도 마찬가지입니다.

* LAR을 사용하여 사이트를 복원할 수 있지만 사이트를 완전히 삭제한 다음 같은 이름으로 새 사이트를 만들어 LAR을 가져와야 합니다. 이렇게 하면 LAR과 사이트 모두에 존재하는 콘텐츠 간의 잠재적인 데이터 충돌을 방지할 수 있습니다.

```{important}
주기적으로 LAR을 내보내는 것은 백업 솔루션이 *아닙니다*. 두 환경 간에 데이터를 마이그레이션하는 데에만 사용해야 합니다. Liferay 설치 백업에 대해 알아보려면 [Liferay Portal 설치 백업](../../installation-and-upgrades/maintaining-a-liferay-installation/backing-up.md)을 참조하십시오.
```

## 사이트 페이지 및 콘텐츠 내보내기

사이트 페이지 및 콘텐츠를 내보내려면 다음 단계를 따르십시오.

1. *사이트 메뉴* (![Site Menu](../../images/icon-product-menu.png))을 열고 *게시*확장한 다음 *내보내기*클릭합니다.

1. *추가* 버튼(![Add Button](../../images/icon-add.png))을 클릭하여 새 내보내기 프로세스를 만듭니다.

   그러면 내보내기 템플릿이 열립니다.

   ![내보내기 템플릿의 이름을 지정하고 구성합니다.](./exporting-importing-site-pages-and-content/images/01.png)

1. 내보내기 프로세스의 제목을 입력합니다.

1. 내보낼 사이트 페이지 및 콘텐츠를 선택하여 내보내기 프로세스를 구성합니다.

   *페이지* 에서 내보내기에 포함할 페이지 및 페이지 세트를 선택할 수 있습니다. 테마 설정, 로고, 사이트 페이지 설정 및 사이트 템플릿 설정이 포함되는지 여부도 결정할 수 있습니다.

   *Content*아래에서 내보내기에 포함할 애플리케이션을 선택할 수 있습니다. 또한 선택한 애플리케이션의 모든 데이터를 내보낼지 아니면 포함된 애플리케이션 데이터를 날짜 또는 시간별로 필터링할지 결정할 수 있습니다.

   *권한*아래에서 내보내기에 페이지 및 콘텐츠 권한을 포함할지 여부를 결정합니다.

1. *내보내기* 클릭하여 내보내기 프로세스를 시작합니다.

완료되면 *다운로드* 버튼(![Download Button](../../images/icon-download.png))을 클릭하여 생성된 LAR 파일을 다운로드합니다. 이제 LAR을 다른 Liferay 사이트로 가져올 수 있습니다.

![LAR 파일을 다운로드합니다.](./exporting-importing-site-pages-and-content/images/02.png)

```{note}
생성된 LAR에는 내보내기 구성 중에 선택한 모든 데이터가 포함됩니다. 여기에 개인 정보나 민감한 정보가 포함된 경우 LAR 파일이 안전한 위치에 저장되어 있는지 확인하십시오.
```

## 사이트 페이지 및 콘텐츠 가져오기

```{important}
내보낸 사이트에서 템플릿을 사용하는 경우 사이트를 가져오기 전에 먼저 사이트 템플릿을 Liferay 설치로 내보내고 가져와야 합니다. 자세한 내용은 [사이트 템플릿 내보내기 및 가져오기](./site-templates/exporting-importing-site-templates.md)를 참조하세요.
```

사이트 페이지 및 콘텐츠를 가져오려면 다음 단계를 따르세요.

1. *사이트 메뉴* (![Site Menu](../../images/icon-product-menu.png))을 열고 *게시*확장한 다음 *가져오기*클릭합니다.

1. *추가* 버튼(![Add Button](../../images/icon-add.png))을 클릭하여 새 가져오기 프로세스를 만듭니다.

1. 원하는 LAR 파일을 추가하고 *계속*클릭합니다.

   ![LAR 파일을 추가하고 계속을 클릭합니다.](./exporting-importing-site-pages-and-content/images/03.png)

1. 가져오기 프로세스를 검토하고 구성합니다.

1. 가져오기를 시작하려면 *가져오기* 클릭합니다.

   ![가져오기 프로세스를 구성하고 시작합니다.](./exporting-importing-site-pages-and-content/images/04.png)

## 내보내기 템플릿 생성

사이트 페이지 콘텐츠를 내보낼 때마다 내보내기 프로세스를 수동으로 만드는 대신 내보내기 템플릿을 만들어 내보내기 설정을 저장하고 재사용할 수 있습니다. 내보내기 템플릿을 만들려면 다음 단계를 따르세요.

1. *사이트 메뉴* (![Site Menu](../../images/icon-product-menu.png))을 열고 *게시*확장한 다음 *내보내기*클릭합니다.

1. 애플리케이션 표시줄에서 *동작* 버튼 ![Actions Button](../../images/icon-actions.png) 클릭하고 *템플릿 내보내기*선택합니다.

   ![애플리케이션 표시줄에서 작업 버튼을 클릭하고 템플릿 내보내기를 선택합니다.](./exporting-importing-site-pages-and-content/images/05.png)

1. *추가* 버튼(![Add Button](../../images/icon-add.png))을 클릭합니다.

1. 내보내기 템플릿의 이름을 지정하고 구성합니다.

1. *저장*클릭합니다.

이제 새로운 내보내기 프로세스를 생성할 때 [템플릿을 선택할 수 있습니다](#exporting-site-pages-and-content) .

## 내보내기/가져오기 프로세스 구성

Liferay는 스테이징 특정 기능뿐만 아니라 내보내기/가져오기 프로세스를 구성하기 위한 설정을 제공합니다. 이러한 설정은 스테이징 및 사이트 템플릿 전파에 영향을 미칩니다.

| 환경              | 묘사                                                                                                                                                     |
|:--------------- |:------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 모든 자산 링크 포함     | 선택하면 지정된 날짜 범위 내 사이트의 모든 자산 링크가 포함됩니다. 체크를 해제하면 해당 콘텐츠에 연결된 자산 링크만 포함됩니다.                                                                              |
| 기본적으로 개별 삭제 복제  | 선택하면 기본적으로 콘텐츠 유형에 대해 수행된 삭제 작업이 항상 대상 사이트에 복제됩니다.                                                                                                     |
| 기본으로 권한 퍼블리싱    | 선택하면 내보낸 페이지 및 개별 포틀릿 창에 할당된 권한이 항상 기본적으로 포함됩니다.                                                                                                       |
| 파일 항목 확인        | 선택한 경우 텍스트가 포함된 콘텐츠 유형을 게시하거나 가져오면 Liferay에서 문서 링크에 대한 콘텐츠를 확인하고 유효한 문서를 가리키는지 확인합니다.                                                                  |
| 저널 피드 참조 유효성 검사 | 선택한 경우 텍스트가 포함된 콘텐츠 유형을 게시하거나 가져오면 Liferay가 콘텐츠에서 웹 콘텐츠 피드에 대한 참조를 확인하고 유효한 피드를 가리키는지 확인합니다.                                                           |
| 레이아웃 참조 유효성 검사  | 선택한 경우 텍스트가 있는 콘텐츠 유형을 게시하거나 가져오면 Liferay가 포털의 가상 호스트 중 하나에 대한 링크에 대한 콘텐츠를 확인하고 유효한 Liferay 페이지를 가리키는지 확인합니다. 가상 호스트에 Liferay에 없는 페이지가 있으면 선택을 취소하십시오. |
| 누락된 참조 유효성 검증   | 선택하면 게시 및 가져오기 프로세스로 인해 Liferay가 누락된 참조를 확인합니다. 선택하지 않으면 누락된 참조 유효성 검사를 건너뜁니다. 따라서 기존 대상이 없는 참조를 게시하거나 대상 사이트로 가져올 수 있습니다.                             |
