# 인스턴스 설정

인스턴스 구성 설정은 모양부터 사용자가 처음 로그인할 때 동의하는 사용 약관에 이르기까지 가상 인스턴스에 대한 구성 정보를 정의합니다. 제어판의 **Configuration** &rarr; **Instance Settings** 섹션에서 Instance Configuration 설정에 액세스하고 **PLATFORM** 아래에서 **Instance Configuration** 범주를 선택합니다. 인스턴스 구성에는 네 가지 항목이 있습니다.

| 설정     | 목적                                      |
|:------ |:--------------------------------------- |
| 모양     | 기본 로고와 전체적인 모양과 느낌을 정의합니다.              |
| 교신 정보  | 인스턴스에 대한 물리적 주소, 이메일 주소, 웹사이트 및 전화번호 구성 |
| 일반     | 방문 페이지, 법적 정보 및 기타 세부 정보를 지정합니다.        |
| 사용의 연한 | 인스턴스에 대한 사용자 지정 사용 약관 제공                |

## 모양

모양 구성 항목은 가상 인스턴스의 기본 로고와 전체적인 모양과 느낌을 정의합니다. LOGO 및 LOOK AND FEEL 섹션으로 구성되어 있습니다.

**로고:** 기본 로고를 변경하고 **사이트 관리자가 자신의 로고를 사용하도록 허용하시겠습니까?** 확인란은 기본적으로 활성화되어 사이트 관리자가 사이트를 구성할 때 로고를 업로드할 수 있는지 여부를 지정합니다. 새로운 로고를 구성할 때 공간에 맞는 이미지 파일을 신중하게 선택하세요. 큰 이미지가 내비게이션과 겹칠 수 있습니다.

**LOOK AND FEEL:** 인스턴스 및 제어판의 기본 테마를 설정합니다.

![가상 인스턴스의 모양을 구성합니다.](./instance-configuration/images/01.png)

## 교신 정보

연락처 정보 구성 항목은 가상 인스턴스를 소유한 조직에 연락하는 방법을 지정합니다. 여러 섹션으로 나뉩니다.

**주소:** 기본, 우편, 배송, PO 지정 조직의 상자 등 주소.

**전화번호:** 조직의 팩스, 지역 전화번호 등을 제공하십시오.

**추가 이메일 주소:** 조직과 관련된 추가 이메일 주소를 지정합니다.

**WEBSITES:** 조직의 공용 및/또는 인트라넷 웹사이트를 지정합니다.

개발자는 애플리케이션에서 이 연락처 정보를 쿼리할 수 있습니다. 더하기 및 빼기 버튼을 각각 사용하여 섹션에서 추가 항목을 추가하고 제거할 수 있습니다.

![가상 인스턴스에 대한 연락처 정보를 구성합니다.](./instance-configuration/images/02.png)

## 일반

일반 항목은 가상 인스턴스의 구성 정보, 랜딩 페이지 및 관련 조직의 기본 법적 정보를 지정합니다. 두 개의 섹션이 있습니다.

**기본 구성:** 가상 인스턴스에 대해 다음 정보를 구성합니다.

* 가상 인스턴스 실행을 담당하는 엔터티의 이름을 설정합니다.
* 메일 도메인을 설정합니다.
* 가상 호스트를 설정합니다.
* URL을 정적 리소스를 제공하도록 구성된 CDN으로 설정합니다.

**탐색:** 가상 인스턴스의 홈 페이지와 기본 랜딩 및 로그아웃 페이지를 설정합니다. 이러한 페이지를 설정하려면 도메인을 따르는 페이지의 상대 URL을 사용하십시오. 예를 들어 기본 방문 페이지를 `http://localhost:8080/web/guest/login`로 설정하려면 `/web/guest/login`을 사용합니다. 주소의 일부로 변수 `${liferay:screenName}` 를 사용하여 로그인 시 사용자를 개인 페이지로 리디렉션할 수 있습니다. 예를 들어 사용자 `marvin` 은 자신의 개인 페이지에 대한 기본 URL을 가지고 있습니다.

```bash
http://localhost:8080/user/marvin
```

그가 로그인할 때 그곳으로 이동하는지 확인하려면 기본 랜딩 페이지 필드에 `/user/${liferay:screenName}` 를 입력하십시오. 이러한 URL은 `portal-ext.properties` 파일의 시스템 범위에서 설정할 수도 있습니다.

```properties
default.landing.page.path=
default.logout.page.path=
company.default.home.url=
```

모든 가상 인스턴스는 속성 파일에 지정된 값을 공유합니다. 인스턴스 설정에서 변경한 사항은 속성 파일에 설정된 값을 재정의합니다. 자세한 내용은 [포털 속성 설명서](https://learn.liferay.com/reference/latest/en/dxp/propertiesdoc/portal.properties.html) 을 참조하십시오.

**추가 정보:** 가상 인스턴스 소유자의 법적 이름, ID, 회사 유형, SIC 코드, 종목 기호, 산업 및 산업 유형을 지정합니다.

![가상 인스턴스에 대한 일반 정보를 구성합니다.](./instance-configuration/images/03.png)

## 사용의 연한

사용 약관 페이지는 기본적으로 활성화되어 있으므로 첫 번째 조치 중 하나는 자리 표시자 약관을 비활성화하거나 교체하는 것입니다. 모든 사용자가 이용 약관을 읽도록 요구 사항을 비활성화하거나 귀하의 이용 약관이 포함된 웹 콘텐츠 기사에 대한 그룹 ID 및 기사 ID를 설정할 수 있습니다. 어느 쪽이든 사용 약관 항목에는 사용자에 대한 사용 약관을 구성하는 데 필요한 모든 것이 포함되어 있습니다.

![가상 인스턴스에 대한 사용 약관을 구성합니다.](./instance-configuration/images/04.png)

그룹 ID는 웹 콘텐츠가 포함된 사이트의 ID입니다.

그룹/사이트 ID를 찾으려면

1. **사이트 메뉴로 이동** &rarr; **구성** &rarr; **설정** .

1. 일반 탭에서 사이트 ID 필드를 찾습니다. 그룹 ID 필드에 입력하십시오.

![사이트 설정의 사이트 ID(20125)는 이용약관 구성의 그룹 ID입니다.](./instance-configuration/images/05.png)

웹 콘텐츠 기사의 ID를 찾으려면,

1. **사이트 메뉴로 이동** &rarr; **콘텐츠 & 데이터** &rarr; **웹 콘텐츠** .

1. 사용 약관 항목을 클릭합니다. 해당 ID는 버전 및 워크플로 상태 아래 화면 오른쪽의 속성 패널에 나타납니다.

![웹 콘텐츠 기사 ID(37808)가 편집 화면에 표시됩니다.](./instance-configuration/images/06.png)

구성을 저장합니다. 처음으로 로그인하는 모든 사용자는 사용 약관 문서를 봅니다. 사용자는 자신의 사용자 계정을 등록하려면 이용 약관에 동의해야 합니다.
