# 사용자 인증

사용자 인증 설정은 사용자가 인증할 수 있는 방법, 필요한 다양한 인증 방법, 예약되어 있고 사용자가 등록할 수 없는 화면 이름과 이메일 주소를 정의합니다.

제어판의 **구성** &rarr; **인스턴스 설정** 섹션에서 사용자 인증 설정에 액세스하고 **PLATFORM** 섹션에서 **사용자 인증** 범주를 선택합니다.

사용자 인증에는 다음 두 항목이 포함됩니다.

* 일반
* 보류하는 신임장

![외부 인증 시스템에 대한 일반 인증 동작 및 설정을 구성합니다.](./user-authentication/images/01.png)

## 일반

일반 구성 항목에는 몇 가지 일반 인증 설정이 포함되어 있습니다.

**사용자는 어떻게 인증합니까?** 사용자가 이메일 주소(기본값), 화면 이름 또는 사용자 ID(데이터베이스에서 자동 생성된 숫자 ID--권장하지 않음)로 인증할지 여부를 선택합니다.

**사용자가 자동으로 로그인하도록 허용하시겠습니까?** 자동 로그인을 활성화/비활성화합니다. 활성화된 경우 사용자는 브라우저에 쿠키를 배치하여 사이트가 로그인 정보를 "기억"하도록 하는 상자를 선택할 수 있습니다. 비활성화된 경우 사용자는 항상 수동으로 로그인해야 합니다.

**사용자가 비밀번호 재설정 링크를 요청하도록 허용하시겠습니까?** 잊어버린 비밀번호에 대한 재설정 링크 요청을 활성화/비활성화합니다.

**낯선 사람이 계정을 만들 수 있도록 허용하시겠습니까?** 인증되지 않은 게스트 사용자의 계정 생성을 활성화/비활성화합니다. 인터넷 사이트를 운영하는 경우 방문자가 계정을 만들 수 있도록 활성화된 상태로 두십시오.

**낯선 사람이 회사 이메일 주소로 계정을 만들 수 있도록 허용하시겠습니까?** 사이트를 실행하는 조직의 도메인에서 이메일 주소를 사용하는 사람들에 의한 계정 생성을 활성화/비활성화합니다(인스턴스 설정의 일반 페이지에서 설정됨). 이것은 하나의 Liferay 설치를 사용하여 내부 및 외부 웹 사이트를 모두 호스팅하는 경우에 편리합니다. 모든 내부 ID는 관리자가 생성하지만 외부 사용자는 ID를 직접 등록할 수 있습니다.

**낯선 사람에게 이메일 주소를 확인하도록 요구하시겠습니까?** 체크하면 새로 생성된 계정은 자가 등록이든 관리자 등록이든 확인이 필요합니다. 관리자가 현재 사용자의 이메일을 수동으로 업데이트하려면 새 이메일 계정의 확인도 필요합니다.

기본적으로 모든 설정이 활성화되어 있습니다. 이메일 주소에 의한 사용자 인증은 다음과 같은 이유로 중요한 기본값입니다.

1. 이메일 주소는 소유자에게 고유합니다.

1. 사람들은 이메일 주소를 기억합니다. 한동안 로그인하지 않은 사용자는 화면 이름을 잊어버릴 수 있습니다.

1. 이메일 주소가 인증에 사용되지 않는 경우 사용자는 이메일 주소를 변경한 다음 프로필에서 이메일 주소를 업데이트하는 것을 잊을 수 있습니다. 이 경우 가상 인스턴스에서 보낸 알림이 사용자에게 도달하지 않습니다. 사용자가 로그인할 때 가장 먼저 이메일 주소를 기억하면 사용자가 이메일 주소를 최신 상태로 유지하는 데 도움이 됩니다.

## 보류하는 신임장

Reserved Credentials 구성 항목은 사용자가 사용할 수 없는 화면 이름과 이메일 주소를 지정합니다. 이것은 사용자가 관리 ID처럼 보이거나 이름에 예약어가 포함된 ID를 생성하는 것을 방지합니다.
