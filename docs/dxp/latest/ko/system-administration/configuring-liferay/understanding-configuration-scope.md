# 구성 범위 이해

Liferay DXP의 서비스 및 응용 프로그램에 대한 설정은 시스템, 가상 인스턴스, 사이트 또는 위젯의 네 가지 수준 또는 **범위** 중 하나에 적용됩니다. 시스템 설정에서 만든 설정은 시스템 범위 수준에서 설정됩니다. 시스템 이외의 레이블이 지정된 범위의 경우 시스템 설정 값이 기본값으로 사용됩니다. 즉, 시스템 전체 구성이 재정의되는 사이트를 식별합니다. 진정한 시스템 범위 구성(카테고리의 **System Scope** 머리글 아래에 있는 구성)은 어디에서도 재정의되지 않습니다.

## 범위 정의

**시스템 범위:** 시스템 범위 항목에 대한 구성은 시스템 전체 방식에서 응용 프로그램의 최종 값입니다. 전체 시스템에 영향을 미치며 다른 곳에서는 무시되지 않습니다.

![일부 시스템 설정 항목은 시스템 범위입니다.](./understanding-configuration-scope/images/01.png)

**가상 인스턴스 범위:** 가상 인스턴스 수준의 구성은 인스턴스 설정에서 재정의됩니다.

![일부 시스템 설정은 가상 인스턴스 범위입니다.](./understanding-configuration-scope/images/05.png)

**사이트 범위:** 이 범위의 구성은 각 사이트에서 재정의됩니다.

![일부 시스템 설정은 사이트 범위입니다.](./understanding-configuration-scope//images/03.png)

**위젯 범위:** 이 범위의 구성은 각 위젯 인스턴스에서 재정의됩니다.

![일부 시스템 설정 항목은 위젯 범위입니다.](./understanding-configuration-scope/images/04.png)

## 예: 블로그 댓글 구성 및 재정의

시스템 설정에서 변경된 구성을 다른 범위에서도 구성할 수 있는 경우 시스템 설정 값은 재정의할 수 있는 기본값입니다. 보다 세분화된 범위에서 구성을 변경하면 시스템 수준에서 변경해도 아무런 작업이 수행되지 않습니다.

예를 들어 댓글 허용은 각 블로그 위젯에 대해 구성할 수 있습니다.

1. 제어판 &rarr; 구성 &rarr; 시스템 설정 &rarr; 블로그(콘텐츠 및 데이터 범주)에서 기본 동작을 설정합니다.
1. 위젯 범위 아래의 블로그 항목에서 **주석 사용** 확인란을 비활성화합니다.
1. 사이트의 콘텐츠 & 데이터 &rarr; 블로그 애플리케이션에 블로그 항목을 추가합니다.
1. 페이지로 이동하여 블로그 위젯을 페이지에 추가합니다.
1. 위젯의 옵션 버튼(![Options](../../images/icon-app-options.png))을 클릭하고 **구성** 를 선택합니다.

동일한 Enable Comments 확인란이 표시되며 기본값은 이제 false(선택 취소됨)입니다. 위젯 구성에서 상자를 선택하면 시스템 설정 항목이 무시됩니다. 시스템 설정 구성을 변경해도 더 이상 이 위젯에 영향을 미치지 않습니다.

![위젯 자체에서 위젯 범위 시스템 설정을 재정의합니다.](./understanding-configuration-scope/images/06.png)

## 시스템 설정 및 인스턴스 설정

인스턴스 설정 <!-- link to Instance Settings --> 시스템 설정과 유사하지만 가상 인스턴스 범위에서 응용 프로그램 및 서비스를 구성합니다. 시스템 설정과 마찬가지로 인스턴스 설정의 구성 항목에는 **기본값으로 재설정** 옵션이 있습니다. 인스턴스 설정에서 이렇게 하면 제공한 사용자 지정 구성이 제거되고 시스템 범위의 현재 값(기본값 또는 사용자 지정)이 가상 인스턴스 범위의 기본값이 됩니다.
