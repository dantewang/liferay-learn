# 시스템 설정

제어판 &rarr; 구성 &rarr; 시스템 설정에서 [시스템 범위](./understanding-configuration-scope.md) 구성을 만들 수 있습니다.

시스템 설정 애플리케이션은 플랫폼, 보안, 콘텐츠 및 데이터, 기타의 네 가지 범주 아래에 구성 섹션을 표시합니다. 각 섹션에는 하나 이상의 구성 항목이 포함되어 있습니다.

관심 있는 섹션(예: 콘텐츠 및 데이터)을 찾아서 필요한 구성으로 이동한 다음 해당 범주(예: 워크플로)를 찾습니다.

![시스템 설정 항목은 섹션(예: 콘텐츠 및 데이터) 및 범주(예: 자산)별로 구성됩니다.](./system-settings/images/02.png)

검색하여 시스템 설정 항목을 찾을 수 없는 경우 검색 표시줄을 사용하여 검색합니다.

![시스템 설정 항목을 검색할 수 있습니다.](./system-settings/images/03.png)

```{note}
일부 시스템 범위 구성은 속성 파일과 제어판의 서버 관리 응용 프로그램에서 만들 수 있습니다.
```

Liferay DXP에는 많은 응용 프로그램이 포함되어 있습니다. 응용 프로그램과 해당 서비스는 응용 프로그램 개발자가 정의한 대로 여러 [범위](./understanding-configuration-scope.md)에서 구성할 수 있습니다. 시스템 설정에서 만든 구성은 시스템 범위이거나 다른 범위에서 재정의할 수 있는 시스템 전체 기본 구성을 설정합니다.

구성 옵션의 기능을 모르는 경우 해당 기능에 대한 설명서를 확인하십시오.

## 시스템 구성 편집

수정할 구성을 찾으면

1. 항목에 대한 구성 양식을 엽니다.

1. 변경한 다음 **저장** 을 클릭합니다. 구성 변경 사항이 저장되고 시스템 전체에 적용됩니다.

   ```{important}
   템플릿(예: FreeMarker 템플릿 및 애플리케이션 표시 템플릿)을 사용하여 생성된 콘텐츠는 캐시됩니다. 캐시된 콘텐츠는 캐시가 무효화(지워질) 때까지 구성 변경 사항을 반영하지 않을 수 있습니다. 서버 관리 -> 리소스 탭은 캐시 지우기 옵션을 제공합니다.
   ```

## 시스템 설정 항목 재설정

작업 버튼(![Actions](../../images/icon-actions.png))을 클릭한 다음 **기본값 재설정** 을 클릭하여 시스템 설정을 기본값으로 재설정할 수 있습니다.

![구성 변경 사항을 저장하면 기본값 재설정 및 내보내기 작업을 사용할 수 있습니다.](./system-settings/images/04.png)

## 구성 내보내기 및 배포

시스템 설정은 이식 가능합니다. 다른 설치에서 동일한 구성을 적용하려면 구성을 내보낼 수 있습니다. 단일 구성 항목 또는 변경한 모든 항목. 내보낸 파일은 동일한 버전의 모든 Liferay DXP 설치에 배포할 수 있습니다.

단일 항목의 구성을 내보내려면 작업 버튼(![Actions](../../images/icon-actions.png))을 클릭한 다음 **내보내기** 를 클릭합니다. 구성이 포함된 `.config` 파일이 시스템에 다운로드됩니다.

시스템 설정에서 변경한 모든 구성을 내보내려면 시스템 설정 옵션 버튼(![Options](../../images/icon-options.png))을 클릭한 다음 **모든 설정 내보내기** 를 클릭합니다. 편집한 모든 항목에 대한 `.config` 파일은 ZIP 파일로 다운로드합니다.

대상 시스템에서 이러한 구성을 활성화하려면 `.config` 파일의 압축을 풀고 [`[Liferay_Home]/osgi/configs` 폴더](../../installation-and-upgrades/reference/liferay-home.md)에 넣습니다.

자세한 내용은 [구성 파일 사용](./configuration-files-and-factories/using-configuration-files.md) 을 참조하십시오.
