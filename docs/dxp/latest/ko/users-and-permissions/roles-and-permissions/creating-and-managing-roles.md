# 역할 생성 및 관리

역할과 권한을 [이해하면 사용자가](./understanding-roles-and-permissions.md) DXP 솔루션 내에서 작업할 수 있도록 권한을 부여하는 데 필요한 역할을 생성할 때입니다.

역할 생성 및 관리는 제어판 &rarr; 사용자 &rarr; 역할에서 수행됩니다.

## 역할 만들기

먼저, [은 생성해야 하는 역할의 범위](./understanding-roles-and-permissions.md) 을 결정합니다. 역할은 전역적으로(일반 역할), 특정 사이트(사이트 역할) 또는 조직(조직 역할)으로 범위를 지정할 수 있습니다.

1. 적절한 역할 범위에 대한 탭을 클릭한 다음 **추가**(![Add](../../images/icon-add.png)) 버튼을 클릭합니다.

1. 제목과 설명을 입력합니다. 제목 필드는 필수이지만 설명은 선택 사항입니다.

1. 원하는 경우 키를 입력합니다. 필수 필드이지만 수동으로 지정하지 않으면 제목을 기준으로 자동으로 생성됩니다.

   키는 프로그래밍 방식으로 역할을 참조하는 데 사용할 수 있습니다.

1. **저장** 을 클릭합니다.

![하나의 필수 필드인 직함만 작성하여 역할을 생성합니다.](./creating-and-managing-roles/images/02.png)

이제 역할이 데이터베이스에 있으며 추가 구성을 위한 준비가 되었습니다. 다음으로 [역할 사용자를 할당하거나](./assigning-users-to-roles.md) 또는 [역할 권한을 정의할 수 있습니다](./defining-role-permissions.md).

## 역할 업데이트

역할 필드는 범위 필드를 제외하고 업데이트할 수 있습니다. 역할의 범위를 변경할 수 없습니다. 원하는 범위에서 새 역할을 만들고 기존 역할을 삭제해야 합니다.

역할을 업데이트하려면 작업 메뉴(![Actions](../../images/icon-actions.png))를 클릭하고 **편집** 를 선택합니다. 편집하고 저장할 준비가 된 역할 생성 양식이 다시 표시됩니다.

## 역할 삭제

역할을 삭제하려면 작업 메뉴(![Actions](../../images/icon-actions.png))를 클릭하고 **삭제** 를 선택합니다. 삭제를 확인하면 역할과 연결된 [개의 워크플로 작업 할당](../../process-automation/workflow/using-workflows/reviewing-assets.md) 과 함께 역할이 즉시 삭제됩니다.

![결과를 수락하려면 확인을 클릭하여 역할을 삭제합니다.](./creating-and-managing-roles/images/03.png)

## 역할 관리 권한

역할의 작업 메뉴(![Actions](../../images/icon-actions.png))에 있는 **권한** 항목은 역할을 관리할 수 있는 사람을 정의하기 위한 것입니다. 이 기능은 역할 담당자에게 부여된 권한을 나열하는 [역할 권한 정의](./defining-role-permissions.md)과 다릅니다.

![역할 생성 및 관리를 위해 권한을 구성할 수 있습니다.](./creating-and-managing-roles/images/01.png)

기본적으로 모든 역할은 모든 범위에서 **소유자** 일반 역할 및 **관리자** 일반 역할로 관리할 수 있습니다. 관리자 역할은 권한을 수정할 수 없기 때문에 권한 매트릭스에 표시되지 않습니다. 테스트 설치에서 기본 사용자를 사용하는 경우 전역 범위에서 이 권한을 가지며 모든 작업을 수행할 수 있을 뿐만 아니라 사용자를 할당하고 역할에 대한 권한을 정의할 수 있습니다.

```{warning}
이러한 권한을 부여할 때 주의하십시오. 이는 소수의 신뢰할 수 있는 사용자에게 맡기는 것이 가장 좋은 관리 기능입니다.
```

## 자산 컨테이너를 삭제하면 해당 자산이 삭제됨

웹 콘텐츠 폴더에는 웹 콘텐츠 기사가 포함되어 있습니다. 웹 콘텐츠 폴더는 자산 컨테이너이고 웹 콘텐츠 기사는 자산입니다. 개별 자산을 삭제할 수 있는 역할 권한을 부여하지 않고 자산 컨테이너를 삭제할 수 있는 역할 권한을 부여할 수 있습니다. 이 경우 조심하십시오. 역할 담당자가 개별 자산이 포함된 자산 컨테이너를 삭제하면 개별 자산 자체도 삭제됩니다.

| 자산 컨테이너     | 애셋                           |
|:----------- |:---------------------------- |
| 웹 내용 급식     | 웹 내용                         |
| 기술 자료 폴더    | 기술 자료 문서                     |
| 전보국 종류      | 전보국 화제                       |
| Wiki마디      | Wiki페이지                      |
| 문서 및 미디어 폴더 | Documents and Media Document |
| 양식          | 양식 레코드                       |
| 동적 데이터 목록   | 동적 데이터 목록 레코드                |
| 앱 빌더 개체     | 앱 빌더 앱                       |

웹 콘텐츠 폴더 외에도 자산 컨테이너의 예로는 책갈피 폴더, 게시판 범주, Wiki 노드, 문서 및 미디어 폴더가 있습니다.
