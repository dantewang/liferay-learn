# 사용자 추가 및 관리

핵심 사용자 관리 활동에는 사용자 추가, 편집 및 삭제가 포함됩니다. 이러한 활동은 일반적으로 관리 사용자로 제한됩니다.

## 사용자 추가

1. 제품 메뉴에서 **제어판** &rarr; **사용자** &rarr; **사용자 및 조직** 을 클릭합니다.
1. 사용자 탭에서 **추가** 버튼 ![Add](../../images/icon-add.png)을 클릭하십시오.

   ![제어판의 사용자 및 조직 섹션에서 사용자를 추가합니다.](./adding-and-managing-users/images/01.png)

1. 사용자 추가 양식을 작성하고 **저장** 을 클릭합니다. 최소한 사용자의 화면 이름, 이름, 성 및 이메일 주소를 제공하십시오.

    ```{note}
    화면 이름과 이메일 주소는 서로 바꿔서 사용할 수 없습니다. 화면 이름은 사용자 페이지의 URL에 사용되기 때문에 `@` 기호를 포함할 수 없습니다.

    사용자 추가 기능은 여러 개의 독립적인 형식으로 나뉩니다. 첫 번째 양식을 저장하여 사용자를 생성하면 'Success'라는 성공 메시지가 표시됩니다. 성공적으로 가공되는 너의 요구.`
    ```

사용자를 생성하면 추가 정보를 추가할 수 있습니다.

### 새 사용자의 암호 설정

사용자를 생성하면 Liferay DXP가 암호를 생성합니다. [메일 서버가 설정된 경우](../../installation-and-upgrades/setting-up-liferay/configuring-mail.md), Liferay DXP는 사용자의 새 비밀번호가 포함된 이메일 메시지를 보냅니다.

메일 서버를 설정하지 않은 경우 일반 메뉴에서 **비밀번호** 항목을 클릭하고 새 사용자의 비밀번호를 수동으로 설정하십시오. 새 비밀번호를 두 번 입력합니다.

![사용자의 비밀번호를 수동으로 설정하려면 비밀번호를 두 번 입력합니다. 사용 중인 암호 정책이 이를 허용하도록 구성된 경우 사용자가 포털에 처음 로그인할 때 암호를 재설정하도록 요구할지 여부를 선택합니다.](./adding-and-managing-users/images/03.png)

## 사용자 편집

1. 제어판에서 **사용자 및 조직** 을 클릭합니다.

1. 편집할 사용자 옆에 있는 **작업** 버튼(![Actions](../../images/icon-actions.png))을 클릭합니다.

1. 다음 옵션을 사용할 수 있습니다.

| 옵션          | 묘사                                                               |
|:----------- |:---------------------------------------------------------------- |
| 수정          | 사용자 계정의 모든 측면을 수정합니다.                                            |
| 권한          | 사용자를 편집할 권한이 있는 역할을 정의합니다.                                       |
| 페이지 관리      | 사용자의 개인 페이지를 구성합니다.                                              |
| 구체화되는 사용자   | 마치 해당 사용자인 것처럼 다른 창에서 사이트를 탐색합니다.                                |
| 군대를 해산하십시요  | [사용자 계정 비활성화](#deactivating-users) .                              |
| 개인 데이터 삭제   | [사용자의 개인 데이터 삭제](../managing-user-data/sanitizing-user-data.md). |
| 개인 데이터 내보내기 | [사용자 개인 데이터 다운로드](../managing-user-data/exporting-user-data.md). |

## 사용자 비활성화 및 삭제

사용자 제거는 여러 가지 이유로 2단계 프로세스입니다.

* 귀하 또는 사용자는 귀하가 해당 계정을 정말로 원한다고 결정할 수 있습니다.
* 사용자는 종종 귀하의 국가에서 가능한 [법적 파급 효과](../managing-user-data.md) 콘텐츠에 연결되어 있습니다.
* 해당 사용자의 존재에 대한 모든 증거를 제거하기 전에 해당 사용자의 계정 활동을 검토하는 데 시간이 필요할 수 있습니다.

이러한 이유 때문에 해당 사용자를 **삭제** 하기 전에 먼저 사용자를 **비활성화** 해야 합니다.

### 사용자 비활성화

사용자를 비활성화하면 사용자가 로그인할 수 없지만 사용자는 보존됩니다. 사용자 보기에서 활성 사용자와 비활성 사용자 간에 전환할 수 있습니다. 모든 사용자가 활성 상태이면 이 필터링 옵션이 나타나지 않습니다.

사용자를 비활성화하려면

1. **제어판** &rarr; **사용자** &rarr; **사용자 및 조직** 로 이동하여 비활성화할 사용자를 찾습니다.
1. **User 의 Actions**(![Actions](../../images/icon-actions.png)) 메뉴 를 클릭 하고 **Deactivate 를 선택 합니다** .

이제 사용자가 비활성화되어 더 이상 로그인할 수 없습니다. 사용자 테이블에서 사용자를 찾고(비활성화된 사용자별로 테이블 결과를 필터링하고 있는지 확인) 작업 메뉴를 클릭하고 **활성화** 을 선택하여 사용자를 재활성화할 수 있습니다. 활성화를 위한 확인 창이 없습니다. 활성화를 클릭하면 자동으로 이전 상태로 복원됩니다.

### 사용자 삭제

실수로 사용자를 삭제하는 것을 방지하려면 먼저 사용자를 비활성화한 다음 삭제할 수 있습니다.

1. 사용자를 비활성화합니다.
1. **필터를 클릭하고 테이블 상단에서** 을 주문하면 선택기가 나타납니다. **비활성** 을 클릭하면 방금 비활성화한 사용자를 볼 수 있습니다.

    ![사용자를 삭제하려면 먼저 비활성 또는 비활성화된 사용자를 표시하도록 필터링하십시오.](./adding-and-managing-users/images/05.png)

1. 작업 메뉴를 다시 클릭하고 정말로 사용자를 삭제하려는 경우 **삭제** 을 클릭합니다.

사용자가 삭제됩니다. 이전 백업에서 복원하는 것 외에는 삭제된 사용자를 복구할 방법이 없습니다.

## 사용자 사칭

사용자로 가장하여 사용자가 보는 것처럼 시스템을 볼 수 있습니다. 이것은 사용자가 중요한 데이터에 액세스할 수 없는지 확인하는 것과 같이 관리자가 볼 수 없는 권한 문제를 진단하는 데 도움이 됩니다. 관리자 역할을 가진 사용자만 가장할 수 있습니다.

1. 제어판 &rarr; **사용자 및 조직** 를 클릭하여 사용자 목록을 보고 가장하려는 사용자를 찾습니다.
1. 해당 사용자의 **작업** 버튼(![Actions](../../images/icon-actions.png))을 클릭합니다.
1. **사용자 가장** 을 클릭합니다.

그러면 해당 사용자로 로그인된 다른 브라우저 창이 열립니다.

## 사용자 비밀번호 재설정

사용자 추가 기능에는 암호 양식 하단에 **암호 재설정 필요** 확인란이 포함되어 있습니다. 기본 암호 정책은 관리자가 이 옵션을 선택 취소하는 것을 방지합니다. 그러나 이 상자를 사용할 수 있도록 기본 암호 정책을 수정할 수 있습니다.

1. 제어판 &rarr; 사용자에서 **암호 정책** 로 이동합니다.
1. **기본 암호 정책** 을 클릭합니다.
1. 비밀번호 변경 섹션에서 **변경 필요** 전환기를 선택 취소합니다. 이제 추가한 사용자가 암호를 재설정해야 하는지 여부를 결정할 수 있습니다.

기본 정책을 편집하거나 직접 만드는 방법에 대한 자세한 내용은 [암호 정책](../roles-and-permissions/configuring-a-password-policy.md) 을 참조하십시오.
