# 사용자 데이터 삭제

일반 데이터 보호 규정(GDPR)의 기술적으로 까다로운 요구 사항 중 하나는 _잊혀질 권리_입니다. 여기서의 목적은 이 요구 사항을 자세히 설명하는 것이 아니라 요구 사항을 충족하는 데 도움이 될 수 있는 두 가지 기능인 개인 데이터 삭제 및 데이터 익명화를 보여주는 것입니다.

소프트웨어에 의해 잊혀진다는 것은 관리자조차도 개인 데이터가 삭제되거나 익명화된 사용자의 신원을 알 수 있는 정보를 수집할 수 있는 능력을 제거하는 것입니다.

개념적으로 사용자를 잊는다는 것은 최소한 다음 두 가지를 의미합니다.

- 시스템에서 사용자의 식별 정보를 삭제합니다. Liferay DXP에서는 데이터베이스 테이블과 검색 색인에서 사용자를 제거해야 합니다.
- 사용자가 상호작용한 콘텐츠를 삭제하거나 익명화하여 실제 사람을 추적할 수 없도록 합니다.

```{tip}
`사용자 비활성화 및 삭제만으로 충분하지 않습니까?`

삭제하면 데이터베이스의 사용자 테이블에서 사용자가 제거됩니다. 단, 이용자의 정보는 다른 장소에 보관됩니다. 표준 사용자 삭제 시나리오에서 사용자가 개인적으로 생성한 모든 콘텐츠는 여전히 시스템 식별자(사용자 ID 및 사용자 이름)를 통해 사용자에게 할당되며 UI에서 콘텐츠 옆에 계속 표시됩니다. 이러한 사용자 식별 데이터의 의도치 않은 보존은 GDPR 요구 사항 중 일부를 충족하기에 부적절하며 데이터 삭제 기능이 추가된 주된 이유입니다.
```

## 개인 데이터 삭제 화면

사용자 데이터 삭제와 익명화는 모두 같은 장소, 즉 개인 데이터 삭제 화면에서 이루어집니다.

1. 애플리케이션 메뉴(![애플리케이션 메뉴](../../images/icon-applications-menu.png))를 열고 제어판 &rarr; 사용자 &rarr; 사용자 및 조직으로 이동합니다.

1. 사용자에 대한 작업 버튼(![작업](../../images/icon-actions.png))을 클릭하고 _개인 데이터 삭제_를 선택합니다. 사용자를 비활성화하지 않은 경우 비활성화하라는 메시지가 표시됩니다.

   사용자의 개인 데이터 삭제 화면이 나타납니다.

사용자가 시스템에 게시한 모든 데이터를 찾아볼 수 있습니다. 해당 사이트의 데이터를 찾아보려면 _개인 사이트_를 클릭하세요.

![From here, you can browse all data the User posted on his or her personal Site.](./sanitizing-user-data/images/01.png)

일반 Liferay 사이트에 게시된 데이터를 찾아보려면 _일반 사이트_를 클릭하세요.

![Choose Regular Sites to browse all data posted by the User on administratively-created Sites.](./sanitizing-user-data/images/02.png)

사용자의 데이터를 검토하려면 해당 항목을 클릭하세요. 예를 들어, Pepper는 자신의 개인 사이트에 블로그 항목을 게시한 것 같습니다. 해당 항목을 클릭하면 해당 블로그 항목의 제목이 표시됩니다.

![Pepper's blog entry might need review.](./sanitizing-user-data/images/03.png)

항목을 검토하려면 해당 항목을 클릭하세요. 필요한 콘텐츠를 변경할 수 있는 애플리케이션의 편집 모드(이 경우 블로그)로 이동됩니다.

애플리케이션의 모든 항목을 한 번에 관리(익명화 또는 삭제)하려면,

1. 애플리케이션에 대한 작업 버튼(![작업](../../images/icon-actions.png))을 클릭합니다.

1. 애플리케이션의 모든 항목을 안전하게 삭제할 수 있다고 확신하는 경우 _삭제_를 선택하세요.

1. 애플리케이션의 모든 항목에 대해 간단한 익명화가 충분하다고 확신하는 경우 _익명화_를 선택하세요.

인터페이스를 사용하여 사이트, 애플리케이션 및 데이터를 탐색합니다.

## 사용자 삭제

모든 데이터가 적절하게 검토, 삭제, 편집 및/또는 익명화되면 사용자를 삭제하십시오. 개인 데이터 삭제 화면이 완료되면 대화 상자가 자동으로 나타납니다. 이 단계는 간단합니다. _확인_을 클릭하세요.

![To finish the data erasure process, delete the User.](./sanitizing-user-data/images/04.png)

이제 사용자의 데이터는 익명화되거나 삭제되며 사용자도 삭제됩니다.

```{warning}
사용자 관리 도구는 사용자 생성 콘텐츠를 익명화하거나 삭제하는 데 도움을 줍니다. 그러나 사이트에 영향을 미칠 수 있는 일부 종속성은 삭제 후에도 계속 유지될 수 있습니다. 예를 들어 사이트 페이지를 만든 사용자를 삭제하면 일부 사이트 오류가 발생할 수 있습니다. 이러한 오류를 수정하려면 기존 사용자가 사이트 콘텐츠를 다시 만들어야 할 수도 있습니다. 사용자 익명화는 사용자 삭제의 대체 옵션입니다.
```

## 수동 익명화

사용자의 식별 필드만 익명화하고 사용자를 삭제하는 것만으로는 충분하지 않은 경우가 많습니다. 예를 들어 John Smith라는 사용자가 개인 식별 정보를 양식에 입력하는 경우 텍스트 입력 필드는 UAD 프레임워크에 의해 익명화되지 않습니다.

![A user inputs identifiable information into an input text field.](./sanitizing-user-data/images/05.png)

익명화 후 _작성자_ 필드는 삭제되었지만 사용자 입력 텍스트는 그대로 남아 있으므로 수동으로 제거해야 합니다.

![User input text is not anonymized.](./sanitizing-user-data/images/06.png)

이와 같은 사용자 생성 콘텐츠는 개인 식별 정보를 제거하기 위해 수동으로 식별하고 편집해야 합니다.
