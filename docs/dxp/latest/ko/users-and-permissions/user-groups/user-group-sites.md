# 사용자 그룹 사이트

사용자 그룹 사이트를 사용하면 각 구성원의 [개인 사이트](../../site-building/sites/personal-sites.md)에 자동으로 추가되는 페이지를 만들 수 있습니다. 이 페이지는 개인 사이트와 마찬가지로 두 개의 [페이지 세트](../../site-building/creating-pages/understanding-pages/understanding-pages.md#page-sets) , **내 프로필**(공용) 및 **내 대시보드**(비공개)로 구성됩니다. 사용자가 여러 사용자 그룹에 속한 경우 해당 그룹의 모든 페이지도 해당 사이트에 추가됩니다.

![각 구성원의 개인 사이트에 추가되는 프로필 및 대시보드 페이지를 만듭니다.](./user-group-sites/images/01.png)

기본적으로 상속된 페이지는 사용자 그룹 사이트에서만 편집할 수 있으며 개인 사이트에서는 편집할 수 없습니다. 필요한 경우 사용자 그룹 관리자는 사용자가 페이지에서 위젯을 추가하고 구성할 수 있도록 사용자 정의 가능한 페이지 영역을 정의할 수 있습니다.

사용자 그룹 사이트는 [수동으로](#designing-user-group-sites-manually) 또는 [사이트 템플릿](#using-site-templates) 을 사용하여 설계할 수 있습니다.

```{important}
Liferay 7.4 U22+ 및 GA22+의 경우 새 Liferay 설치에는 내 대시보드 페이지를 포함하여 개인 페이지 생성을 비활성화하는 릴리스 기능 플래그가 포함됩니다. 사용자 그룹에 대한 사용자 정의 대시보드 페이지를 수동으로 생성하려면 Liferay 시스템에 대해 개인 페이지를 활성화해야 합니다. 자세한 내용은 [비공개 페이지 활성화](../../site-building/creating-pages/understanding-pages/understanding-pages.md#enabling-private-pages) 를 참조하세요.

또는 비공개 페이지가 비활성화된 경우 사이트 템플릿을 사용하여 내 대시보드 페이지를 추가할 수 있습니다. 템플릿에 추가되고 내 대시보드 페이지 집합에 적용된 페이지는 비공개 페이지로 취급됩니다.
```

## 사용자 그룹 사이트 수동 설계

사용자 그룹 사이트를 수동으로 디자인하려면 다음 단계를 따르십시오.

1. **글로벌 메뉴**(![Global Menu](../../images/icon-applications-menu.png))을 열고 **제어판** 으로 이동한 다음 **사용자 그룹** 을 클릭합니다.

1. 원하는 그룹에 대해 **작업** 버튼(![Actions Button](../../images/icon-actions.png))을 클릭하고 **페이지** 를 선택합니다.

1. [사이트 페이지 추가 및 디자인](../../site-building/creating-pages/adding-pages/adding-a-page-to-a-site.md).

   ```{note}
   비공개 페이지가 비활성화된 상태에서 이렇게 추가된 모든 페이지는 *내 프로필* 페이지 집합에 추가됩니다. *내 대시보드* 세트에 페이지를 추가하려면 개인 페이지를 활성화하고 생성해야 합니다.
   ```

1. (선택 사항) 추가 사이트 응용 프로그램 및 구성 옵션에 액세스하려면 **사이트 메뉴**(![Site Menu](../../images/icon-product-menu.png))을 클릭합니다.

페이지가 사용자 그룹 사이트에 추가되면 **사용자 그룹** 애플리케이션을 통해 페이지를 볼 수 있습니다. 원하는 그룹의 **작업** 버튼(![Actions Button](../../images/icon-actions.png))을 클릭하고 **프로필 페이지로 이동** 또는 **대시보드 페이지로 이동** 을 선택합니다.

## 사이트 템플릿 사용

[사이트 템플릿](../../site-building/sites/site-templates.md)을 사용하여 프로필 및 대시보드 페이지로 사용자 그룹 사이트를 채우려면 다음 단계를 따르십시오.

1. **글로벌 메뉴**(![Global Menu](../../images/icon-applications-menu.png))을 열고 **제어판** 으로 이동한 다음 **사용자 그룹** 을 클릭합니다.

1. 원하는 그룹에 대해 **작업** 버튼(![Actions Button](../../images/icon-actions.png))을 클릭하고 **편집을 선택** .

1. **내 프로필** 및 **내 대시보드** 드롭다운 메뉴를 사용하여 각 페이지 세트에 대한 사이트 템플릿을 선택합니다.

   ![내 프로필 및 내 대시보드 페이지 세트에 대한 사이트 템플릿을 선택합니다.](./user-group-sites/images/02.png)

1. 사이트 템플릿에서 변경 사항 전파를 활성화할지 여부를 결정합니다.

   ```{note}
   활성화되면 템플릿의 변경 사항이 사용자 그룹 사이트에 자동으로 전파됩니다. 이 옵션을 비활성화했다가 나중에 다시 활성화하면 템플릿의 페이지가 사용자 사이트에 복사되어 모든 변경 사항을 덮어씁니다.
   ```

1. **저장** 을 클릭합니다.

일단 저장되면 템플릿의 페이지가 **내 프로필** 또는 **내 대시보드** 페이지 세트 아래에 있는 각 그룹 구성원의 개인 사이트에 추가됩니다.

## 레거시 사용자 그룹 사이트 동작

수백만 명의 사용자가 있는 경우에도 사용자 그룹 사이트 페이지의 상속에 대한 성능에는 영향이 없습니다. 그러나 7.0 이전의 Liferay Portal 및 Liferay DXP 버전에서는 필수 사용자 그룹 페이지를 각 사용자의 개인 사이트에 복사해야 합니다.

Liferay DXP 7.2를 사용 중이고 해당 동작을 유지해야 하는 경우 `Portal-ext.properties` 파일에 다음 줄을 추가하여 활성화합니다.

```
user.groups.copy.layouts.to.user.personal.site=true
```

이 속성은 Liferay DXP 7.3에서 제거되었습니다.

이 속성이 `true`로 설정되면 템플릿 페이지가 사용자의 개인 사이트에 한 번 복사된 다음 사용자가 수정할 수 있습니다. 즉, 나중에 템플릿 페이지를 변경하면 변경 후 사용자 그룹에 추가된 사용자에게만 영향을 미칩니다. 개인 사이트에 대한 관리 권한이 있는 사용자는 템플릿에 대해 **사이트 관리자가 이 사이트 템플릿과 연결된 페이지를 수정하도록 허용** 상자가 선택된 경우 페이지와 해당 콘텐츠를 수정할 수 있습니다. 사용자가 사용자 그룹에서 제거되면 관련 페이지가 사용자의 개인 사이트에서 제거됩니다. 사용자가 그룹에서 제거된 후 다시 추가되면 그룹의 템플릿 페이지가 사용자 사이트에 두 번째로 복사됩니다. 사용자 그룹의 사이트가 사이트 템플릿을 기반으로 하고 사용자가 그룹에 이미 추가된 후 관리자가 사용자 그룹의 사이트 템플릿을 수정하는 경우 해당 변경 사항은 **사이트 템플릿** 에서 변경 전파 활성화 상자가 선택된 경우에만 적용됩니다. 사용자 그룹에 대한 항목이 선택되었습니다.
