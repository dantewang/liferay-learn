# 詳細設定リファレンス

{bdg-secondary}`利用可能な7.4 DXP U23+または7.4 GA23+`

ページフラグメントの「詳細設定」タブには、より複雑な設定や技術的な設定を行うためのフィールドが用意されています。 例えば、高度な設定により、ウェブブラウザがフラグメントのHTMLを処理する方法を変更することができます。

フラグメントの詳細設定メニューにアクセスするには、次の手順に従います。

1. フラグメントに対応したページやテンプレートの編集を開始します。

1. サイドバーメニューの **ブラウザ** パネル（![Browser](../../../../../images/icon-browser.png)）を開き、［Page Elements］の下で目的の **フラグメント** を選択します。

   また、ページやテンプレート内の目的のフラグメントをクリックしても、このメニューが表示されます。

1. ページ要素サイドメニューの「**Advanced**」タブをクリックします。

![編集サイドバーメニューの「ブラウザ」パネルから、フラグメントの詳細設定にアクセスします。](./advanced-settings-reference/images/01.png)

## HTML Tag (Automatic Copy)

ウェブスクレイパー、ブラウザ、サイトレンダラーが、ページにあるコンテンツの種類をより簡単に検索して取得できるようにするためのHTMLタグがいくつか存在します。 **コンテナ** ページフラグメントには、ページ上のコンテナフラグメントのHTML要素にどのタグを生成するかを選択するオプションが用意されています。

| 項目                        | Description                                                                                                                                  |
|:------------------------- |:-------------------------------------------------------------------------------------------------------------------------------------------- |
| HTML Tag (Automatic Copy) | フラグメント用に生成されたHTMLタグを選択します（コンテナフラグメントのみ）。 利用可能なオプションは、 `div` （デフォルト）、 `header`, `nav`, `section`, `article`, `main`, `aside`, and `footer`です。 |

```{note}
選択されたタグは、コンテナフラグメント自体にのみ適用され、その中に入れ子になっている他のフラグメントやコンテンツには適用されない。
```

![コンテナフラグメントにどのHTMLタグを使用するかは、「詳細設定」タブで選択することができます。](./advanced-settings-reference/images/02.png)

## カスタム CSS

{bdg-secondary}`利用可能 7.4 DXP U34+ または 7.4 GA34+`

フラグメントにカスタムCSSを追加するには、「詳細設定」タブの「CSS」の「カスタムCSS」テキストボックスで行います。 このフィールドを使用するには、フラグメントのUPDATE権限が必要です。

デフォルトでは、カスタムCSSのテキストボックスには、このプレースホルダーCSSが含まれています。

```css
.[$FRAGMENT_CLASS$] {

}
```

`[$FRAGMENT_CLASS$]` は、設定するフラグメントの一意のクラス名を表す CSS クラスのトークンです。 そのフラグメントにのみ CSS を適用する場合に使用する。

例えば、このカスタムCSSでは、フラグメントの背景を通常は赤に、ユーザーがカーソルを合わせたときは青に設定します。

```css
.[$FRAGMENT_CLASS$] {
    background: red;
}

.[$FRAGMENT_CLASS$]:hover {
    background: blue
}
```

ページがレンダリングされるとき、 `[$FRAGMENT_CLASS$]` はバックエンドでフラグメントの実際のクラス名に置き換えられます ( `lfr-layout-structure-item-01d752e4-32e3-ed4a-9cd6-fcaf483d0c34`のように)。

```{tip}
テキストボックスの上にある*拡張*アイコンをクリックすると、画面にモーダルが表示され、フラグメントのカスタムCSSを編集するためのスペースが確保されます。
```

また、現在使用している [スタイルブック](../../../../site-appearance/style-books/using-a-style-book-to-standardize-site-appearance.md) に由来する変数をカスタム CSS で使用することができます。 この例では、スタイルブックのInfoカラー（ダークブルーと定義）を使用して、フラグメントの背景色を設定しています。

![スタイルブックの変数を使用して、特定のフラグメントのCSSを微調整することができます。](./advanced-settings-reference/images/03.png)

### カスタムCSSクラス

上記のテキストボックス（**CSS Classes**）を使用して、カスタムCSSで使用するクラスを追加します（スペース、カンマ、または改行で区切ります）。 このフィールドに追加したクラスは、同じページ上の他のフラグメントに再利用することができます。 同じページの他のフラグメントに追加したクラスのサジェストが、入力中に表示されます。

![このフィールドにカスタムCSSクラスを入力すると、同じページ上の他のフラグメントの詳細設定に候補として表示されます。](./advanced-settings-reference/images/04.png)

```{note}
CSSクラス]テキストボックスには、自分で追加したCSSクラスだけが候補として表示されます。 既存のクラスや変数（`[$FRAGMENT_CLASS$]`など）は表示されないので、このフィールドに追加する必要はありません。
```

## 検索結果からのコンテンツ非表示

サイト内の検索結果からページ断片を隠したい場合があります。 この設定は、特に、あるフラグメントが多くのページで共通して再利用されるなど、コンテンツ検索に関連性がない場合、無関係な検索結果が繰り返されることを防ぐのに役立ちます。

![無関係なフラグメントやよく再利用されるフラグメントを非表示にして、サイトの検索結果を改善します。](./advanced-settings-reference/images/05.png)
