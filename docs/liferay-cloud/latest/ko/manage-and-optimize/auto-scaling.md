# 자동 확장

Liferay Cloud의 자동 확장 기능은 성능을 최적화하기 위해 필요에 따라 Liferay 서비스 인스턴스를 자동으로 생성 및 소멸합니다. 이렇게 하면 서버 트래픽 증가, 메모리 누수 또는 기타 문제와 같은 갑작스러운 변경 사항을 해결할 수 있습니다. 기본적으로 이 기능은 모든 Liferay 클라우드 계정에서 *비활성화* 되어 있습니다.

이 기능을 사용하면 서비스에서 Liferay DXP 인스턴스 수를 [정의된 최대](#setting-the-maximum-number-of-instances) (기본값은 10개)로 자동 증가(업스케일링)하거나 [`LCP.json`](../reference/configuration-via-lcp-json.md)의 `scale` 속성에서 지정된 수로 감소(다운스케일링)할 수 있습니다. `scale` 속성은 실행할 최소 인스턴스 수를 지정합니다:

```json
  "scale": 2,
```

```{note}
자동 스케일링은 프로덕션 환경의 Liferay DXP 서비스에서만 사용할 수 있습니다. 자동 확장이 활성화되면 서비스의 각 추가 인스턴스에는 시간당 요금이 부과됩니다. 자동 스케일링이 청구되는 방식에 대한 자세한 내용은 [자동 스케일링이 청구되는 방식](https://help.liferay.com/hc/en-us/articles/360030843592-How-Auto-Scaling-is-charged-)을 참조하세요.
```

## JVM 메모리 구성

자동 스케일링이 제대로 작동하려면 Liferay 이미지의 JVM에 적절한 메모리 할당을 설정하는 것이 중요합니다. 이 할당은 서버의 부하에 따라 메모리 사용량을 확장하거나 축소할 수 있도록 하는 데 필요합니다.

`-Xms` 및 `-Xmx` 플래그를 사용하여 메모리를 할당하도록 `liferay` 서비스의 `LIFERAY_JVM_OPTS` 환경 변수를 설정합니다. `-Xms` 플래그는 서비스가 시작될 때 초기 메모리 할당을 설정하고, `-Xmx` 플래그는 JVM의 최대 메모리 할당을 결정합니다. 예를 들어 이 서비스에 대해 총 16GB가 프로비저닝되어 있고 사용 가능한 경우 속성을 다음과 같이 설정할 수 있습니다:

```
-Xms4096m -Xmx12288m
```

권장 구성은 사용 가능한 메모리의 25%를 사용하여 `-Xms` 플래그를 설정하고, 사용 가능한 메모리의 75%를 사용하여 `-Xmx` 플래그를 설정하는 것입니다. 다음 표를 참조하여 `liferay` 서비스에서 사용할 수 있는 다양한 수준의 메모리에 대한 권장 값을 확인하세요:

| **사용 가능한 메모리** | **권장 LIFERAY_JAVA_OPTS** |
|:-------------- |:-------------------------- |
| 8GB            | -Xms2048m -Xmx6144m        |
| 16GB           | -Xms4096m -Xmx12288m       |
| 24GB           | -Xms6144m -Xmx18432m       |
| 32GB           | -Xms8192m -Xmx24576m       |
| 64GB           | -Xms16384m -Xmx49152m      |

```{note}
LIFERAY_JAVA_OPTS` 변수는 때때로 `-Xms` 및 `-Xmx` 외에 다른 플래그와 함께 사용될 수 있습니다. 다른 플래그가 있는 경우 다른 플래그는 제거하지 않고 메모리 인수를 사용하여 환경 변수를 업데이트합니다.
```

[환경 변수 정의](../reference/defining-environment-variables.md) 를 참조하여 `liferay` 서비스에 이 환경 변수를 추가하는 데 도움을 받으세요.

## 자동 확장 관리

Liferay Cloud 콘솔에서 자동 스케일링을 활성화 또는 비활성화하려면 다음 단계를 따르세요:

1. 프로덕션 환경으로 이동합니다.
1. *서비스로 이동* &rarr; *Liferay* &rarr; *규모*.
1. 자동 스케일링이 비활성화된 경우 *자동 스케일링 활성화* 를 클릭하여 활성화합니다. 자동 스케일링이 이미 활성화되어 있는 경우 *자동 스케일링 비활성화* 를 클릭하여 비활성화하세요.

자동 확장 기능을 활성화하면 Liferay Cloud가 서비스를 모니터링하고 사전 정의된 임계값에 따라 자동으로 확장합니다.

## 목표 평균 사용률 지정

시스템 관리자는 *목표 평균 사용률*을 지정할 수 있습니다. 이 값은 Liferay DXP 서비스 전반의 메모리 및 CPU 사용량의 평균입니다. 자동 스케일링이 트리거되기 전에 해당 값 임계값을 넘어야 합니다.

예를 들어 세 개의 서비스 인스턴스가 각각 70%, 90%, 95%의 메모리를 사용하는 경우 평균 메모리 사용률은 85%입니다. 목표 평균 사용률이 90으로 설정된 경우 업스케일링이 필요하지 않으며, 이 상황에서는 평균 메모리 사용률이 목표를 초과할 때만 업스케일링이 발생합니다.

사용 가능한 총 메모리는 `memory` 속성에 의해 지정됩니다. `LCP.json`, 참조: [LCP.json을 통한 구성](../reference/configuration-via-lcp-json.md).

서비스의 `자동 스케일` 속성( `LCP.json`)에서 목표 평균 사용률을 지정합니다:

```json
"autoscale": {
    "cpu": 80,
    "memory": 90
}
```

`autoscale` 속성이 설정되어 있지 않으면 CPU 및 메모리 사용률 모두에 대해 목표 평균 사용률이 기본적으로 80으로 설정됩니다.

## 최대 인스턴스 수 설정

기본적으로 자동 확장을 사용하면 `liferay` 서비스의 인스턴스 수를 최대 10개까지 늘릴 수 있습니다. 그러나 필요한 경우 이 기본값을 재정의하여 더 많은 인스턴스를 사용할 수 있습니다. 서비스에서 기본 10개 이상의 인스턴스를 사용할 수 있도록 하려면 두 곳에서 기본값을 재정의해야 합니다:

1. [웹 서버 서비스](../platform-services/web-server-service.md) 에서 `LCP_HAPROXY_SERVER_TEMPLATE_BACKEND_NUM` [환경 변수](../reference/defining-environment-variables.md) 를 필요한 가장 높은 값으로 설정합니다. `liferay` 서비스는 `LCP_HAPROXY_SERVER_TEMPLATE_BACKEND_NUM` (기본값 10)에 정의된 최대 인스턴스 수를 초과하여 확장할 수 없습니다.

1. `liferay` 서비스의 `LCP.json` 파일에서 기본값인 10개보다 많은 인스턴스가 필요한 경우 원하는 최대 인스턴스를 지정합니다. [`자동 스케일` 객체](#specifying-target-average-utilization)내에서 `maxInstances` 필드를 설정합니다:

    ```json
    "autoscale": {
        "cpu": 80,
        "memory": 80,
        "maxInstances": 15
    }
    ```

이 두 가지 구성을 모두 업데이트하면 자동 스케일링이 필요에 따라 새로 정의한 최대값까지 `liferay` 서비스의 인스턴스를 증가시킵니다.

## 자동 스케일링 및 DXP 활성화 키

일반적으로 온프레미스에서 Liferay DXP를 사용하려면 활성화 키를 배포하고 유효성을 검사해야 합니다. 자동 확장이 활성화된 Liferay Cloud에서 Liferay Cloud 팀은 모든 DXP 활성화 키 문제를 해결하고 필요에 따라 활성화 키를 추가 및 제거합니다.
