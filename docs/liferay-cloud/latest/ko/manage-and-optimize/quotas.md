# 할당량

할당량은 지정된 Liferay Cloud 프로젝트에 사용할 수 있는 최대 리소스 수입니다. 관리자는 프로젝트 필요에 따라 리소스를 사용하도록 서비스 및 환경을 구성할 수 있습니다. (예를 들어 프로덕션 환경에는 추가 CPU 코어가 할당될 수 있지만 개발 환경에는 더 적은 코어가 할당될 수 있습니다.)

프로젝트가 리소스 할당량에 도달하면 _프로덕션 환경이 영향을 받지 않는다는 점에 유의하는 것이 중요합니다_. 그러나 어떤 리소스가 사전 결정된 할당량에 도달했는지에 따라 일시적인 ** 일부 기능 손실이 발생합니다. 예를 들어 20개의 사용자 지정 도메인에 대한 할당량에 도달하면 관리자가 새 도메인을 만들 수 없습니다. 메모리 할당량에 도달하면 메모리 부족 오류가 발생하고 배포가 실패할 수 있습니다. [실시간 경고](../manage-and-optimize/real-time-alerts.md) 은 사용자가 리소스 할당량에 도달하기 전에 알림을 받도록 구성할 수 있습니다.

[자동 크기 조정](./auto-scaling.md) 이 활성화되면 특정 임계값에 도달해도 미리 결정된 할당량에 포함되지 않습니다.

다음 리소스에는 할당량이 적용됩니다.

* CPU
* 기억
* 맞춤 도메인
* 공동작업자
* 최대 코어 수
* 경우
* 최대 프로젝트 수
* 최대 서비스 수
* 일일 최대 빌드
* 서비스당 CPU
* 서비스당 규모
* 서비스당 메모리

위의 리소스 할당은 각 서비스의 `lcp.json` 파일에서 구성할 수 있습니다. [LCP.json](../reference/configuration-via-lcp-json.md)을 통한 구성을 참조하십시오.

모든 유형의 리소스 할당량 증가를 요청하려면 [헬프 센터 티켓](https://liferay-support.zendesk.com/agent/) 을 여십시오.
