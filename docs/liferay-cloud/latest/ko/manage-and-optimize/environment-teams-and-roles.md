# 환경 팀 및 역할

관리자는 Liferay Cloud Console을 사용하여 각 프로젝트 환경에서 팀 구성원과 역할을 관리할 수 있습니다. 각 환경에는 고유한 구성원이 있을 수 있으며 각 구성원은 서로 다른 환경에서 서로 다른 역할을 가질 수 있습니다.

구성원을 초대하고 다양한 액세스 수준으로 역할을 할당하여 각 프로젝트 환경에서 원하는 팀 구조를 달성하세요.

## 팀원 초대

사용자를 환경에 초대하려면 다음 단계를 따르십시오.

1. 프로젝트 환경으로 이동합니다.

1. 환경 메뉴에서 **팀** 을 클릭합니다.

1. **이메일** 필드에 사용자의 이메일을 입력합니다.

1. 드롭다운 **역할** 메뉴를 클릭하여 새 사용자에게 역할을 할당합니다.

1. **초대 보내기** 을 클릭하십시오.

![그림 1: 팀 페이지에서 새 팀 구성원을 환경에 초대합니다.](./environment-teams-and-roles/images/01.png)

이메일 초대는 사용자에게 단일 프로젝트 환경에 대한 액세스 권한을 부여합니다. 초대된 사용자는 초대 이메일을 열고 **초대 수락** 을 클릭해야 합니다.

![그림 2: Accept Invite를 클릭하여 환경에 대한 액세스 권한을 얻습니다.](./environment-teams-and-roles/images/02.png)

신규 Liferay Cloud 사용자는 계정을 생성해야 하며, 복귀 사용자는 Liferay Cloud 자격 증명을 사용하여 **로그인** 할 수 있습니다.

```{note}
모든 사용자는 팀 페이지에 액세스하여 환경의 다른 사용자와 해당 역할(관리자 포함)을 볼 수 있습니다.
```

## 팀원 관리

현재 및 초대된 팀 구성원은 **구성원** 섹션의 별도 탭에 나타납니다. 여기에서 관리자는 각 탭의 **작업** 버튼을 통해 팀원을 관리할 수 있습니다.

**Invited** 탭에서 관리자는 보류 중인 환경 초대를 취소할 수 있습니다.

   ![그림 3: 작업 버튼을 사용하여 초대됨 탭에서 보류 중인 초대를 취소합니다.](./environment-teams-and-roles/images/03.png)

**현재** 탭에서 관리자는 팀 구성원 역할을 변경하거나 환경에서 구성원을 제거할 수 있습니다.

   ![그림 4: 작업 버튼을 사용하여 현재 팀 구성원을 관리합니다.](./environment-teams-and-roles/images/04.png)

## 팀 역할 이해

역할은 환경 내에서 사용자의 액세스 수준을 결정합니다. Liferay Cloud에는 네 가지 기본 사용자 역할이 있습니다.

* **소유자:** 프로젝트의 단독 소유자가 가장 완전한 액세스 권한을 가집니다.
* **관리자:** 관리자는 Liferay Cloud 환경과 해당 구성원을 완전히 제어할 수 있습니다.
* **기여자:** 기여자는 애플리케이션 관리 및 대부분의 개발 수명 주기를 처리할 수 있지만 팀 구성원을 관리하거나 기타 관리자 전용 작업을 수행할 수는 없습니다.
* **게스트:** 게스트는 보기 전용 액세스 권한이 있습니다. 게스트는 환경에서 일어나는 일을 볼 수 있지만 작업을 수행하거나 변경할 수는 없습니다. 그들은 환경에서 자신을 제거할 수 있는 권한만 가지고 있습니다.
