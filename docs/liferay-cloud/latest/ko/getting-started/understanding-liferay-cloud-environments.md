# Liferay 클라우드 환경 이해

Liferay Cloud 프로젝트는 각각 다른 목적(예: 개발, 스테이징, 생산 등)을 위한 여러 환경을 가질 수 있습니다. 각 환경에는 프로젝트 ID에 환경 ID를 더한 고유한 이름이 있습니다. 예를 들어 **acme** 이라는 프로젝트의 스테이징 및 프로덕션 환경은 각각 **acme-uat** 및 **acme-prd** 로 이름을 지정할 수 있습니다.

```{warning}
환경 이름과 사이트를 선택한 후에는 자동으로 이름을 바꾸거나 다른 사이트로 변경할 수 없습니다. 이러한 작업을 수행해야 하는 경우 지원팀에 문의하세요.
```

이러한 명명 연결에도 불구하고 각 환경은 독립적입니다. 동일한 환경의 서비스가 공용 인터넷에 액세스하지 않고도 안전한 방식으로 통신할 수 있도록 하는 [개인 네트워크](..//configuring-the-cloud-network/private-network.md) 에 의해 분리됩니다.

왼쪽 상단의 선택기 메뉴에서 이러한 환경에 액세스할 수 있습니다.

## 환경 사이트

다른 환경은 다른 지역에 있을 수 있습니다. 예를 들어 유럽에 엔지니어링 팀이 있지만 대부분의 고객은 북미에 있는 회사의 경우 개발 환경은 프랑크푸르트에, 생산 환경은 오레곤에 둘 수 있습니다. 사용자의 요청에 가까운 사이트에 있으면 네트워크 대기 시간이 향상됩니다.

다음은 현재 사용 가능한 [Liferay Cloud 지역 목록](https://help.liferay.com/hc/ko/articles/360019177512) 입니다.

## 환경 유형

환경 유형은 환경 구성을 구분하는 분류입니다. 두 가지 환경 유형이 있습니다.

* 생산
* 비생산

가격과 컴퓨팅 성능이 다른 것 외에도 이러한 환경 유형은 백업 및 데이터베이스 서비스가 작동하는 방식이 다릅니다.

**생산**

* 백업: 데이터 백업은 기본적으로 정기적인 일정에 따라 자동으로 생성됩니다.
* 데이터베이스: 데이터는 여러 가용 영역에 복제되며 향상된 IOPS를 포함합니다.

**비생산**

* 백업: 데이터 백업은 기본적으로 자동으로 생성되지 않습니다.
* 데이터베이스: 데이터는 단일 가용성 영역에 존재하며 일반 IOPS를 포함합니다.
