# 개인 네트워크

모든 환경에는 자체 사설 네트워크가 있습니다. 이를 통해 동일한 환경의 서비스가 공용 인터넷과 상호 작용하지 않고도 여러 보안 통신 프로토콜을 통해 통신할 수 있습니다.

예를 들어 프로젝트는 기본적으로 웹 서버 서비스를 공개 연결에만 노출합니다. 다른 서비스(예: Liferay DXP, 데이터베이스 등) 간의 연결은 사설 네트워크를 통해 라우팅됩니다.

이 개인 네트워크에 구성된 모든 연결에 대해 다음 변수를 지정해야 합니다.

`targetPort`: 노출할 서비스의 내부 포트.

`포트`: 연결할 서비스의 외부 포트.

`프로토콜`: 생성할 연결 유형(TCP 및 UDP 지원).

`외부`: 연결이 외부 연결에 사용 가능한지 여부. 기본값 `false` 은 내부 Liferay Cloud 연결에 대한 연결을 제한합니다.

```{tip}
외부 연결에 대한 연결을 노출하는 경우 서비스의 셸을 사용하여 연결 문제를 해결해야 할 수 있습니다. 자세한 내용은 [쉘 액세스](../../support-and-troubleshooting/troubleshooting/shell-access.md)를 참조하십시오.
```

다음은 구성 예입니다.

```json
{
  "id": "db",
  "ports": [
    {
      "port": 3400,
      "targetPort": 7000,
      "protocol": "TCP"
    },
    {
      "port": 9000,
      "targetPort": 8000,
      "protocol": "TCP",
      "external": true
    }
  ]
}
```

## 환경 변수 참조

| 이름           | 가치   | 묘사                                                                            |
|:------------ |:---- |:----------------------------------------------------------------------------- |
| `항구`         | 3400 | 연결할 서비스의 외부 포트입니다.                                                            |
| `targetPort` | 7000 | 노출할 서비스의 내부 포트                                                                |
| `의정서`        | TCP  | 생성할 연결 유형(TCP 및 UDP가 지원됨)                                                     |
| `external`   | 여실한  | 외부 연결에서 연결을 사용할 수 있는지 여부입니다. 기본값 `false` 은 내부 Liferay Cloud 연결에 대한 연결을 제한합니다. |
