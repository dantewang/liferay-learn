# 3단계: 데이터 백업 업로드 및 복원

호환 가능한 형식의 Liferay DXP 데이터베이스 및 문서 라이브러리를 사용하여 다음 단계는 Liferay Cloud에 단일 백업으로 함께 업로드하고 선택한 환경으로 복원하는 것입니다.

## 백업 업로드

먼저 선택한 환경에서 `백업` 서비스에 데이터를 업로드해야 합니다. 그러면 복원할 수 있는 새 백업이 생성됩니다.

```{warning}
파일 크기와 인터넷 연결 속도에 따라 백업 업로드를 완료하는 데 몇 분 또는 몇 시간이 걸릴 수 있습니다. 업로드가 진행되는 동안 `backup` 서비스는 다른 백업을 생성, 복원 또는 삭제할 수 없습니다. 프로세스가 멈추는 것을 방지하려면 완료될 때까지 *업로드 창을 닫거나 장치를 절전 모드로 두지 마십시오*.
```

백업 페이지를 사용하여 백업을 업로드합니다.

1. Liferay Cloud 콘솔에서 선택한 환경으로 이동하고 왼쪽 메뉴에서 *백업* 을 클릭합니다.

    ![백업 화면으로 이동하여 데이터를 업로드합니다.](./uploading-and-restoring-the-data-backup/images/01.png)

1. 백업 페이지에서 화면 상단의 *백업 업로드...* 을 클릭합니다.

1. 각각의 단추를 클릭하여 이전에 만든 데이터베이스 덤프 및 문서 라이브러리 아카이브 [을 추가](./creating-data-backup-files.md).

    ![버튼을 클릭하여 데이터베이스와 문서 라이브러리를 새 백업으로 업로드합니다.](./uploading-and-restoring-the-data-backup/images/02.png)

    ```{note}
    데이터베이스 덤프 및 문서 라이브러리 업로드는 총 **2TB**를 초과하지 않아야 합니다. 더 큰 파일을 업로드하려면 [Liferay 클라우드 지원팀에 문의](https://help.liferay.com/hc/ko/articles/360030208451-DXP-Cloud-Support-Overview) 하십시오.
    ```

1. *업로드 시작* 을 클릭하고 업로드가 완료될 때까지 기다립니다.

업로드가 완료되면 백업 페이지의 목록 상단에서 데이터와 함께 새 백업을 사용할 수 있습니다.

## 백업 복원

백업 페이지의 업로드에서 생성된 백업을 복원합니다.

1. 올바른 백업을 위해 작업 메뉴에서 *복원 대상...* 을 클릭합니다.

    ![복원 대상 작업을 클릭하여 올바른 백업을 위한 복원 페이지로 이동합니다.](./uploading-and-restoring-the-data-backup/images/03.png)

1. 드롭다운 메뉴에서 백업을 복원할 적절한 환경을 선택합니다.

    ![백업을 복원할 올바른 환경을 선택하십시오.](./uploading-and-restoring-the-data-backup/images/04.png)

    ```{warning}
     올바른 대상 환경을 선택했는지 확인하십시오. 백업을 복원하면 대상 환경의 데이터베이스와 Liferay 데이터 저장소가 백업의 데이터로 덮어쓰여지며 덮어쓰기는 되돌릴 수 없습니다.
    ```

1. *환경* 로 복원을 클릭하고 백업 복원이 완료될 때까지 기다립니다.

    ```{warning}
    백업 복원이 진행되는 동안 대상 환경을 사용할 수 없습니다.
    ```

백업 복원이 완료되면 대상 환경에는 업로드한 데이터베이스 및 문서 라이브러리와 동일한 데이터가 있습니다.

```{tip}
대상 환경의 상단 표시줄에서 *사이트 방문*을 클릭하여 Liferay Cloud에서 Liferay DXP 인스턴스의 상태를 확인할 수 있습니다.
```
