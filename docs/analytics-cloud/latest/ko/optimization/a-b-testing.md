# A/B 테스트

A/B 테스트는 페이지의 여러 버전을 한 번에 테스트하여 콘텐츠 페이지의 효율성을 평가합니다. 이 프로세스에는 페이지 변형 생성, 메트릭(예: 클릭 수)으로 테스트, 가장 효과적인 변형 게시가 포함됩니다. 이러한 테스트는 사이트 사용자를 위한 최상의 경험을 선택하는 데 도움이 될 수 있습니다.

Liferay DXP의 [A/B 테스트](https://learn.liferay.com/dxp/latest/ko/site-building/optimizing-sites/ab-testing/ab-testing.html) 에서 A/B 테스트 생성 및 콘텐츠 페이지 구성에 대해 자세히 알아볼 수 있습니다.

Analytics Cloud는 Liferay DXP에서 실행되는 A/B 테스트의 모든 결과를 추적합니다. 생성된 테스트는 관리할 수 있는 Analytics Cloud와 자동으로 동기화됩니다. 작성, 실행, 종료 및 완료된 A/B 테스트를 보려면 왼쪽 열의 **테스트** 메뉴로 이동하십시오.

![테스트 메뉴에는 사이트에 대해 생성되고 정의된 모든 A/B 테스트가 표시됩니다.](a-b-testing/images/01.png)

초안 A/B 테스트의 경우 다음 세부 정보를 구성할 수 있습니다.

* 대상: 경험 및 사용자 세그먼트.
* 메트릭: 추적할 목표(예: 이탈률 또는 클릭).
* 변형: 사용자가 상호 작용할 수 있는 페이지 변형입니다.
* 트래픽 분할: 변형 간에 임의로 분할된 방문자의 비율입니다.
* 신뢰 수준: 테스트 결과의 정확도.

![이 A/B 테스트가 구성되었으며 실행할 준비가 되었습니다.](a-b-testing/images/02.png)

## 개요

요약 패널은 완료율, 실행 시간(일) 및 총 방문자 세션을 포함하여 테스트 세부 정보의 개요를 제공합니다.

또한 메트릭 세부 정보와 현재 최고 성능 변형에 대한 빠른 요약을 제공합니다.

![요약 패널은 테스트 개요를 제공합니다.](a-b-testing/images/03.png)

## 변형 보고서

대안 보고서 패널은 대안에 대한 자세한 분석과 실적을 제공합니다.

![대안 보고서 패널은 대안에 대한 자세한 분석과 실적을 제공합니다.](a-b-testing/images/04.png)

다음은 각 변형에 대해 보고된 측정항목입니다.

**중앙값:** 샘플 값 집합의 중간 숫자입니다. 이것은 일반적인 사용자의 행동을 추정합니다.

**신뢰 구간:** 모집단의 실제 평균을 포함할 것으로 예상되는 값의 범위. 예를 들어 95% 신뢰 구간은 시스템이 실제 평균을 포함한다고 95% 확신하는 값의 범위입니다. 이는 측정된 목표에 대해 타당해 보이는 가능한 값의 범위를 제공합니다.

**개선:** 대조군의 상대적인 개선. 이 메트릭은 상승도라고도 합니다. 예를 들어 제어 페이지의 유지율이 15%이고 변형 페이지의 유지율이 16%라고 가정합니다. 개선 계산은 `((16 - 15) / 15) = ~6.67%` 개선입니다. 이는 변경의 영향을 보여줍니다. 약간의 개선 사항만 있다면 해당 변경 사항을 구현할 가치가 없을 수 있습니다.

**승리 확률:** 변형이 다른 모든 참여 변형을 능가할 가능성을 예측합니다. 여러 측정항목이 서로 어떻게 비교되는지 보여줍니다. 예를 들어, 경마 이벤트를 생각해 보십시오. 각 말은 경주를 수천 번 시뮬레이션하여 계산한 경주 전에 게시된 우승 기회(즉, 우승 확률)를 가지고 있습니다. 이와 동일한 방법을 변형이 A/B 테스트에서 이길 확률을 계산하는 데 사용합니다.

**고유 방문자 수:** 변형에 기여하는 방문자 수입니다. 변형이 무작위로 할당된 방문자는 테스트가 끝날 때까지 항상 동일한 변형을 봅니다. 페이지에 도달하는 트래픽 양을 아는 것 외에도 이 메트릭은 A/B 테스트의 구성 문제를 확인하는 데 도움이 됩니다. 예를 들어 하나의 변형으로 가는 트래픽이 너무 많을 수 있습니다(일반적으로 잘못된 세그먼트 구성으로 인해 발생).

## 테스트 세션

테스트 세션 패널에는 시간 경과에 따른 일일 테스트 노출수를 보는 세션 수가 표시됩니다. 이를 통해 청중이 A/B 테스트 노출로 연결되고 있는지 확인하는 데 도움이 됩니다. 또한 테스트가 시작되기 전과 비교하여 테스트가 페이지 트래픽에 미치는 영향을 보여줍니다.

![테스트 세션 패널에는 시간 경과에 따른 일일 테스트 노출수를 보는 세션이 표시됩니다.](a-b-testing/images/05.png)

### 테스트 실행 중

이는 테스트가 실행 중이며 원하는 신뢰 수준에 도달하고 승자를 선언하기 전에 더 큰 샘플 크기가 필요함을 의미합니다. 현재 최상의 변형을 볼 수 있습니다.

![현재 최상의 변형을 볼 수 있지만 테스트가 실행 중입니다.](a-b-testing/images/06.png)

테스트가 실행 중일 때 요약 표시줄에서 **테스트** 종료를 선택하여 테스트를 종료할 수 있습니다.

![실행 중인 테스트를 종료하려면 테스트 종료를 클릭합니다.](a-b-testing/images/07.png)

```{important}
A/B 테스트가 성공적으로 실행되려면 많은 양의 트래픽(예: 하루에 수천 건의 조회수)이 예상됩니다. 따라서 공개 사이트는 테스트에 적합합니다. 내부 웹 사이트 또는 포털의 경우 테스트가 완료되는 데 훨씬 더 오래 걸릴 수 있습니다.
```

### 승자 선언

A/B 테스트가 성공적으로 완료되면 변형이 승자로 선언됩니다. 이 상태에서 다음 작업을 수행할 수 있습니다.

* 우승한 변형을 기본 환경으로 게시합니다.
* 변형을 게시하지 않고 테스트를 완료합니다.

![A/B 테스트 완료 후 승자를 게시할 수 있습니다.](a-b-testing/images/08.png)

### 명확한 승자 없음

경우에 따라 Analytics Cloud는 컨트롤 페이지에서 훨씬 뛰어난 성능을 보이는 변형이 없기 때문에 승자를 결정할 수 없습니다. 이 경우 게시하지 않고 테스트를 완료할 수 있으며 컨트롤은 기본 환경으로 유지됩니다.

![변형에 의한 성능 부족은 확실한 승자가 없는 A/B 테스트로 이어질 수 있습니다. ](a-b-testing/images/09.png)
