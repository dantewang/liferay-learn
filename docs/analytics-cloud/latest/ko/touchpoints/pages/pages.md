# 페이지

Analytics Cloud는 페이지에 대해 다음을 수행합니다.

* Analytics Cloud 클라이언트가 페이지 상호 작용을 처음 감지할 때 분석을 위해 페이지를 등록하고 추적합니다.
* 최신 데이터에 대한 일정에 따라 페이지를 쿼리합니다.
* 여기에 설명된 메트릭이 있는 페이지에 대한 보고서입니다.

## 데이터 기간

Analytics Cloud의 모든 페이지 데이터는 지정된 기간 동안 나타납니다. 기간 선택기는 선택한 기간에 대한 메트릭을 다시 계산합니다. 메트릭을 보려면 기간을 지정해야 합니다. 기본값은 30일이지만 구성 가능합니다. 다음 기간을 사용할 수 있습니다.

**지난 24시간:** 지난 24시간 동안 생성된 데이터를 표시합니다.

**지난 7일:** 지난 7일(당일 제외) 동안 생성된 데이터를 일 단위로 표시합니다.

**지난 30일(기본값):** 지난 30일 동안(당일 제외) 생성된 데이터를 주 단위로 표시합니다.

**지난 90일:** 지난 90일(당일 제외) 동안 생성된 데이터를 15일 단위로 표시합니다.

**추가 사전 설정 기간:** 추가 기간 표시: 어제, 지난 28일, 지난 180일, 작년.

**사용자 지정 범위:** 원하는 시작 날짜와 종료 날짜를 선택합니다.

## 방문자 행동

페이지 메트릭의 개요 탭에는 여러 데이터 프레젠테이션이 있습니다. 방문자 행동이라는 첫 번째 차트에는 다음 측정항목이 포함됩니다.

* 고유 방문자
* 전망
* 바운스 비율
* 페이지에 머문 시간
* 입구
* 종료율

```{note}
조회수 측정항목은 즉시 업데이트됩니다. 기타 측정항목은 사용자 세션이 종료된 후(예: 비활성 30분) 또는 하루가 끝날 때(예: 23:59) 업데이트됩니다.
```

표시되는 데이터의 기간을 선택합니다. 이전 기간의 두 번째 비교 라인을 보려면 **이전** 과 비교를 선택합니다.

![선택한 기간 동안 추세를 식별할 수 있습니다.](pages/images/01.png)

그렇다면 방문자 행동 차트에서 이러한 지표는 무엇입니까?

### 고유 방문자

기간 필터와 함께 유용한 페이지 방문자는 지정된 기간 동안 페이지에 액세스한 방문자 수입니다.

고유한 방문자는 Analytics Cloud에서 고유한 IP 주소를 가집니다. 따라서 동일한 사람이 다른 장치에서 페이지를 본 경우 두 명의 순 방문자로 기록됩니다.

### 전망

기간 필터와 함께 유용한 보기는 지정된 기간 동안 페이지의 조회수입니다. 고유한 IP 주소만 세지 않기 때문에 방문자 수와 같지 않습니다. 지난 30일 동안 한 명의 방문자(IP 주소)가 페이지를 100번 다시 방문할 수 있었습니다. 즉, 100개의 페이지 보기가 있지만 방문자는 한 명뿐입니다(매번 동일한 시스템을 사용하여 페이지에 액세스한다고 가정). 그러나 사용자가 페이지와 다시 상호 작용하기 전에 페이지에서 최소 30분 동안 활동이 없으면 단일 사용자에 대한 고유 보기가 기록되지 않습니다.

### 바운스 비율

이탈률은 페이지를 처음 로드한 후 페이지 상호작용(페이지 스크롤 포함) 없이 사이트 밖으로 이동한 페이지 방문자의 비율입니다. 일일 요율(일당 백분율)로 계산되며 선택한 기간 동안 일일 요율 추세선이 표시됩니다.

### 페이지에 머문 시간

페이지에 머문 시간은 매일 모든 보기에 대해 페이지에서 보낸 평균 시간을 계산합니다. 선택한 기간 동안 표시됩니다.

이 메트릭은 24시간마다 다음과 같이 계산됩니다.

```
(view-1-time + view-2-time + ...) / total-number-views
```

### 입구

**입구** 페이지에서 귀하의 사이트에 도착한 방문자 수입니다(다른 페이지에서 시작하여 해당 페이지를 클릭한 방문자와 반대).

### 종료율

종료율은 페이지와 상호작용(예: 스크롤 또는 클릭)한 다음 사이트를 떠난 방문자의 비율입니다.

## 청중

잠재고객 보고서는 차트를 사용하여 잠재고객과 페이지의 상호작용에 대한 정보를 제공합니다. 다음 질문에 답합니다.

* 얼마나 많은 사용자가 내 콘텐츠와 상호 작용하고 있습니까?
* 이 사용자 중 알려지거나 익명인 사용자는 몇 명입니까?
* 내 콘텐츠와 상호작용한 것으로 알려진 사용자 중 Analytics Cloud 세그먼트에 속한 사용자는 몇 명입니까?
* 세그먼트의 사용자 중 상위 15개 세그먼트는 무엇입니까?
* 잠재고객 보고서의 차트는 다음과 같습니다.

**방문자:** 도넛형 차트는 알려져 있거나 익명인 총 고유 방문자의 백분율을 나타냅니다. 차트 중앙에는 총 고유 방문자 수가 표시됩니다. 각 차트 섹션의 도구 설명에는 해당 섹션의 사용자 수(예: 알려진 사용자 또는 익명 사용자 수)가 표시됩니다.

```{note}
로그인 시 이메일 주소가 Analytics Cloud와 동기화된 사용자 데이터와 일치하면 개인이 알려진 것으로 간주됩니다. 신규 사용자가 사이트에 등록하면 해당 데이터가 Analytics Cloud와 동기화되고 알려진 개인으로 표시되기까지 최대 2시간이 걸릴 수 있습니다.
```

**세그먼트 뷰어:** 도넛형 차트는 하나 이상의 Analytics Cloud 세그먼트에 속하거나 속하지 않은 알려진 개인의 백분율을 나타냅니다. 차트 중앙에는 알려진 개인의 총 수가 표시됩니다. 각 차트 섹션의 도구 설명에는 해당 섹션의 사용자 수(예: 하나 이상의 세그먼트에 속한 사용자 수)가 표시됩니다.

**뷰어 세그먼트:** 막대 차트는 각 세그먼트를 구성하는 알려진 개인의 백분율을 보여줍니다. 차트는 상위 5개 세그먼트 각각에 대한 막대를 표시한 다음 나머지 세그먼트를 마지막 막대에 집계합니다. 마지막 막대의 툴팁은 나머지 각 세그먼트의 값을 보여줍니다.

![사이트의 사용자를 신속하게 식별하고 분류합니다.](pages/images/02.png)

## 사이트별 조회수

사이트별 조회수 패널의 지도에는 선택한 기간 동안 국가별 조회수가 표시됩니다.

![사용자 사이트는 사이트 패널 맵을 통해 볼 수 있습니다.](pages/images/03.png)

## 기술별 조회수

기본 탭에서 운영 체제(장치 유형별로 그룹화)별로 페이지 보기의 누적 막대 그래프를 봅니다. 각 막대 위로 마우스를 가져가면 데이터의 세부 분석을 볼 수 있습니다.

![사용자는 운영 체제 및 장치 유형별로 분류됩니다.](pages/images/04.png)

선택한 기간 동안 상위 8개의 웹 브라우저를 표시하는 도넛형 차트를 보려면 웹 브라우저를 클릭합니다. 해당되는 경우 나머지 웹 브라우저는 9번째 도넛 세그먼트에 집계됩니다.

![여러 브라우저 에이전트에 분산된 사용자 수를 알아봅니다.](pages/images/05.png)

이러한 모든 기본 메트릭을 마친 후에는 실제로 페이지를 알게 됩니다. 하지만 본 적이 없는 흥미로운 페이지 데이터가 있습니다. 사람들이 처음에 어떻게 페이지를 방문했는지 알아보세요. 다음으로 [Path Analytics](./paths.md) 에 대해 알아봅니다.
