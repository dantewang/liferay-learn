# 페이지

Analytics Cloud는 페이지에 대해 다음을 수행합니다.

* Analytics Cloud 클라이언트가 페이지 상호 작용을 처음 감지할 때 분석을 위해 페이지를 등록하고 추적합니다.
* 최신 데이터를 일정에 따라 페이지에 쿼리합니다.
* 여기에 설명된 측정항목이 포함된 페이지에 대한 보고서입니다.

특정 페이지에 대한 분석을 보려면,

1. 왼쪽 탐색 메뉴에서 **사이트** 를 클릭하세요. **페이지** 탭을 클릭하세요. 사이트 페이지 목록이 나타납니다.

   이 사이트 페이지 목록을 CSV 파일로 다운로드하려면 **보고서 다운로드** 를 클릭하세요. 자세한 내용은 [보고서 다운로드](../../reference/downloading-reports.md) 참조하세요.

1. 목록에서 페이지를 선택하면 자세한 분석을 볼 수 있습니다.

## 데이터 기간

Analytics Cloud의 모든 페이지 데이터는 지정된 기간 동안 표시됩니다. 기간 선택기는 선택한 기간에 대한 측정항목을 다시 계산합니다. 측정항목을 보려면 기간을 지정해야 합니다. 기본값은 30일이지만 구성 가능합니다. 다음 기간을 사용할 수 있습니다.

**지난 24시간:** 지난 24시간 동안 생성된 데이터를 표시합니다.

**지난 7일:** 이전 7일(당일 제외) 동안 생성된 데이터를 일 단위로 표시합니다.

**지난 30일(기본값):** 지난 30일(당일 제외) 동안 생성된 데이터를 주 단위로 표시합니다.

**지난 90일:** 지난 90일(당일 제외) 동안 생성된 데이터를 15일 단위로 표시합니다.

**추가 사전 설정 기간:** 추가 기간 표시: 어제, 지난 28일, 지난 180일, 작년.

**사용자 정의 범위:** 원하는 시작 날짜와 종료 날짜를 선택합니다.

## 방문자 행동

페이지 측정항목의 개요 탭에는 여러 가지 데이터 프레젠테이션이 있습니다. 방문자 행동이라는 첫 번째 차트에는 다음 측정항목이 포함되어 있습니다.

* 고유 방문자
* 전망
* 바운스 비율
* 페이지에 머문 시간
* 입구
* 종료율

```{note}
조회수 측정항목은 즉시 업데이트됩니다. 다른 측정항목은 사용자 세션이 종료된 후(예: 30분 동안 활동이 없음) 또는 하루가 끝날 때(예: 23:59) 업데이트됩니다.
```

표시되는 데이터의 기간을 선택합니다. 이전 기간의 두 번째 비교 라인을 보려면 **이전과 비교** 를 선택하세요.

![Trends can be identified over selected periods of time.](pages/images/01.png)

이 대시보드 보기를 PDF 파일로 다운로드하려면 **보고서 다운로드** 를 클릭하세요. 자세한 내용은 [보고서 다운로드](../../reference/downloading-reports.md) 참조하세요.

### 고유 방문자

기간 필터와 함께 유용한 페이지 방문자는 특정 기간 동안 페이지에 액세스한 방문자 수입니다.

고유 방문자는 Analytics Cloud에서 고유한 IP 주소를 갖습니다. 따라서 동일한 사람이 다른 장치에서 페이지를 보는 경우 두 명의 고유 방문자로 기록됩니다.

### 전망

기간 필터와 함께 유용한 보기는 특정 기간 동안 페이지에 대한 조회수입니다. 고유한 IP 주소만 계산하려고 하지 않기 때문에 방문자 수와 동일하지 않습니다. 지난 30일 동안 한 명의 방문자(IP 주소)가 100번 페이지를 다시 방문할 수 있었습니다. 즉, 페이지 조회수는 100회이지만 방문자는 단 한 명입니다(매번 동일한 컴퓨터를 사용하여 페이지에 액세스한다고 가정). 그러나 사용자가 페이지와 다시 상호 작용하기 전에 페이지에서 최소 30분 동안 활동이 없으면 단일 사용자에 대한 고유 보기가 기록되지 않습니다.

### 바운스 비율

이탈률은 초기 페이지 로드 후 페이지 상호작용(페이지 스크롤 포함) 없이 사이트를 떠난 페이지 방문자의 비율입니다. 일별 요율(일당 백분율)로 계산되며, 선택한 기간 동안의 일별 요율 추세선이 표시됩니다.

### 페이지에 머문 시간

페이지에 머문 시간은 매일 모든 보기에 대해 페이지에서 보낸 평균 시간을 계산합니다. 선택한 기간 동안 표시됩니다.

이 측정항목은 24시간마다 다음과 같이 계산됩니다.

```
(view-1-time + view-2-time + ...) / total-number-views
```

### 입구

**Entrances** 는 해당 페이지에서 귀하의 사이트에 도착한 방문자 수입니다(다른 페이지에서 시작하여 해당 페이지를 클릭한 방문자와 반대).

### 종료율

종료율은 페이지와 상호작용(예: 스크롤 또는 클릭)한 후 사이트를 떠난 방문자의 비율입니다.

## 청중

잠재고객 보고서는 차트를 사용하여 페이지와 잠재고객의 상호 작용에 대한 정보를 제공합니다. 다음 질문에 답합니다.

* 내 콘텐츠와 상호작용하는 사용자는 몇 명입니까?
* 이들 사용자 중 알려지거나 익명인 사용자는 몇 명입니까?
* 내 콘텐츠와 상호작용한 알려진 사용자 중 Analytics Cloud Segments에 속하는 사용자는 몇 명입니까?
* 세그먼트의 사용자 중 상위 15개 세그먼트는 무엇입니까?
* 잠재고객 보고서의 차트는 다음과 같습니다.

**방문자:** 도넛 차트는 알려졌거나 익명인 총 고유 방문자의 비율을 나타냅니다. 차트 중앙에는 총 순 방문자 수가 표시됩니다. 각 차트 섹션의 도구 설명에는 해당 섹션의 사용자 수(예: 알려진 사용자 또는 익명 사용자 수)가 표시됩니다.

```{note}
로그인 시 이메일 주소가 Analytics Cloud와 동기화된 사용자 데이터와 일치하면 개인이 알려진 것으로 간주됩니다. 신규 사용자가 사이트에 등록하면 해당 데이터가 Analytics Cloud와 동기화되고 알려진 개인으로 표시되는 데 최대 2시간이 걸릴 수 있습니다.
```

**세분화된 뷰어:** 도넛 차트는 하나 이상의 Analytics Cloud 세그먼트에 속하거나 속하지 않는 알려진 개인의 비율을 표시합니다. 차트 중앙에는 알려진 개인의 총 수가 표시됩니다. 각 차트 섹션의 도구 설명에는 해당 섹션의 사용자 수(예: 하나 이상의 세그먼트에 속한 사용자 수)가 표시됩니다.

**시청자 세그먼트:** 막대 차트는 각 세그먼트를 구성하는 알려진 개인의 비율을 보여줍니다. 차트에는 상위 5개 세그먼트 각각에 대한 막대가 표시되고 나머지 세그먼트는 마지막 막대에 집계됩니다. 마지막 막대의 도구 설명에는 나머지 각 세그먼트의 값이 표시됩니다.

## 기술별 견해

기본 탭에서 운영 체제별(장치 유형별로 그룹화) 페이지 보기의 누적 막대 그래프를 봅니다. 각 막대 위로 마우스를 가져가면 데이터의 세부 분석을 볼 수 있습니다.

![Users are segmented by operating system and device type.](pages/images/04.png)

웹 브라우저를 클릭하면 선택한 기간 동안 상위 8개 웹 브라우저를 표시하는 도넛 차트를 볼 수 있습니다. 해당하는 경우 나머지 웹 브라우저는 9번째 도넛 세그먼트에 집계됩니다.

![Learn how many users are distributed across different browser agents.](pages/images/05.png)

이러한 기본 측정항목을 모두 마친 후에는 실제로 페이지에 대해 알게 됩니다. 하지만 아직 보지 못한 흥미로운 페이지 데이터가 있습니다. 사람들이 처음에 어떻게 페이지를 방문했는지 알아보세요. 다음으로 [경로분석](./paths.md) 에 대해 알아보세요.
